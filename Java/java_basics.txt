[Beginner] What is a varargs parameter?	A method parameter that accepts a variable number of arguments, declared with `...` (e.g., `void log(String... msgs)`).

# === String & Type System ===
[Intermediate] What is the difference between `String`, `StringBuilder`, and `StringBuffer`?	`String` is immutable — every modification creates a new object; thread-safe by nature and stored in the String pool if a literal. `StringBuilder` is mutable, not thread-safe, O(1) amortized append — use in single-threaded string building. `StringBuffer` is mutable and thread-safe via synchronized methods but slower than `StringBuilder`. Use `StringBuilder` for most string building. The JVM compiler uses `StringBuilder` / `StringConcatFactory` (Java 9+, via `invokedynamic`) for `+` concatenation.
[Intermediate] What is the Integer cache in Java and why does it matter?	Java caches `Integer` objects for values -128 to 127 in a static array. `Integer.valueOf(100)` always returns the same cached object so `==` holds; `Integer.valueOf(200)` returns a new object, so `==` fails. Similar caches exist for `Boolean`, `Byte`, `Short`, `Long`, and `Character` (0–127). Lesson: always use `.equals()` when comparing `Integer` values; never rely on `==` outside the cached range. The upper bound can be raised with `-XX:AutoBoxCacheMax=<max>`.
[Intermediate] What are the pitfalls of autoboxing and unboxing?	(1) `NullPointerException` when unboxing a null wrapper: `Integer i = null; int x = i;` throws NPE. (2) Performance: wrapper object creation in tight loops — use `IntStream` or `int[]` for numeric work. (3) Unexpected `==` comparisons: autoboxed `Integer` values outside -128..127 produce different objects; use `.equals()`. (4) Confusing overloads: `list.remove(1)` calls `remove(int index)` not `remove(Object)` — use `list.remove(Integer.valueOf(1))` to remove by value.
[Intermediate] What is `var` (local variable type inference) in Java 10+?	`var` instructs the compiler to infer the local variable's type from its initializer. `var list = new ArrayList<String>()` infers `ArrayList<String>`. Still statically typed. Restrictions: only local variables with initializers; not fields, method parameters, return types, or catch variables. Java 11 allows `var` in lambda parameters for annotations: `(@NonNull var x) -> x`. Use when the type is obvious from context (`var map = new HashMap<String, List<Integer>>()`); avoid when the right-hand side doesn't make the type clear — readability suffers.
[Intermediate] What are Text Blocks (Java 15+) and how does indentation stripping work?	Text blocks are multi-line string literals delimited by `"""`. The compiler strips *incidental* whitespace: it finds the leftmost non-whitespace column across all content lines (and the closing `"""` if not on its own line) and removes that many leading spaces from every line. New escapes: `\` at end of line = line continuation (no newline emitted), `\s` = explicit trailing space (prevents stripping). Ideal for embedding JSON, SQL, HTML. The result is a plain `String` — no new type. Example: `String sql = """
    SELECT *
    FROM users
    """;` produces `SELECT *\nFROM users\n`.

[Intermediate] What are Java's operators and operator categories?	Arithmetic: `+`, `-`, `*`, `/`, `%` (modulus), `++`, `--` (pre/post). Relational: `==`, `!=`, `<`, `>`, `<=`, `>=` — produce `boolean`. Logical: `&&` (short-circuit AND), `||` (short-circuit OR), `!` (NOT). Bitwise: `&` (AND), `|` (OR), `^` (XOR), `~` (complement), `<<` (left shift), `>>` (signed right shift), `>>>` (unsigned right shift). Ternary: `condition ? val1 : val2`. Assignment operators: `+=`, `-=`, `*=`, `/=`, `%=`, `&=`, `|=`, `^=`, `<<=`, `>>=`. Operator precedence (high→low, partial): postfix `++`/`--` > unary > `*`/`/`/`%` > `+`/`-` > shifts > relational > equality > bitwise > logical > ternary > assignment.
[Intermediate] What are Java's control flow constructs?	`if/else if/else` — branches on a `boolean` expression. `switch` statement — matched on `int`, `char`, `enum`, or `String`; cases fall through unless `break` is used; `default` is the catch-all. `switch` expression (Java 14+) — see separate card. `for` loop — classic: `for (init; condition; update)`. Enhanced for-each: `for (Type x : collection)` — works on arrays and `Iterable`. `while` — pre-condition, may never execute. `do-while` — post-condition, executes at least once. `break` — exits the nearest enclosing loop or switch. `continue` — skips to the next iteration. Labeled `break`/`continue` — target an outer loop by label. `return` — exits the current method.
[Intermediate] How do you declare and use arrays in Java?	Declaration: `int[] arr;` or (old style) `int arr[]`. Initialization: `int[] a = new int[5]` — zero-filled; `int[] b = {1, 2, 3}` — array literal. Multi-dimensional: `int[][] matrix = new int[3][4]`; jagged arrays allowed: `int[][] jagged = new int[3][]; jagged[0] = new int[2];`. Access: zero-indexed — `arr[0]` through `arr[arr.length - 1]`. `ArrayIndexOutOfBoundsException` on invalid index. Arrays are objects on the heap; length accessible via `.length` (field, not method). `Arrays.sort(arr)` — in-place sort. `Arrays.copyOf(arr, len)` — copy with new length. `Arrays.asList(arr)` — fixed-size `List` wrapper (for object arrays only). Prefer `ArrayList` over arrays when size changes dynamically.
[Intermediate] What `String` methods were added in Java 11+?	`isBlank()` — returns `true` if the string is empty or contains only whitespace (`Character.isWhitespace()` — Unicode-aware). `strip()` / `stripLeading()` / `stripTrailing()` — Unicode-aware whitespace removal (contrast `trim()` which only removes `<= \u0020`). `lines()` — returns a lazy `Stream<String>` of lines, recognising `\n`, `\r`, `\r\n`. `repeat(n)` — returns the string concatenated with itself `n` times (`"ab".repeat(3)` → `"ababab"`); `n=0` returns `""`. These complement the earlier `String` methods like `contains()`, `startsWith()`, `replace()`, and the Java 8+ `String.join()` and `String.chars()` stream.
[Intermediate] What are `String.indent()` and `String.transform()` (Java 12+)?	`String.indent(n)` (Java 12+) — adjusts the indentation of every line: positive `n` prepends `n` spaces; negative `n` removes up to `n` leading spaces. A trailing newline is always appended if not present. Useful for generating indented output. `String.transform(Function<String,R> f)` (Java 12+) — applies a function to the string and returns the result; enables chaining without a local variable: `"  hello  ".strip().transform(s -> s + "!")`. Neither is widely used but both appear in the text-block support toolset.
[Intermediate] What are `String.formatted()`, `stripIndent()`, and `translateEscapes()` (Java 15+)?	`String.formatted(Object... args)` (Java 15+) — instance method equivalent of `String.format(fmt, args)`; improves readability when building formatted strings from a template variable: `template.formatted(name, value)`. `String.stripIndent()` (Java 15+) — removes consistent incidental leading whitespace from all lines (same algorithm as text blocks). `String.translateEscapes()` (Java 15+) — processes Java escape sequences (`\n`, `\t`, `\\`, etc.) within a string; makes string content read directly from files behave like compiled string literals. These three methods were introduced specifically to support text block (`"""`) processing.	Switch expressions produce a value and use arrow labels (`case X -> expr`) or `yield` in colon-style arms. Arrow labels have no fall-through — each arm is independent. The compiler enforces exhaustiveness (all enum values or a `default` must be covered). Example: `int days = switch(month) { case JAN, MAR -> 31; case FEB -> 28; default -> 30; };`. Switch statements are void — they execute side effects and can fall through between cases. Switch expressions can also be used as statements. Pattern matching in switch was finalized in Java 21.

[Pro] When should you use `String.intern()` and what are the caveats?	`String.intern()` returns the pool's canonical copy of the string (adds it if not present). Benefits: memory savings when many strings share the same value (e.g., repeated keys or identifiers). Caveats: `intern()` is relatively expensive (hashing, synchronization) — only intern strings that will have many duplicates with long lifetimes. `new String("abc")` creates a new object outside the pool; string literals are automatically interned. In Java 8+, the String Pool resides in the heap and is GC'd like normal objects (unlike PermGen in Java 7-). For large-scale deduplication, consider `String.intern()` or, from Java 8, `StringTable` GC options like `-XX:+UseStringDeduplication` (G1GC only).
