# === HTTP ===
What are the main HTTP methods and their semantics?	`GET` — retrieve a resource; safe and idempotent; no body. `POST` — submit data to create or trigger an action; not idempotent. `PUT` — replace a resource entirely at the given URI; idempotent. `PATCH` — partially update a resource; not necessarily idempotent. `DELETE` — remove the resource; idempotent. `HEAD` — like GET but returns only headers (used to check existence/metadata). `OPTIONS` — returns supported methods (used in CORS preflight). Safe methods (GET, HEAD) must not change server state.
What are the key HTTP status code categories?	1xx Informational: `100 Continue`. 2xx Success: `200 OK`, `201 Created`, `204 No Content`. 3xx Redirection: `301 Moved Permanently`, `302 Found`, `304 Not Modified`. 4xx Client errors: `400 Bad Request`, `401 Unauthorized` (unauthenticated), `403 Forbidden` (authenticated but not authorized), `404 Not Found`, `409 Conflict`, `422 Unprocessable Entity`, `429 Too Many Requests`. 5xx Server errors: `500 Internal Server Error`, `502 Bad Gateway`, `503 Service Unavailable`, `504 Gateway Timeout`.
What are the components of a URL?	`scheme://userinfo@host:port/path?query#fragment`. Example: `https://user:pass@api.example.com:8443/v1/users?active=true&role=admin#section2`. `scheme` — protocol (`https`, `ftp`). `host` — domain or IP. `port` — optional, default 443 for HTTPS, 80 for HTTP. `path` — resource location. `query` — key=value pairs after `?`, separated by `&`. `fragment` — client-side anchor after `#`, NOT sent to the server.

# === Servlets ===
What is the lifecycle of an `HttpServlet`?	(1) `init(ServletConfig)` — called once when the servlet is first loaded; use for one-time setup (DB connections, config). (2) `service(HttpServletRequest, HttpServletResponse)` — called for every HTTP request; routes to `doGet`, `doPost`, etc. based on method. (3) `destroy()` — called once when the servlet is unloaded; use for cleanup. Multiple requests may call `service()` concurrently — servlets must be thread-safe (avoid instance variables for request state).
What is the difference between `HttpServletRequest.getParameter()` and `getInputStream()`?	`getParameter(name)` reads URL query parameters or `application/x-www-form-urlencoded` body parameters as strings. Easy to use for simple form data. `getInputStream()` reads the raw request body as bytes — required for JSON, XML, file uploads (`multipart/form-data`), or binary data. You can call one or the other but NOT both (reading the input stream prevents parameter parsing). Use `getReader()` for the body as characters.
What are Servlet Filters and how do they work?	Filters implement `javax.servlet.Filter` and intercept requests/responses before they reach a servlet (or after). Configured in `web.xml` or with `@WebFilter`. The `doFilter(request, response, chain)` method can: pre-process the request (auth checks, logging, CORS headers), call `chain.doFilter(request, response)` to pass to the next filter/servlet, and post-process the response (compression, caching headers). Multiple filters form a chain executed in declaration order.
What is `HttpSession` and how does session management work?	`HttpSession` stores server-side state per user across multiple requests. Obtained via `request.getSession()` (creates if absent) or `request.getSession(false)` (returns null if absent). The session has a unique `sessionId` sent to the client as a cookie (`JSESSIONID`) or appended to URLs. Session attributes: `session.setAttribute("user", user)`, `session.getAttribute("user")`. Sessions expire after inactivity (`setMaxInactiveInterval`). Security: regenerate session ID after login to prevent session fixation (`request.changeSessionId()` in Servlet 3.1+).
What are Servlet Listeners and what events do they observe?	Listeners are classes that observe lifecycle events in the Servlet container without inserting themselves into the request processing chain (unlike Filters). Key listener interfaces: `ServletContextListener` — `contextInitialized()` and `contextDestroyed()`: fired when the webapp starts/stops; use for application-wide setup and teardown (database connection pools, caches). `HttpSessionListener` — `sessionCreated()` and `sessionDestroyed()`: use for tracking active sessions. `HttpSessionAttributeListener` — fires when session attributes are added, removed, or replaced. `ServletRequestListener` — fires on each request start/end. Register with `@WebListener` or in `web.xml`.

# === REST ===
What are the principles of RESTful API design?	(1) Client-Server — UI separate from data storage. (2) Stateless — each request must contain all information needed; server stores no session state. (3) Cacheable — responses must define themselves as cacheable or not. (4) Uniform Interface — resources identified by URIs; manipulated through representations; self-descriptive messages; HATEOAS (Hypermedia as the Engine of Application State). (5) Layered System — client doesn't know if connected directly to server or intermediary. URL convention: nouns for resources (`/users/{id}`), HTTP verbs for operations, plural nouns.

# === JSON Processing ===
How do you serialize/deserialize JSON with Jackson?	Jackson's `ObjectMapper` is the entry point. Serialization (object → JSON): `String json = mapper.writeValueAsString(obj)`. Deserialization (JSON → object): `MyClass obj = mapper.readValue(json, MyClass.class)`. For generic types: `mapper.readValue(json, new TypeReference<List<User>>(){})`. Key annotations: `@JsonProperty("field_name")` — map field to a custom JSON key. `@JsonIgnore` — exclude a field. `@JsonInclude(NON_NULL)` — omit null fields. `@JsonFormat(pattern="yyyy-MM-dd")` — date format. Configure `ObjectMapper`: `mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false)`.

# === HttpClient (Java 11+) ===
How do you make HTTP requests with `java.net.http.HttpClient` (Java 11+)?
What is Jakarta EE and what are its core specifications?	Jakarta EE (formerly Java EE) is the community-governed enterprise Java platform — a set of specifications for building server-side applications. Core specs: **EJB (Enterprise JavaBeans)** — server-managed components (stateless/stateful session beans, message-driven beans); now largely superseded by CDI + Spring. **CDI (Contexts and Dependency Injection)** — standardises IoC and component lifecycle (`@ApplicationScoped`, `@RequestScoped`, `@Inject`). **JPA (Jakarta Persistence API)** — ORM specification; Hibernate, EclipseLink are implementations. **JAX-RS (Jakarta REST)** — annotation-driven REST/HTTP services (`@Path`, `@GET`, `@Produces`). **JSF (Jakarta Faces)** — component-based web UI framework. **JTA (Jakarta Transactions)** — distributed transaction API. Implementations: WildFly, Open Liberty, Payara, Quarkus (for MicroProfile). Spring is not a Jakarta EE implementation but adopts many of the same concepts.	`HttpClient` is the modern replacement for `HttpURLConnection`. Create (immutable, reusable): `HttpClient client = HttpClient.newHttpClient()` or `HttpClient.newBuilder().connectTimeout(Duration.ofSeconds(10)).build()`. Build a request: `HttpRequest request = HttpRequest.newBuilder(URI.create("https://api.example.com/users")).GET().header("Accept","application/json").build()`. Send synchronously: `HttpResponse<String> resp = client.send(request, BodyHandlers.ofString())`. Send asynchronously: `client.sendAsync(request, BodyHandlers.ofString()).thenApply(HttpResponse::body).thenAccept(System.out::println)`. Status code: `resp.statusCode()`. Body: `resp.body()`.
