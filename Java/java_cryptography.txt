# Java Cryptography & Hashing
# Tier: Intermediate
# Format: Front[TAB]Back

# === Concepts ===
[Beginner] What is the difference between hashing, encoding, and encryption?	Hashing: one-way mathematical transformation producing a fixed-size digest. Cannot be reversed. Used for integrity checks and password storage. Examples: SHA-256, bcrypt. Encoding: reversible transformation of representation format — NOT a security measure. Anyone can decode it. Examples: Base64, URL encoding. Encryption: reversible transformation that requires a secret key to decrypt. Used for confidentiality. Examples: AES (symmetric), RSA (asymmetric). Common mistake: treating Base64 as security — it is not.
[Beginner] What is symmetric vs asymmetric encryption?	Symmetric (secret-key) encryption uses the SAME key for encryption and decryption. Fast, suitable for bulk data. Key challenge: securely exchanging the shared key. Example: AES. Asymmetric (public-key) encryption uses a KEY PAIR: anything encrypted with the public key can only be decrypted with the private key, and vice versa. Slower, used for key exchange and digital signatures. Example: RSA, EC. In practice: use asymmetric key exchange to establish a shared symmetric key, then use symmetric encryption for the data.
[Beginner] What is a cryptographic hash function and what properties must it have?	A cryptographic hash function maps arbitrary input to a fixed-size digest. Required properties: (1) Deterministic — same input always produces same output. (2) One-way (pre-image resistance) — infeasible to find input from output. (3) Collision resistance — infeasible to find two different inputs with the same hash. (4) Avalanche effect — small input change produces completely different output. Use SHA-256 or SHA-3 for integrity checking. DO NOT use MD5 or SHA-1 — they are cryptographically broken (collision attacks exist).
[Beginner] Why are MD5 and SHA-1 not suitable for password hashing even if they're not broken?	Even unbroken hash functions like SHA-256 are unsuitable for password hashing because they are designed to be FAST. An attacker with a GPU can compute billions of SHA-256 hashes per second, making brute-force and dictionary attacks trivial. Password hashing requires algorithms that are intentionally SLOW (configurable cost factor) and include a per-user random salt. Use: bcrypt (standard, widely supported), Argon2id (winner of Password Hashing Competition, recommended), or PBKDF2 (FIPS compliant). These make brute-force attacks computationally impractical.

# === Java Crypto APIs ===
[Intermediate] How do you compute a SHA-256 hash in Java?	Use `java.security.MessageDigest`: `MessageDigest md = MessageDigest.getInstance("SHA-256"); byte[] digest = md.digest(inputBytes);`. Convert to hex string for display: `HexFormat.of().formatHex(digest)` (Java 17+) or loop with `String.format("%02x", b)`. `MessageDigest` is NOT thread-safe — create a new instance per thread or synchronize. Available algorithms: `"SHA-256"`, `"SHA-384"`, `"SHA-512"`, `"SHA3-256"`, `"SHA3-512"`.
[Intermediate] How do you perform AES encryption/decryption in Java?	Use `javax.crypto.Cipher`. (1) Generate key: `KeyGenerator kg = KeyGenerator.getInstance("AES"); kg.init(256); SecretKey key = kg.generateKey();`. (2) Encrypt: `Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding"); cipher.init(Cipher.ENCRYPT_MODE, key, new GCMParameterSpec(128, iv)); byte[] ciphertext = cipher.doFinal(plaintext);`. (3) Decrypt: `cipher.init(Cipher.DECRYPT_MODE, key, new GCMParameterSpec(128, iv)); byte[] plaintext = cipher.doFinal(ciphertext);`. Always use GCM mode (authenticated encryption). Never use ECB mode — it leaks patterns.
[Intermediate] What is AES-GCM and why is it preferred over AES-CBC?	AES-GCM (Galois/Counter Mode) is an authenticated encryption mode that provides both confidentiality AND integrity/authentication in one operation. If the ciphertext is tampered with, decryption fails with an `AEADBadTagException`. AES-CBC provides confidentiality only — an attacker can modify the ciphertext without detection, enabling padding oracle attacks. For AES-GCM: use a 12-byte random IV (nonce), never reuse an IV with the same key (critical security property), use 128-bit authentication tag. `GCMParameterSpec(128, iv)`.
[Intermediate] How do you generate a secure random key or IV in Java?	Use `java.security.SecureRandom`, which uses a cryptographically strong random number generator (CSPRNG): `SecureRandom sr = new SecureRandom(); byte[] iv = new byte[12]; sr.nextBytes(iv);`. For key generation: `KeyGenerator kg = KeyGenerator.getInstance("AES"); kg.init(256, new SecureRandom()); SecretKey key = kg.generateKey();`. Never use `java.util.Random` or `Math.random()` for cryptographic purposes — they are predictable pseudo-random generators.
[Intermediate] What is an HMAC and when do you use it?	HMAC (Hash-based Message Authentication Code) is a keyed cryptographic hash that provides BOTH integrity (data wasn't modified) AND authenticity (it came from someone with the key). Unlike a plain hash, an HMAC requires the secret key to verify — an attacker who doesn't know the key cannot forge a valid MAC. In Java: `Mac mac = Mac.getInstance("HmacSHA256"); mac.init(new SecretKeySpec(keyBytes, "HmacSHA256")); byte[] hmac = mac.doFinal(messageBytes);`. Use for signing API requests, verifying webhook payloads, securing cookies.
[Intermediate] How do you do password hashing in Java (bcrypt/PBKDF2)?	For PBKDF2 (built into JDK): `SecretKeyFactory skf = SecretKeyFactory.getInstance("PBKDF2WithHmacSHA256"); KeySpec spec = new PBEKeySpec(password, salt, 310_000, 256); byte[] hash = skf.generateSecret(spec).getEncoded();`. Iterations (310,000) should be tuned so hashing takes ~100ms on your hardware. For bcrypt: use Spring Security's `BCryptPasswordEncoder` or the jBCrypt library (not in JDK). For Argon2: use Bouncy Castle or Spring Security's `Argon2PasswordEncoder`. Store: `salt + hash` (salt is random per-user, NOT secret).
[Intermediate] What is `java.util.Base64` and how is it used?	`Base64` (Java 8+) encodes binary data as printable ASCII text and decodes it back. Three encoders/decoders: `Base64.getEncoder()` — standard Base64 (uses `+` and `/`; may include `=` padding). `Base64.getUrlEncoder()` — URL-safe Base64 (uses `-` and `_`; safe in URLs). `Base64.getMimeEncoder()` — MIME Base64 (line breaks every 76 chars for email). Usage: `String encoded = Base64.getEncoder().encodeToString(bytes)`. Decode: `byte[] bytes = Base64.getDecoder().decode(encodedStr)`. Base64 is NOT encryption — it is trivially reversible and provides zero security.
