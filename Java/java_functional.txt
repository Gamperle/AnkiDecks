# === Lambdas & Functional Interfaces ===
[Intermediate] What is a lambda expression in Java?	A lambda is a concise, anonymous function that can be used as a value ‚Äî assigned to a functional-interface variable, passed as a method argument, or returned from a method. Syntax: `(parameters) -> expression` or `(parameters) -> { statements; }`. Examples: `() -> 42`, `x -> x * 2`, `(a, b) -> a + b`. Lambdas capture variables from the enclosing scope only if those variables are effectively final (not reassigned).
[Intermediate] What is a functional interface?	An interface with exactly one abstract method (SAM ‚Äî Single Abstract Method). It may have any number of `default` and `static` methods. The `@FunctionalInterface` annotation enforces this at compile time. Used as the target type for lambdas and method references. Built-in examples in `java.util.function`: `Predicate<T>`, `Function<T,R>`, `Consumer<T>`, `Supplier<T>`, `Runnable`, `Callable<V>`.
[Intermediate] What are the four core functional interfaces in `java.util.function`?	`Predicate<T>` ‚Äî takes T, returns boolean; used for testing/filtering. Key methods: `test()`, `and()`, `or()`, `negate()`. `Function<T,R>` ‚Äî takes T, returns R; used for transformation. Key methods: `apply()`, `andThen()`, `compose()`. `Consumer<T>` ‚Äî takes T, returns void; used for side effects. Key methods: `accept()`, `andThen()`. `Supplier<T>` ‚Äî takes nothing, returns T; used for lazy creation. Key method: `get()`.
[Intermediate] What are the four types of method references in Java?	(1) Static method: `Integer::parseInt` ‚â° `x -> Integer.parseInt(x)`. (2) Bound instance method: `str::toUpperCase` ‚â° `() -> str.toUpperCase()` ‚Äî receiver is fixed. (3) Unbound instance method: `String::toUpperCase` ‚â° `s -> s.toUpperCase()` ‚Äî receiver is first argument. (4) Constructor reference: `ArrayList::new` ‚â° `() -> new ArrayList<>()`. Use method references when the lambda body does nothing but delegate to an existing method ‚Äî they're more readable.
[Intermediate] What is `Predicate.not()` (Java 11+) and when is it useful?	`Predicate.not(predicate)` creates the logical negation of the given predicate as a static factory. Particularly useful for negating method references, which cannot be negated with `!` directly. Example: `stream.filter(Predicate.not(String::isBlank))` is cleaner than `stream.filter(s -> !s.isBlank())`. Also works with any predicate: `Predicate.not(list::contains)`.
[Intermediate] What does `var` in lambda parameters enable (Java 11+)?	Using `var` for lambda parameters allows adding annotations to them, which is impossible with inferred types without `var`. Example: `(@NotNull var s) -> s.length()`. The `var` type is still inferred ‚Äî it doesn't change runtime behaviour. All parameters must consistently either use `var` or not; mixing `var` with explicit types or untyped params is a compile error.

# === Streams ===
[Intermediate] What is a Stream in Java and what are its key characteristics?	`Stream<T>` (Java 8+) is a declarative pipeline for processing sequences of elements from a source (collection, array, I/O channel, generator). Key properties: (1) Lazy ‚Äî intermediate operations do not execute until a terminal operation is called. (2) Single-use ‚Äî a stream cannot be consumed twice; attempting to reuse throws `IllegalStateException`. (3) Non-mutating ‚Äî stream operations do not modify the source. (4) Optionally parallel ‚Äî `parallelStream()` or `.parallel()`. (5) Short-circuiting ‚Äî `limit()`, `findFirst()`, `anyMatch()` etc. can stop processing early.
[Intermediate] What is the difference between intermediate and terminal stream operations?	Intermediate operations are lazy ‚Äî they return a new Stream and build the pipeline without executing anything. Examples: `filter`, `map`, `flatMap`, `sorted`, `distinct`, `limit`, `skip`, `peek`, `mapToInt`. Terminal operations are eager ‚Äî they trigger the whole pipeline and produce a result or side-effect. Examples: `collect`, `forEach`, `reduce`, `count`, `findFirst`, `anyMatch`, `min`, `max`, `toList`. A stream pipeline must have exactly one terminal operation and may have zero or more intermediate operations.
[Intermediate] What is the difference between `map()` and `flatMap()` in streams?	`map(f)` applies function f to each element, producing exactly one output element per input: `Stream<R>`. If f returns a collection, you get `Stream<List<R>>` ‚Äî nested. `flatMap(f)` applies f to each element where f returns a `Stream<R>`, then flattens all the resulting streams into a single `Stream<R>`. Use `flatMap` when your mapping function produces a stream or collection per element. Example: `lines.flatMap(line -> Arrays.stream(line.split(" ")))` produces a stream of all words.
[Intermediate] What does `Stream.toList()` (Java 16+) return, and how does it differ from `Collectors.toList()`?	`Stream.toList()` is a convenient terminal operation that returns an **unmodifiable** `List` containing the stream elements in encounter order. It cannot be modified after creation (throws `UnsupportedOperationException`). `Collectors.toList()` returns a **mutable** `ArrayList` that can be modified. This is a subtle but important difference. Prefer `toList()` when you don't need to modify the result ‚Äî it's shorter and communicates immutability intent.
[Intermediate] What are `takeWhile()` and `dropWhile()` in streams (Java 9+)?	`takeWhile(predicate)` returns elements from the stream while the predicate is true, stopping at the first element that fails it (short-circuit for ordered streams). `dropWhile(predicate)` skips elements while the predicate is true, then passes all remaining elements. Both are analogues of their list-processing counterparts. For unordered/parallel streams, behaviour is non-deterministic. Example: `IntStream.of(1,2,3,4,5).takeWhile(n -> n < 4)` produces `1, 2, 3`.
[Intermediate] What does `Collectors.groupingBy()` do?	Groups stream elements by a classifier function, producing a `Map<K, List<V>>`. Example: `people.stream().collect(groupingBy(Person::getDepartment))`. A downstream collector can be chained for further aggregation: `groupingBy(Person::getDept, counting())` ‚Üí `Map<String, Long>`. `groupingBy(Person::getDept, mapping(Person::getName, toList()))` ‚Üí `Map<String, List<String>>`. Multi-level: `groupingBy(Person::getDept, groupingBy(Person::getCity))` ‚Üí `Map<String, Map<String, List<Person>>>`.
[Intermediate] What does `Collectors.teeing()` do (Java 12+)?	`Collectors.teeing(downstream1, downstream2, merger)` collects the stream into two downstream collectors simultaneously in a single pass, then combines both results using a `BiFunction` merger. Useful for computing two different aggregates in one stream pass. Example: compute both min and max salary: `stream.collect(teeing(minBy(comparingInt(Employee::getSalary)), maxBy(comparingInt(Employee::getSalary)), (min, max) -> new SalaryRange(min, max)))`.
[Intermediate] What are `Stream.gather()` and `Gatherer` (Java 24+)?	`Stream.gather(Gatherer)` is a new intermediate operation that applies a custom stateful transformation to a stream ‚Äî filling the gap between what built-in intermediate ops can express. A `Gatherer` has four optional components: `initializer` (creates state), `integrator` (processes one element, can emit 0..n downstream elements), `combiner` (merges state for parallel), `finisher` (emits final elements after all input). Built-in `Gatherers` utility class provides: `windowFixed(n)`, `windowSliding(n)`, `scan(identity, accumulator)`, `fold(identity, accumulator)`, `mapConcurrent(n, mapper)`.
[Intermediate] What is `Optional<T>` and how should it be used?	`Optional<T>` is a container that may or may not hold a non-null value. Its purpose is to explicitly represent optional return values ‚Äî as a better alternative to returning `null`. Correct uses: as a method return type for values that may be absent. Incorrect uses: as a method parameter, as a field type, in collections. Key methods: `isPresent()`, `isEmpty()` (Java 11+), `get()` (avoid: throws if empty), `orElse(default)`, `orElseGet(Supplier)`, `orElseThrow()`, `map(Function)`, `flatMap(Function)`, `ifPresent(Consumer)`, `ifPresentOrElse(Consumer, Runnable)` (Java 9+), `or(Supplier<Optional>)` (Java 9+), `stream()` (Java 9+).
What are primitive streams (`IntStream`, `LongStream`, `DoubleStream`) and why do they exist?
[Intermediate] What are the `Optional` additions in Java 9+ and 11+?	Java 9 added three methods: `or(Supplier<Optional<T>>)` ‚Äî returns itself if present, otherwise the Optional produced by the supplier (compose fallback Optionals without unwrapping). `stream()` ‚Äî returns a `Stream<T>` of zero or one element; allows Optional to be flattened into a stream pipeline: `list.stream().flatMap(Optional::stream)`. `ifPresentOrElse(Consumer, Runnable)` ‚Äî runs the consumer if present, the empty-action if absent. Java 11 added `isEmpty()` ‚Äî the clean negation of `isPresent()`; more readable than `!opt.isPresent()`.
[Intermediate] What is `Optional.isEmpty()` (Java 11+) and why is it preferred?	`Optional.isEmpty()` returns `true` when the Optional holds no value. It is the logical negation of `isPresent()`. Before Java 11 you had to write the awkward `!opt.isPresent()`. With `isEmpty()` the intent reads naturally in conditions: `if (opt.isEmpty()) return;`. No functional difference ‚Äî it is purely a readability improvement. Consistent with `Collection.isEmpty()` / `String.isBlank()` patterns already in the JDK.
[Intermediate] What are parallel streams and when should you use them (and avoid them)?	Parallel streams split the source, process chunks concurrently using `ForkJoinPool.commonPool()`, and combine results. Enable with `collection.parallelStream()` or `stream.parallel()`. **When to use:** CPU-bound operations on large data sets (the overhead of splitting/combining is amortised only with enough work per element). **When to avoid:** (1) Small data ‚Äî sequential is faster due to overhead. (2) Ordered pipelines with `findFirst`, `forEachOrdered` ‚Äî ordering requires extra synchronisation. (3) Stateful lambdas or shared mutable state ‚Äî race conditions. (4) I/O-bound work ‚Äî threads block and pollute commonPool (use virtual threads instead). (5) `forEach` with side effects ‚Äî non-deterministic ordering breaks correctness. Always benchmark ‚Äî parallel is not automatically faster.	Primitive streams are specialised stream variants that avoid boxing/unboxing overhead for `int`, `long`, and `double`. `IntStream`, `LongStream`, `DoubleStream` have additional numeric terminal operations: `sum()`, `average()` (returns `OptionalDouble`), `min()`, `max()`, `summaryStatistics()`. Created via `IntStream.range(0, 10)`, `IntStream.of(1,2,3)`, or by mapping: `stream.mapToInt(String::length)`. Convert back to `Stream<T>` with `boxed()` or `mapToObj()`.

[Intermediate] What is the difference between `findFirst()` and `findAny()` in Streams?	Both return an `Optional<T>` and short-circuit the stream. `findFirst()` always returns the first element in encounter order ‚Äî deterministic. `findAny()` returns any element ‚Äî in sequential streams it usually returns the first, but in parallel streams it returns whichever thread finds one first, making it faster at the cost of non-determinism. Prefer `findAny()` in parallel streams where order doesn't matter.
[Intermediate] What is the difference between `Stream.of()` and `Arrays.stream()` for primitive arrays?	`Arrays.stream(arr)` handles primitive arrays (`int[]`, `long[]`, `double[]`) by returning specialized primitive streams (`IntStream`, `LongStream`, `DoubleStream`) ‚Äî no boxing overhead. Also supports sub-arrays: `Arrays.stream(arr, start, end)`. `Stream.of(arr)` treats a primitive array as a single element, returning `Stream<int[]>`. For object arrays both work the same, but `Arrays.stream()` is idiomatic. For primitives, always prefer `Arrays.stream()` to avoid boxing overhead.
[Intermediate] What is `reduce()` in Streams and when should you use `collect()` instead?	`reduce()` folds all stream elements into a single result using an associative accumulation function. Three forms: (1) `reduce(identity, BinaryOperator)` ‚Äî always returns T. (2) `reduce(BinaryOperator)` ‚Äî returns `Optional<T>` (empty stream safe). (3) `reduce(identity, BiFunction, BinaryOperator)` ‚Äî for parallel streams combining different types. Example: `stream.reduce(0, Integer::sum)`. For mutable accumulation (e.g., building a collection), prefer `collect()` ‚Äî it uses a mutable container and is more efficient.
