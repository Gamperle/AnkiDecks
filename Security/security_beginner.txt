# Security Beginner Questions
# Format: Front[TAB]Back

# === General Security Fundamentals ===

What is the CIA Triad in information security?	The three core security principles: Confidentiality (only authorized access), Integrity (data is accurate and unaltered), Availability (systems are accessible when needed).
Which element of the CIA Triad does encryption primarily protect?	Confidentiality — encryption makes data unreadable to unauthorized parties.
Which element of the CIA Triad does a DDoS attack target?	Availability — it overwhelms a system to make it unavailable to legitimate users.
Which element of the CIA Triad does a checksum or hash verify?	Integrity — it confirms that data has not been tampered with or corrupted.
What is the Principle of Least Privilege?	Users, processes, and systems should have only the minimum permissions necessary to perform their tasks — nothing more.
Give an example of violating the Principle of Least Privilege.	Running a web application as the root/admin user — if compromised, the attacker has full system access instead of limited app-level access.
What is Defense in Depth?	A security strategy that uses multiple layers of controls (firewalls, encryption, authentication, monitoring, etc.) so that if one layer fails, others still protect the system.
What is "security through obscurity" and why is it insufficient?	Relying on secrecy of the system's design (e.g., hiding source code, using non-standard ports) as the primary defense. It's insufficient because attackers can discover hidden details; true security requires proper controls regardless of what an attacker knows.
What is the STRIDE threat model?	A framework for categorizing threats: Spoofing (impersonation), Tampering (modifying data), Repudiation (denying actions), Information Disclosure (data leaks), Denial of Service, Elevation of Privilege.
What is CVSS and what does a score of 9.8 indicate?	Common Vulnerability Scoring System — a standardized scale (0–10) for rating vulnerability severity. A 9.8 is Critical, meaning easily exploitable with severe impact.
What is a CVE?	Common Vulnerabilities and Exposures — a standardized identifier (e.g., CVE-2021-44228) for publicly known security vulnerabilities.
How do you read a CVE entry? What key information does it contain?	A CVE includes: a unique ID (CVE-YYYY-NNNNN), a description of the vulnerability, affected products/versions, CVSS severity score, and references to patches or advisories.
Which two CIA Triad properties often conflict in a highly available public system?	Availability and Confidentiality: maximizing availability (open access, redundancy, caching) can reduce confidentiality controls. Example: a public API must balance open access for uptime against strict data-access restrictions. Security design must explicitly address this tension.
What is "privilege creep" and how do you prevent it?	Privilege creep is the gradual accumulation of more access rights than needed, typically as users change roles over time without their old permissions being revoked. Prevent it with periodic access reviews (recertification), automated removal of permissions when roles change, and enforcing Least Privilege from the start.
Which STRIDE threat category does a DDoS attack fall into?	Denial of Service — it overwhelms a system to prevent legitimate users from accessing it, directly targeting the Availability property of the CIA Triad.
What is the difference between a CVE and a CWE?	A CVE (Common Vulnerabilities and Exposures) identifies a specific known vulnerability in a particular product (e.g., CVE-2021-44228 = Log4Shell in Log4j 2.x). A CWE (Common Weakness Enumeration) describes a class of software weakness (e.g., CWE-89: SQL Injection). CVEs are instances; CWEs are the categories they belong to.
What are the three metric groups in CVSS v3?	Base (inherent characteristics that don't change: attack vector, complexity, privileges required, user interaction, scope, impact), Temporal (time-dependent factors: exploit maturity, remediation level, report confidence), Environmental (organization-specific modifications to base metrics reflecting actual deployment context).
Give a real-world example of a security incident that compromised all three CIA Triad elements.	The 2017 Equifax breach: Confidentiality (143 million records exposed), Integrity (attackers could modify data during access), Availability (systems taken offline for remediation). A single web application vulnerability (Apache Struts) cascaded to impact all three elements.
What is the difference between a DoS and a DDoS attack?	A DoS (Denial of Service) originates from a single source. A DDoS (Distributed DoS) uses many compromised machines (a botnet) simultaneously, making it much harder to mitigate by blocking a single IP. DDoS attacks can generate terabits of traffic.
Why is a simple checksum (like CRC) not sufficient to prove data hasn't been tampered with by an attacker?	An attacker who modifies data can recalculate a new matching checksum. Checksums detect accidental corruption only. To detect intentional tampering, use a keyed hash (HMAC) or digital signature that requires a secret the attacker doesn't possess.
How does Least Privilege apply to database accounts used by applications?	The application's DB account should only have permissions for specific tables and operations it needs (e.g., SELECT on users, INSERT on orders) — never DBA or schema-alter privileges. If the app is compromised, the attacker can only perform the limited allowed operations.
What is "blast radius" in security and how does Least Privilege minimize it?	Blast radius is the scope of damage possible when a component is compromised. Least Privilege minimizes it by restricting what the compromised component can access — reducing impact from "entire system" to "only what was specifically authorized."
Name three specific layers in a Defense in Depth strategy for a web application.	Network layer (firewall, WAF), application layer (input validation, authentication, authorization), data layer (encryption at rest, access controls, backups). If the WAF is bypassed, input validation still blocks injection. If both fail, encryption protects stored data.
What is Kerckhoffs's principle and how does it relate to "security through obscurity"?	Kerckhoffs's principle states a cryptographic system should be secure even if everything about the system is public knowledge, except the secret key. It directly rejects security through obscurity — the algorithm itself should never need to be secret for the system to remain secure.
How do you use STRIDE in practice during a threat modeling session?	Draw a Data Flow Diagram (DFD) of the system, then apply each STRIDE category to each element: processes can suffer all six threats, data stores face Tampering/Information Disclosure/DoS, data flows face Tampering/Information Disclosure/DoS, and external entities face Spoofing/Repudiation.
What does a CVSS Base Score with Attack Vector "Network" and Attack Complexity "Low" tell you?	The vulnerability can be exploited remotely over the network without special conditions, prerequisites, or user interaction — the most dangerous combination. No physical access or specific configuration is needed, meaning any internet-connected instance is at risk.
Where can you look up CVE details and what national database aggregates them?	The National Vulnerability Database (NVD) at nvd.nist.gov aggregates CVE entries with CVSS scores, affected products (CPE), and references. CVE details are also published on cve.mitre.org, vendor-specific advisories, and security mailing lists.
What is CPE (Common Platform Enumeration) and how does it relate to CVEs?	CPE is a standardized naming scheme for identifying software products and versions (e.g., cpe:2.3:a:apache:log4j:2.14.1). CVE entries reference CPEs to precisely identify which products and versions are affected by the vulnerability, enabling automated matching.
What is an access recertification review and how often should it be performed?	A periodic review where managers verify that each team member's access rights are still appropriate for their current role. Typically quarterly for sensitive systems, annually for others. Unauthorized or outdated access is revoked immediately, preventing privilege creep.
How do you use CWE IDs in practice for secure development?	CWEs categorize findings in SAST tools, code reviews, and vulnerability databases. A development team can create a "CWE watch list" of weaknesses relevant to their stack (e.g., CWE-89 SQL Injection, CWE-79 XSS) and specifically test for those during development and review.
Why might the same CVE have different effective risk levels across different organizations?	Organizations apply Environmental CVSS metrics differently based on their specific deployment. A CVE affecting a critical internet-facing server with sensitive data receives a higher score than the same CVE on an internal test system. Mitigating controls, compensating factors, and asset value all change the effective risk.

# === Authentication & Authorization Basics ===

What is the difference between authentication and authorization?	Authentication verifies *who you are* (identity). Authorization determines *what you can do* (permissions). Authentication always comes first.
Why should passwords never be stored in plaintext?	If the database is breached, all passwords are immediately exposed. Hashing (with salt) ensures passwords can't be read even if stolen.
What is salting in password hashing?	Adding a unique random value (salt) to each password before hashing, so identical passwords produce different hashes. Prevents rainbow table and precomputation attacks.
What is Multi-Factor Authentication (MFA)?	Requiring two or more independent factors to verify identity: something you know (password), something you have (phone/token), something you are (biometric).
What is a session ID and why must it be kept secret?	A unique token issued after login that identifies the user's session. If stolen (via XSS, network sniffing, etc.), an attacker can impersonate the user.
What is Role-Based Access Control (RBAC)?	An access control model where permissions are assigned to roles (e.g., Admin, Editor, Viewer), and users are assigned to roles — not given individual permissions.
What is the difference between OAuth 2.0 and OpenID Connect?	OAuth 2.0 is an authorization framework (grants access to resources). OpenID Connect is an identity layer on top of OAuth 2.0 that also provides authentication (who the user is).
Why should you never send passwords over unencrypted HTTP?	An attacker on the network can read the password in plaintext via packet sniffing. Always use HTTPS to encrypt credentials in transit.
What is IDOR and which security principle does it violate?	Insecure Direct Object Reference — accessing another user's resource by changing an identifier in the request (e.g., changing `?invoice_id=1001` to `?invoice_id=1002`). It violates authorization: the server fails to verify that the authenticated user is permitted to access the requested object.
Which password hashing algorithms are safe for storing passwords today?	bcrypt, scrypt, and Argon2 (recommended). They are slow by design, automatically salted, and resist GPU/ASIC-based cracking. MD5, SHA-1, and plain SHA-256 are NOT safe for passwords — they are far too fast and enable brute-force at scale.
What is the difference between a salt and a pepper in password hashing?	A salt is a random per-user value stored alongside the hash — it prevents precomputation (rainbow table) attacks. A pepper is a secret server-side value (NOT stored in the database) added before hashing — it protects hashes even if the database is breached, but must be stored separately in a secret manager.
What are the main weaknesses of SMS-based MFA?	SIM swapping (attacker convinces the carrier to transfer your number), SS7 protocol vulnerabilities (allow interception of SMS), phishing (victim is tricked into forwarding the code), and malware on the device reading SMS messages. Prefer TOTP apps (Google Authenticator) or hardware security keys.
What is ABAC (Attribute-Based Access Control) and how does it differ from RBAC?	ABAC grants access based on attributes of the user, resource, and environment (e.g., "managers can access salary records for their own department during business hours"). RBAC uses predefined roles. ABAC is more flexible and fine-grained but more complex to administer and audit.
What happens if you implement authorization without proper authentication?	Authorization checks become meaningless — an attacker can impersonate any user and bypass all permission checks. Authentication must establish a trusted identity first; only then can authorization enforce what that identity is allowed to do.
What is a rainbow table attack and how does salting defeat it?	A rainbow table is a precomputed lookup table mapping common passwords to their hashes. Without salt, an attacker can instantly look up any hash. Salting adds a unique random value per user, so even identical passwords produce different hashes — making precomputed tables completely useless.
Why must each user have a unique salt rather than one application-wide salt?	A single application salt means identical passwords still produce identical hashes — an attacker can detect users sharing the same password. Per-user salts ensure every hash is unique even for the same password, forcing attackers to crack each hash individually.
Why is MFA dramatically more secure than a strong password alone?	Even the strongest password can be phished, keylogged, or breached from another site. MFA requires a second independent factor (a physical device or biometric) the attacker doesn't have, making stolen credentials alone insufficient for access.
How should a server generate and manage session IDs securely?	Use a CSPRNG with at least 128 bits of entropy. Set HttpOnly, Secure, and SameSite attributes on session cookies. Regenerate the session ID after login. Set idle and absolute session timeouts. Invalidate sessions on logout server-side.
What problem does RBAC solve compared to assigning permissions directly to individual users?	Direct user-permission assignments don't scale — managing permissions for hundreds of users individually is error-prone and hard to audit. RBAC creates reusable roles (Admin, Editor, Viewer) with defined permissions, making management, auditing, and onboarding/offboarding much simpler.
What are the key tokens in OpenID Connect and what is each used for?	The ID Token (JWT) contains user identity claims (who they are) — consumed by the client app. The Access Token grants access to protected resources (what they can do) — sent to the resource server. The Refresh Token obtains new access tokens without re-authentication.
What is a man-in-the-middle (MitM) attack and how does HTTPS prevent it?	A MitM intercepts communication between two parties, reading or modifying data in transit. HTTPS prevents this with TLS: the server's certificate proves its identity (preventing impersonation), and encryption prevents reading or modifying the data in transit.
How do UUIDs help mitigate IDOR, and why aren't they sufficient alone?	UUIDs are non-sequential and hard to guess, making enumeration impractical (unlike auto-incrementing IDs). However, they only provide security through obscurity — a leaked UUID still grants access. Server-side authorization checks verifying the user is permitted to access the resource are always required.
What does "work factor" or "cost factor" mean in bcrypt?	The cost factor (rounds parameter, e.g., bcrypt cost 12) controls how many iterations the algorithm performs, exponentially increasing computation time. Higher cost = slower hashing = harder to brute-force. Tune it so hashing takes ~100-500ms on your hardware, and increase over time as hardware improves.
If the pepper is lost or destroyed, what happens to all stored passwords?	All passwords become unverifiable — no one can log in because the correct hash can never be reproduced without the pepper. This is why the pepper must be backed up in a separate secret manager (HSM, Vault) with proper redundancy.
What is TOTP (Time-based One-Time Password) and why is it more secure than SMS codes?	TOTP generates codes using a shared secret and the current time (RFC 6238), changing every 30 seconds. More secure because: the shared secret stays on the device (not transmitted like SMS), works offline, and is not vulnerable to SIM swapping or SS7 interception.
When should you choose ABAC over RBAC for access control?	When access decisions depend on dynamic context (time, location, resource classification, department) or require fine granularity (e.g., "doctors can access medical records only for their own patients during their shift"). If simple role-based rules suffice, RBAC is easier to manage and audit.

# === TLS & Transport Security Basics ===

What is TLS and what does it provide?	Transport Layer Security — a cryptographic protocol that provides confidentiality (encryption), integrity (tamper detection), and authentication (server identity verification) for network communication.
Why did TLS replace SSL?	SSL (Secure Sockets Layer) had critical vulnerabilities (e.g., POODLE). TLS is the successor with stronger algorithms and improved protocol design. "SSL" is often used colloquially but modern connections use TLS.
Which TLS versions are considered safe to use today?	TLS 1.2 and TLS 1.3. TLS 1.0 and 1.1 are deprecated due to known weaknesses; SSL 3.0 is completely broken.
What key improvement does TLS 1.3 have over TLS 1.2?	TLS 1.3 reduces the handshake to 1-RTT (one round trip), removes insecure algorithms (RSA key exchange, CBC ciphers), and supports 0-RTT resumption for faster connections.
What is the role of asymmetric encryption in a TLS connection?	Asymmetric encryption (e.g., RSA, ECDH) is used during the handshake to securely exchange or derive the symmetric session key. The actual data transfer uses faster symmetric encryption.
What does a TLS/SSL certificate prove?	It proves the server's identity — a trusted Certificate Authority (CA) has verified that the certificate holder controls the domain. It also contains the server's public key for encryption.
What is a Certificate Authority (CA)?	A trusted organization that issues digital certificates after verifying the identity of the certificate requester. Browsers and operating systems ship with a list of trusted root CAs.
What is a trust anchor?	A root CA certificate that is pre-installed in the operating system or browser's trust store. It's the starting point for validating certificate chains.
How does HTTPS work at a high level?	The browser connects to the server, performs a TLS handshake (verifies certificate, agrees on encryption), then all HTTP data is encrypted inside the TLS tunnel.
What does the padlock icon in the browser address bar guarantee?	That the connection is encrypted with TLS and the server's certificate was issued by a trusted CA for that domain. It does NOT guarantee the website is safe or legitimate.
What is the difference between a self-signed certificate and a CA-signed certificate?	A self-signed cert is signed by its own creator (not trusted by browsers by default). A CA-signed cert is signed by a trusted CA and is automatically trusted by browsers.
What is Let's Encrypt?	A free, automated, open Certificate Authority that provides domain-validated TLS certificates. It uses the ACME protocol for automated issuance and renewal (every 90 days).
What does "certificate expired" mean and what should you do?	The certificate's validity period has passed. Browsers will show a warning. The server admin needs to renew the certificate. Users should not bypass the warning.
What does a "hostname mismatch" TLS error mean?	The domain in the URL doesn't match any of the domains listed in the certificate (CN or SAN fields). This could indicate misconfiguration or an attack.
What data travels in plaintext with HTTP (no TLS)?	Everything: URLs, headers, cookies, form data (including passwords), request/response bodies. Anyone on the network can read and modify it.
What does TLS NOT protect against?	TLS only encrypts data in transit between endpoints. It does NOT protect against: malware on the client or server, server-side vulnerabilities (SQL injection, XSS), a compromised CA issuing fraudulent certificates, or attacks that occur after the data is decrypted on the server.
What is SSL stripping and how does HSTS prevent it?	SSL stripping downgrades an HTTPS connection to HTTP by intercepting the initial plain HTTP request before the redirect to HTTPS occurs. HSTS (HTTP Strict Transport Security) prevents this: once a browser has seen the HSTS header, it refuses to connect over HTTP at all and redirects internally. The HSTS preload list enforces this even on the very first visit.
When is using a self-signed certificate acceptable?	For internal tools, development environments, and private service-to-service communication where both parties can explicitly and deliberately trust the specific certificate. Never for public-facing sites — users cannot safely verify a self-signed cert and are exposed to MitM attacks without manual configuration.
What are the two main phases of a TLS connection?	The handshake phase (authenticate server, negotiate cipher suite, exchange keys) and the data transfer phase (encrypt/decrypt application data using the agreed symmetric key). The handshake uses slow asymmetric crypto; the data transfer uses fast symmetric crypto.
What was the POODLE attack and why did it end SSL 3.0?	POODLE exploited non-deterministic padding in SSL 3.0's CBC mode, allowing attackers to decrypt data byte-by-byte. It proved SSL 3.0 was fundamentally broken with no possible fix — the only mitigation was disabling SSL 3.0 entirely.
Why were TLS 1.0 and 1.1 deprecated?	Both support weak cipher suites (RC4, CBC without AEAD), lack modern defenses against padding oracle and downgrade attacks, and have structural weaknesses that can't be fully mitigated without a protocol redesign. All major browsers dropped support in 2020.
What cipher suites and features did TLS 1.3 remove compared to 1.2?	Static RSA key exchange (no forward secrecy), CBC cipher suites (padding oracle risk), RC4, DES, 3DES, MD5, SHA-1, compression (CRIME attack), and renegotiation. This dramatically reduces the attack surface and simplifies the protocol.
Why doesn't TLS use asymmetric encryption for all data transfer?	Asymmetric encryption (RSA, ECC) is ~1000x slower than symmetric encryption (AES). It's only used during the handshake to securely establish a shared symmetric key, which then encrypts the actual data at high speed and low overhead.
What are the three validation levels for TLS certificates?	Domain Validation (DV): proves domain control only. Organization Validation (OV): also verifies the legal identity of the organization. Extended Validation (EV): the most rigorous, verifies legal, physical, and operational existence. All three provide the same encryption strength.
What happens if a Certificate Authority is compromised?	A compromised CA can issue fraudulent certificates for any domain, enabling MitM attacks. The CA's root certificate must be revoked from all trust stores (browsers, OS). Certificate Transparency (CT) logs help detect such fraudulent issuance.
How many root CAs does a modern browser typically trust, and why is this a concern?	Modern browsers trust around 100-150 root CAs. This large trust base is a known weakness — any one compromised CA can issue valid certificates for any domain. Certificate Transparency partially mitigates this by making all issued certificates publicly auditable.
With HTTPS, what information is still visible to a network observer?	The destination IP address, the domain name (via DNS lookup and SNI in TLS handshake), connection timing, and approximate data volume. The full URL path, query parameters, headers, cookies, and body are all encrypted inside the TLS tunnel.
Why can a phishing site have a valid padlock (HTTPS)?	HTTPS only means the connection is encrypted and the certificate matches the domain. Services like Let's Encrypt issue DV certificates without verifying who owns the site or what it does. A phishing site at example-bank-login.com can get a valid cert — HTTPS protects the connection, not the legitimacy of the content.
What warning will a browser show for a self-signed certificate and why?	The browser shows "Your connection is not private" (NET::ERR_CERT_AUTHORITY_INVALID) because the certificate isn't signed by a trusted CA. The browser can't verify the server's identity, so a legitimate connection is indistinguishable from a MitM attack.
What is the ACME protocol used by Let's Encrypt?	Automatic Certificate Management Environment — a protocol for automating domain validation and certificate issuance. The server proves domain control via an HTTP challenge file at /.well-known/acme-challenge/ or a DNS TXT record. Tools like Certbot automate the entire process.
What is the typical maximum validity period for TLS certificates and why has it been shortened?	Currently 398 days (13 months). Shortened from multi-year periods to limit the damage window if a private key is compromised, and to ensure certificates reflect current organizational information. Shorter lifetimes encourage automation.
What are the CN and SAN fields in a TLS certificate?	CN (Common Name) is the legacy field for the domain name (deprecated for matching). SAN (Subject Alternative Name) lists all valid domains/IPs — modern browsers only check SANs. Multiple domains and wildcards (*.example.com) are specified as SANs.
On which network positions can an attacker intercept HTTP traffic?	Any hop between client and server: the local WiFi (especially public/open), the ISP, backbone routers, VPN exit points, or any proxy/CDN in between. This is why HTTPS is essential for all traffic, not just logins and payments.
What is end-to-end encryption (E2EE) and how does it differ from TLS?	TLS encrypts data between two endpoints (e.g., browser and server) — the server can read the plaintext. E2EE encrypts from sender to final recipient (e.g., Signal messages) — intermediate servers cannot read the content. TLS protects the transport channel; E2EE protects the message itself.
How do you check if a domain is on the HSTS preload list?	Visit hstspreload.org and search for the domain. Browsers ship with the preload list compiled in, so domains on it enforce HTTPS from the very first connection — even before any HTTP header is received. Adding a domain requires meeting strict criteria (valid cert, max-age ≥ 1 year, includeSubDomains).

# === Web Security Fundamentals ===

What is the Same-Origin Policy (SOP)?	A browser security mechanism that prevents scripts on one origin (scheme + host + port) from reading data from a different origin. It isolates potentially malicious documents.
What is Cross-Site Scripting (XSS)?	An attack where malicious JavaScript is injected into a web page and executed in other users' browsers. It can steal cookies, session tokens, or modify page content.
What are the three types of XSS?	Reflected (malicious script in URL/request, reflected back in response), Stored (script saved in database, served to all users), DOM-based (script manipulates the page's DOM directly in the browser).
How do you prevent XSS?	Output encode all user-supplied data before rendering in HTML. Use Content Security Policy (CSP). Validate input. Use frameworks that auto-escape (React, Angular).
What is Cross-Site Request Forgery (CSRF)?	An attack where a malicious site tricks a user's browser into making an authenticated request to another site where the user is logged in, performing unwanted actions.
How is CSRF typically prevented?	Using anti-CSRF tokens (a unique, unpredictable token included in forms and validated server-side). Also: SameSite cookie attribute, checking Origin/Referer headers.
What is SQL Injection?	An attack where malicious SQL is inserted into application queries via user input, allowing an attacker to read, modify, or delete database data, or even execute OS commands.
How do you prevent SQL Injection?	Use parameterized queries (prepared statements) — never concatenate user input into SQL strings. Also: use ORMs, apply input validation, limit database permissions.
What is input validation?	Checking that user-supplied data conforms to expected format, type, length, and range before processing it. First line of defense against injection attacks.
What is output encoding?	Converting special characters (e.g., `<`, `>`, `"`, `'`) into safe representations (HTML entities, URL encoding) before rendering user data, preventing injection.
What is Content Security Policy (CSP)?	An HTTP header that tells the browser which sources of scripts, styles, images, etc. are allowed. Prevents inline script execution, mitigating XSS.
What is HSTS (HTTP Strict Transport Security)?	An HTTP response header that tells browsers to only connect to the site over HTTPS, preventing SSL-stripping downgrade attacks. Format: `Strict-Transport-Security: max-age=31536000; includeSubDomains`.
What does the X-Frame-Options header do?	Prevents the page from being embedded in an `<iframe>`, protecting against clickjacking attacks. Values: `DENY`, `SAMEORIGIN`.
What does the HttpOnly cookie attribute do?	Prevents JavaScript from accessing the cookie via `document.cookie`, mitigating cookie theft through XSS attacks.
What does the Secure cookie attribute do?	Ensures the cookie is only sent over HTTPS connections, preventing transmission over unencrypted HTTP.
What does the SameSite cookie attribute do?	Controls when cookies are sent with cross-site requests. `Strict`: never sent cross-site. `Lax`: sent with top-level navigations. `None`: always sent (requires `Secure`).
What can an attacker do after successfully executing a stored XSS attack?	Steal session cookies (session hijacking), redirect users to phishing sites, log keystrokes in forms, make authenticated requests as the victim, deface the page, deliver drive-by malware, or mine cryptocurrency in the victim's browser — anything executable JavaScript can do.
Why does HTTPS NOT prevent CSRF attacks?	CSRF exploits the browser's automatic inclusion of cookies in cross-origin requests. The malicious request travels over HTTPS (encrypted), but the server still receives a fully authenticated request because the victim's session cookies are attached automatically. HTTPS secures the channel — it cannot distinguish authorized user actions from forged cross-site requests.
What is blind SQL injection?	A variant where the application does not return query results directly. The attacker infers data through boolean differences in the page response (true vs. false conditions) or time delays (`SLEEP()`, `WAITFOR`), extracting data one bit at a time without ever seeing output directly.
Why is client-side input validation insufficient as a security control?	An attacker can bypass it entirely by modifying the HTTP request directly with tools like Burp Suite or curl, or by disabling JavaScript. Client-side validation improves user experience; server-side validation is the actual security control.
Why is `unsafe-inline` in a Content Security Policy dangerous?	It permits any inline `<script>` tag on the page to execute, defeating CSP's primary purpose of blocking injected scripts. If an attacker injects a `<script>` tag via XSS, `unsafe-inline` allows it to run even though CSP is enabled.
Does the Same-Origin Policy block a page from sending cross-origin requests?	No — SOP blocks reading the response, not sending the request. A page can submit forms or make fetch/XHR requests cross-origin; the browser blocks JavaScript from reading the returned data. This is why CSRF works — the request (with cookies) is sent and the side effect occurs, even though the response is blocked.
What is the difference between XSS and HTML injection?	HTML injection inserts HTML elements (text, images, fake forms) that mislead users but don't execute scripts. XSS specifically injects JavaScript that runs in the victim's browser context. XSS is more dangerous because it can steal cookies, exfiltrate data, and perform arbitrary actions.
Which type of XSS is hardest to detect with server-side security tools and why?	DOM-based XSS — because the payload never reaches the server. The malicious data stays in the URL fragment (#) or is processed entirely in client-side JavaScript. Server-side WAFs and SAST tools can't see it; it requires client-side code review or DAST tools that inspect the DOM.
What is context-specific output encoding and why does using the wrong encoding still allow XSS?	HTML encoding (&lt;) for HTML body, URL encoding (%3C) for URL parameters, JavaScript encoding (\x3C) for JS strings. Each context has different syntax and escape rules. Using HTML encoding inside a JavaScript string doesn't neutralize the payload — context-aware encoding is essential.
Why do APIs using token-based auth (Bearer tokens in headers) not need CSRF protection?	CSRF exploits the browser's automatic inclusion of cookies. Bearer tokens in Authorization headers are manually set by JavaScript and never automatically attached by the browser. A cross-origin form submission can't set custom headers, so the request arrives without the token.
Besides reading data, what else can SQL injection achieve?	Writing/modifying/deleting data (INSERT, UPDATE, DELETE, DROP TABLE), executing OS commands (xp_cmdshell on SQL Server), reading system files (LOAD_FILE on MySQL), and performing DNS-based data exfiltration. A successful SQLi can lead to full system compromise.
What is a stored procedure and does it prevent SQL injection?	A stored procedure is precompiled SQL code stored in the database. It prevents SQLi only if it uses parameterized queries internally. If the stored procedure concatenates user input into dynamic SQL (EXEC with string building), it's still fully vulnerable.
What is the difference between input validation and input sanitization?	Validation checks whether input conforms to expected rules (format, type, length, range) and rejects invalid input entirely. Sanitization modifies input to remove or neutralize dangerous parts (stripping tags, encoding characters). Validation should come first; sanitization is a secondary defense layer.
What is a CSP nonce and how does it work?	A nonce is a random, per-request value added to the CSP header (script-src 'nonce-abc123') and matching <script nonce="abc123"> tags. Only scripts with the current request's nonce execute. Since the attacker can't predict the nonce, injected scripts are blocked.
What is the CSP frame-ancestors directive and how does it improve upon X-Frame-Options?	frame-ancestors 'self' https://trusted.com specifies which origins can embed the page in frames. Unlike X-Frame-Options (only DENY or SAMEORIGIN), it supports multiple trusted origins and is part of the more flexible, modern CSP standard.
Can the HttpOnly cookie flag alone prevent all session hijacking?	No — HttpOnly only prevents JavaScript access to the cookie (mitigating XSS-based theft). Sessions can still be hijacked via network interception (without Secure flag/HTTPS), session fixation, CSRF, or physical access to the browser. HttpOnly is one layer in defense-in-depth.
What happens to a Secure cookie when the site is accessed over HTTP?	The browser does not send the cookie with the HTTP request. The server receives no session cookie and treats the request as unauthenticated. This is the intended behavior — preventing cookie transmission over an insecure, eavesdroppable channel.
What is the default SameSite cookie value in modern browsers?	Modern browsers (Chrome 80+, Firefox 86+, Edge) default to SameSite=Lax when no attribute is specified. Cookies are sent on same-site requests and top-level cross-site GET navigations, but not on cross-site POST or sub-resource requests — providing basic CSRF protection by default.
How does an XSS worm work?	A stored XSS payload that automatically propagates: when a user views the infected content, the script executes and posts the same payload to the victim's profile/feed, infecting anyone who views it. The 2005 "Samy" MySpace worm added over 1 million friends in 20 hours.
What is out-of-band SQL injection and when is it used?	When neither direct output nor boolean/time-based inference is possible, the attacker uses database features to make network requests (DNS lookups via UTL_HTTP in Oracle, xp_dirtree in SQL Server) to exfiltrate data to an attacker-controlled server. Used when the application gives no response differences.
What tools do attackers use to bypass client-side controls?	Burp Suite (intercept and modify HTTP requests), browser DevTools (edit JavaScript/HTML/DOM), curl/wget (craft raw requests), Postman (modify API calls), and custom scripts. Any control that only exists in the browser can be completely bypassed.
What is the safest approach to CSP for preventing XSS?	Use strict-dynamic with nonces: script-src 'nonce-{random}' 'strict-dynamic'; object-src 'none'; base-uri 'self'. This allows only explicitly nonced scripts and scripts they dynamically load. No unsafe-inline, no domain allowlists (which can be bypassed via JSONP or open redirects on allowed domains).

# === Memory & Process Security Basics ===

What is a buffer overflow?	Writing data beyond the allocated boundary of a buffer in memory, potentially overwriting adjacent memory and allowing code execution or crashes.
Why are buffer overflows dangerous?	An attacker can overwrite the return address on the stack or function pointers to redirect execution to malicious code, gaining control of the program.
What is the difference between the stack and the heap?	The stack stores local variables and function call data (fixed, LIFO). The heap stores dynamically allocated memory (`malloc`, `new`). Both are in process memory but managed differently.
Why are C and C++ considered memory-unsafe?	They allow direct pointer arithmetic, manual memory management, and unchecked array access. There's no runtime bounds checking, making buffer overflows, use-after-free, and other memory bugs possible.
How do managed languages like Java and C# prevent buffer overflows?	They perform automatic bounds checking on array access, use garbage collection for memory management, and don't allow direct pointer arithmetic.
What is a segmentation fault (segfault)?	A crash that occurs when a program tries to access memory it's not allowed to access — typically dereferencing a null or dangling pointer, or writing to read-only memory.
What is a dangling pointer?	A pointer that references memory that has already been freed. Using it leads to undefined behavior and potential security vulnerabilities (use-after-free).
What is a use-after-free vulnerability?	Accessing memory through a dangling pointer after it has been freed. An attacker can fill that freed memory with malicious data, potentially gaining code execution.
What is a null pointer dereference?	Attempting to read or write memory through a pointer that is NULL. It typically crashes the program; in some contexts it can be exploited.
What is a memory leak and why does it matter for security?	Memory that is allocated but never freed. Over time it can exhaust available memory, causing denial of service (availability impact).
Why should secrets be cleared from memory after use?	If secrets (passwords, keys) remain in memory, they can be extracted via memory dumps, core dumps, swap files, or cold boot attacks.
What is ASLR (Address Space Layout Randomization)?	An OS security feature that randomizes the memory addresses of the stack, heap, and libraries each time a program runs, making it harder for attackers to predict where code and data reside.
What is DEP (Data Execution Prevention) / NX bit?	A hardware/OS feature that marks memory regions (stack, heap) as non-executable. Even if an attacker injects shellcode, the CPU refuses to execute it.
How do garbage collectors help prevent memory bugs?	They automatically reclaim unused memory, preventing use-after-free, double-free, and memory leaks. They also typically enforce bounds checking.
Why is `strcpy` dangerous in C?	It copies a string without checking the destination buffer's size, making buffer overflows trivial. Use `strncpy`, `strlcpy`, or `snprintf` instead.
What is the difference between a stack-based and a heap-based buffer overflow?	Stack-based: overflows a local buffer on the stack, typically overwriting the saved return address to redirect execution. Heap-based: overflows a heap-allocated buffer, corrupting adjacent heap objects or allocator metadata (function pointers, free lists) — harder to exploit but equally dangerous.
What is the primary purpose of Return-Oriented Programming (ROP)?	To execute arbitrary code even when DEP/NX marks the stack and heap as non-executable. ROP reuses small existing code sequences ("gadgets") already in the program, each ending in a `ret` instruction, chaining them via attacker-controlled return addresses on the stack — no new code is injected.
Name two programming languages designed to prevent buffer overflows at the language level.	Rust (compile-time ownership and borrow checker eliminate dangling pointers and out-of-bounds access without runtime overhead) and Go (automatic bounds checking on slices and arrays). Java and C# also prevent them via managed runtime bounds checking.
What is the difference between a buffer overflow and a buffer over-read?	A buffer overflow writes beyond the buffer boundary (can overwrite code, pointers, return addresses → code execution). A buffer over-read reads beyond the buffer boundary (leaks sensitive data from adjacent memory). Heartbleed (CVE-2014-0160) was a buffer over-read.
What is shellcode in the context of buffer overflow exploitation?	Shellcode is a small piece of machine code injected by the attacker into the overflowed buffer. When execution is redirected to the shellcode (via overwritten return address), it typically opens a command shell (hence the name) or downloads further malware.
Why does the stack grow downward and the heap grow upward in typical process memory layouts?	They grow toward each other to maximize use of the virtual address space between them. The stack base is at a high address and grows down with each function call; the heap starts lower and grows up with allocations. If they collide, the process is out of memory.
What is undefined behavior (UB) in C/C++ and why is it a security concern?	UB means the language standard imposes no requirements on what happens — the compiler can produce any result. Common UB (signed integer overflow, null dereference, out-of-bounds access) may seem to "work" during testing but produce exploitable behavior in production, and compilers may optimize away related security checks.
What is the approximate performance cost of bounds checking and why is it worth it?	Array bounds checking adds a comparison instruction per access, typically 1-5% overhead. This negligible cost eliminates buffer overflows, which account for ~70% of all CVEs in memory-unsafe languages like C/C++.
What information does a core dump contain and why is it a security concern?	A core dump is a snapshot of the process's entire memory at the time of a crash, including any secrets (passwords, keys, tokens) that were in memory. Core dumps should be disabled in production (ulimit -c 0) or access-restricted to prevent exposure of sensitive data.
How is a dangling pointer different from a wild (uninitialized) pointer?	A dangling pointer once pointed to valid memory that has since been freed — the memory may be reallocated with attacker-controlled data. A wild pointer was never initialized and holds random garbage. Both cause undefined behavior, but dangling pointers are more exploitable in practice.
Why are use-after-free vulnerabilities particularly common in web browsers?	Browsers have complex DOM trees with many objects referencing each other (elements, events, styles). When one object is freed but another still holds a reference, the dangling reference can access reallocated memory. The complexity of JavaScript-DOM interaction creates many opportunities for reference counting errors.
How do you detect memory leaks in a running application?	Use profiling tools: Valgrind or AddressSanitizer (C/C++), VisualVM or Java Flight Recorder (Java). Monitor for steadily increasing memory consumption without corresponding load increase. In managed languages, look for unintended object references preventing garbage collection.
What is a cold boot attack?	An attacker physically removes RAM from a running or recently-powered-off machine and reads its contents. DRAM retains data for seconds to minutes after power loss (longer if cooled). Encryption keys in memory can be extracted. This is why in-memory secrets should be zeroed immediately after use.
On a 32-bit system, how much entropy does ASLR typically provide and why is it insufficient?	32-bit ASLR typically provides only 8-16 bits of entropy (256-65536 possible positions). An attacker can brute-force the correct address in seconds. 64-bit ASLR provides 28+ bits of entropy, making brute-force computationally infeasible.
How does DEP/NX interact with Just-In-Time (JIT) compilers?	JIT compilers (V8, HotSpot) need to generate and execute code at runtime, requiring writable and executable memory. This conflicts with DEP. Modern JITs use W^X (Write XOR Execute): allocate memory as writable, write generated code, then change permissions to executable-only before execution.
Can garbage-collected languages still have memory-related security issues?	Yes — memory leaks (unintentional references keeping objects alive), resource exhaustion (filling the heap intentionally for DoS), and information disclosure (reading uninitialized buffer contents in some runtimes). They prevent use-after-free and buffer overflows but not all memory-related issues.
Why are even "safe" alternatives to strcpy like strncpy still not fully safe in C?	strncpy does not guarantee null termination — if the source is longer than n, the destination string is not null-terminated, causing potential reads into adjacent memory. Safer alternatives: strlcpy (BSD), snprintf, or higher-level string libraries.
What is a "gadget" in Return-Oriented Programming (ROP)?	A gadget is a short sequence of existing machine instructions ending with a ret instruction, found in the program's code or loaded libraries. The attacker chains gadgets by placing their addresses on the stack — each ret pops the next gadget's address, forming an arbitrary program from existing code fragments.
What is Rust's ownership model and how does it prevent use-after-free at compile time?	Each value has exactly one owner. When the owner goes out of scope, the value is automatically dropped (freed). References must follow lifetime rules — you can't create a reference that outlives its data. The borrow checker enforces these rules at compile time, making dangling references a compilation error.

# === Secure Coding Basics ===

What is the difference between an allowlist and a denylist for input validation?	An allowlist (whitelist) specifies what IS allowed — reject everything else. A denylist (blacklist) specifies what is NOT allowed. Allowlists are more secure because they can't be bypassed with unknown-bad inputs.
Why should you never hardcode secrets in source code?	Source code is often stored in version control, shared with teams, or decompiled. Hardcoded secrets (API keys, passwords) are easily discovered. Use environment variables or secret managers.
Why should error messages not reveal implementation details?	Detailed error messages (stack traces, SQL errors, file paths) help attackers understand the system's internals and find vulnerabilities. Show generic messages to users; log details server-side.
What is the OWASP Top 10?	A regularly updated list of the 10 most critical web application security risks, published by the Open Web Application Security Project. Used as a baseline for secure development.
What is Static Application Security Testing (SAST)?	Analyzing source code (without running it) for security vulnerabilities. Tools scan for patterns like SQL injection, XSS, hardcoded secrets. Examples: SonarQube, Semgrep, Checkmarx.
What should you look for in a security code review?	Input validation, output encoding, authentication/authorization checks, hardcoded secrets, unsafe deserialization, proper error handling, SQL injection, and dependency vulnerabilities.
Why is keeping dependencies up to date important for security?	Third-party libraries frequently have vulnerabilities discovered. Outdated dependencies with known CVEs are a common attack vector. Use tools like Dependabot, Snyk, or OWASP Dependency-Check.
Give an example of how a denylist for input validation can be bypassed.	A denylist blocking the word "script" can be bypassed with `<ScRiPt>` (case variation), `<sc\x72ipt>` (encoding), or `<img onerror=alert(1)>` (different XSS vector entirely). An allowlist that only accepts expected input patterns is far harder to bypass because unknown-bad inputs are rejected by default.
What should you do if a secret (API key, password) is accidentally committed to a git repository?	Immediately revoke and rotate the secret — treat it as fully compromised, regardless of how quickly you act. Remove it from the codebase. Use `git filter-branch` or BFG Repo-Cleaner to purge it from history. Add the file to `.gitignore`. Revocation is the critical step since the history may already have been cloned or indexed.
What is the #1 risk in the OWASP Top 10 (2021)?	Broken Access Control — failures to enforce what authenticated users are allowed to do. Includes IDOR, privilege escalation, missing authorization checks on actions. It rose from #5 in 2017 to #1 in 2021, reflecting how common authorization failures are.
Give a concrete example of when an allowlist is the right choice for input validation.	Accepting a country code: allowlist the ~250 valid ISO 3166-1 codes and reject everything else. A denylist approach (blocking "bad" codes) is impossible because the set of invalid inputs is infinite while valid inputs are finite and well-defined.
What are three safe ways to manage secrets in an application?	(1) Environment variables (simple but limited scope), (2) Secret managers like HashiCorp Vault or AWS Secrets Manager (centralized, audited, auto-rotated), (3) CI/CD secret injection (GitHub Secrets, GitLab CI variables — available only during build/deploy). Never commit secrets to version control.
What should a production error response look like?	Return a generic message ("An error occurred. Reference ID: abc123") with a correlation ID that support can use to look up details in server-side logs. No stack traces, SQL errors, file paths, or technology details should reach the end user.
How should a development team use the OWASP Top 10?	As a baseline for security requirements and testing. Map each risk to specific controls: Broken Access Control → authorization checks on every endpoint, Injection → parameterized queries everywhere. Use it as a checklist during code reviews and security testing.
What are the limitations of SAST (Static Application Security Testing)?	High false positive rate (flags safe code as vulnerable), can't detect runtime issues (misconfigurations, auth bugs), struggles with dynamic languages, and doesn't understand application logic or business context. SAST should be combined with DAST, manual review, and SCA.
What is the difference between a security code review and a regular code review?	A regular review focuses on functionality, code quality, and maintainability. A security review specifically checks for injection vectors, auth/authz gaps, data exposure, insecure dependencies, crypto misuse, and race conditions. It requires a security-aware reviewer (security champion or AppSec engineer).
What is an SBOM (Software Bill of Materials) and why does it matter for security?	An SBOM lists all software components (libraries, frameworks, versions) in an application. When a new CVE is disclosed, you can immediately check if your applications use the vulnerable component. Required by the US Executive Order on Cybersecurity and many compliance frameworks.
Why is simply deleting a file and making a new git commit NOT enough after committing a secret?	Git preserves all history. The secret remains in previous commits and can be retrieved with git log -p, git reflog, or by anyone who cloned/forked the repo. You must use git filter-branch or BFG Repo-Cleaner to rewrite history AND revoke/rotate the secret immediately.
Give three examples of Broken Access Control vulnerabilities.	(1) Horizontal privilege escalation: accessing another user's data by changing an ID parameter (IDOR). (2) Vertical privilege escalation: accessing admin functions as a regular user. (3) Missing function-level access control: admin API endpoints accessible without checking the caller's role.

# === Unix / Linux Security Basics ===

What do the three groups in Unix file permissions represent?	Owner (user who owns the file), Group (users in the file's group), Other (everyone else). Each has independent read (r), write (w), execute (x) permissions.
What does `chmod 755 file` mean?	Owner: read+write+execute (7), Group: read+execute (5), Other: read+execute (5). The file is executable by all but only writable by the owner.
Why is running processes as root dangerous?	Root has unrestricted access to the entire system. If a process running as root is compromised, the attacker gains full control. Always run services with minimal privileges.
What does `sudo` do?	Allows a permitted user to run a command as root (or another user) based on rules in `/etc/sudoers`. Provides auditable, temporary privilege escalation without sharing the root password.
What is `/etc/shadow` and why is it restricted?	Stores hashed passwords for user accounts. It's readable only by root (mode 640) to prevent non-root users from accessing password hashes for offline cracking.
What is the setuid bit and why is it a security concern?	When setuid is set on an executable, it runs with the file owner's privileges (often root) regardless of who executes it. A vulnerability in a setuid-root binary gives root access.
What is the sticky bit on a directory?	When set on a directory (like `/tmp`), only the file owner, directory owner, or root can delete or rename files within it — preventing users from deleting each other's files.
Why is SSH key-based authentication more secure than passwords?	Keys are much longer and harder to brute-force, they can't be phished, and the private key never leaves the client machine. Combined with a passphrase, they provide two-factor protection.
What is a basic firewall and what does it do?	A firewall filters network traffic based on rules (source/destination IP, port, protocol). It blocks unauthorized connections while allowing legitimate traffic.
What is `ufw` on Linux?	Uncomplicated Firewall — a simple frontend for `iptables` that makes it easy to define allow/deny rules. Example: `ufw allow 22/tcp` permits SSH.
Why should you review log files like `/var/log/auth.log`?	They record authentication events (successful logins, failed attempts, sudo usage). Reviewing them helps detect brute-force attacks, unauthorized access, and suspicious activity.
What are world-readable/writable files and why are they dangerous?	Files with permissions allowing any user to read or write. Sensitive data in world-readable files is exposed; world-writable files can be modified by attackers.
What security risks do cron jobs present?	Cron jobs run with the crontab owner's privileges. If a cron job executes a world-writable script, an attacker can modify the script to run arbitrary commands as that user.
Why does package manager security matter?	Installing unsigned or untrusted packages can introduce malware. Always use official repositories, verify package signatures, and keep packages updated for security patches.
Why is setting file permissions to `chmod 777` dangerous?	It gives read, write, and execute permission to all users on the system (owner, group, and everyone else). Any user or process — including a compromised one — can read sensitive data, modify the file, or execute it, potentially enabling privilege escalation.
What should you verify when accepting an SSH host key fingerprint for the first time?	Verify it out-of-band — confirm the fingerprint matches what the server administrator reports through a trusted channel (phone, internal documentation, or a secure portal). Accepting an unknown fingerprint blindly risks connecting to a MitM machine that intercepts your session.
What does the numeric value 4, 2, and 1 represent in Unix file permissions?	Read=4, Write=2, Execute=1. They are summed per group: 7=rwx, 6=rw-, 5=r-x, 4=r--, 0=---. For example, 640 means owner=rw- (6), group=r-- (4), other=--- (0).
What permissions should a sensitive configuration file containing passwords have?	600 (owner read+write only) or 400 (owner read-only). No group or other access. The owner should be the service account that needs the file. Never 644 or 755 for files containing secrets.
How do you create a dedicated service user for a daemon on Linux?	useradd -r -s /usr/sbin/nologin servicename creates a system account with no login shell. Run the service as this user with minimal privileges to only the specific files, ports, and directories it needs.
What is the difference between su and sudo?	su switches to another user entirely (requires that user's password). sudo runs a single command with elevated privileges (requires the invoking user's password). sudo is preferred: it provides auditing (logs who ran what), doesn't share the root password, and can limit privilege to specific commands.
What are PAM and /etc/nsswitch.conf and how do they relate to authentication?	PAM (Pluggable Authentication Modules) provides a framework for auth policies (password complexity, MFA, LDAP). /etc/nsswitch.conf tells the system where to look up users/groups/passwords (files, LDAP, NIS). Together they control how authentication works beyond just /etc/shadow.
How do you find all setuid binaries on a system and why should you audit them?	find / -perm -4000 -type f 2>/dev/null lists all setuid files. Each is a potential privilege escalation vector — a vulnerability in any setuid-root binary gives root access. Audit regularly and remove setuid from any binary that doesn't strictly need it.
What is the SGID (Set Group ID) bit on a directory?	When SGID is set on a directory, new files created inside inherit the directory's group (not the creator's primary group). Useful for shared project directories where all team members need group access to created files.
What is an SSH key passphrase and what happens if your private key is stolen without one?	A passphrase encrypts the private key file on disk. Without a passphrase, anyone who copies the key file (via malware, backup exposure, or physical access) can immediately authenticate as you. With a passphrase, the stolen key is useless without also knowing the passphrase.
What is a default-deny firewall policy and why is it recommended?	A default-deny policy blocks all traffic by default and only allows explicitly permitted connections. New services or ports can't become accidentally exposed. The opposite (default-allow) requires anticipating and blocking every possible attack — an impossible task.
What is iptables and how does ufw relate to it?	iptables is the underlying Linux firewall framework (netfilter). ufw is a user-friendly frontend that generates iptables rules. For complex setups, iptables or nftables (the modern replacement) provides full control over chains, tables, and advanced rules.
What is log rotation and why is it important for security?	Log rotation (logrotate on Linux) archives and compresses old logs, prevents disk exhaustion, and ensures logs are retained for forensic analysis. Without rotation, logs can fill the disk (DoS) or be overwritten. Retain security logs for 90+ days and forward to a central SIEM.
How do you find world-writable files on a Linux system?	find / -perm -o+w -type f 2>/dev/null lists world-writable files. Review each for sensitivity. World-writable files in system directories are particularly dangerous — an attacker can modify scripts, configs, or binaries to escalate privileges.
How do you secure a cron job?	Ensure the script it runs is owned by root (or the intended user) and not world-writable. Use absolute paths in the script and crontab. Specify PATH explicitly. Avoid SUID. Log output. Use cron.allow/cron.deny to restrict who can create crontabs.
What is a supply chain attack via a package manager?	An attacker publishes a malicious package to a public repository (typosquatting, name confusion, or compromising an existing package). When installed, it runs arbitrary code. Mitigate by verifying package signatures, using lockfiles, pinning hashes, and using trusted sources.
What is the known_hosts file and how does it prevent future MitM attacks after first connection?	After you verify and accept an SSH host key, it's stored in ~/.ssh/known_hosts. On subsequent connections, SSH compares the server's key to the stored one. If it doesn't match, SSH warns of a possible MitM attack and refuses to connect. This is "Trust On First Use" (TOFU).

# === Java Security Basics ===

What was the Java Security Manager and why was it removed?	A sandbox mechanism that restricted untrusted code (file access, network, system calls) via policy files. Deprecated in Java 17 and removed in Java 21 due to complexity, poor adoption, and maintenance burden.
Why is `java.util.Random` insecure for security purposes?	`Random` uses a predictable algorithm (LCG). If the seed is known or guessed, all future values can be predicted. Use `SecureRandom` for tokens, keys, and security-sensitive values.
Why is `SecureRandom` preferred for generating security tokens?	It uses a cryptographically strong random number generator (CSPRNG) seeded from OS entropy. Values are unpredictable even if previous outputs are known.
What makes Java serialization (`ObjectInputStream`) dangerous?	Deserializing untrusted data can instantiate arbitrary objects and trigger side effects in their constructors/methods, leading to remote code execution (gadget chain attacks).
How does `PreparedStatement` prevent SQL injection?	It separates SQL code from data. User input is treated as parameter values, not SQL syntax — quotes, semicolons, and SQL keywords in input are safely escaped.
Why should you never concatenate user input into SQL queries?	Input like `'; DROP TABLE users;--` will be interpreted as SQL code, allowing injection attacks. Parameterized queries prevent this by treating input as data only.
What does OWASP Dependency-Check do for Java projects?	Scans project dependencies (JARs) against the NVD (National Vulnerability Database) to identify libraries with known CVEs. Integrates with Maven/Gradle.
Why should Java applications use bcrypt, scrypt, or Argon2 for password storage?	These are slow, salted hashing algorithms designed to resist brute-force and GPU-based cracking. Standard hash functions (MD5, SHA-256) are too fast for password hashing.
What is the risk of leaking stack traces to users?	Stack traces reveal internal class names, file paths, library versions, and database details — information that helps attackers identify vulnerabilities and craft exploits.
Why is logging sensitive data (passwords, tokens, SSNs) dangerous?	Log files are often stored long-term, shared across systems, and accessible to operations teams. Sensitive data in logs can be exposed through log aggregation, breaches, or insider access.
What is the risk of loading untrusted JARs on the classpath?	An untrusted JAR can contain malicious code that executes when classes are loaded. It could override trusted classes, intercept data, or open backdoors.
What replaced the Java Security Manager for restricting untrusted code after its removal?	There is no direct JVM replacement. Instead, use OS-level sandboxing: container isolation (Docker with seccomp/AppArmor), separate JVM processes with minimal permissions, and the Java module system (JPMS) for encapsulation. Application-level security handles access control.
How could an attacker exploit predictable values from java.util.Random?	If session tokens, CSRF tokens, or password reset links are generated with Random, an attacker who observes a few outputs can reconstruct the internal state and predict all future values — enabling session hijacking, CSRF bypass, or account takeover.
What is the performance difference between Random and SecureRandom, and when does it matter?	SecureRandom is slower because it reads from OS entropy sources (/dev/urandom). For high-throughput scenarios (thousands of tokens/second), use SecureRandom.getInstanceStrong() only where essential, and standard SecureRandom (non-blocking) elsewhere. Never fall back to Random for security values.
What is a serialization gadget chain?	A chain of existing Java classes (in libraries like Commons Collections, Spring) where deserializing one object triggers a cascade of method calls: readObject() → method1() → method2() → ... → Runtime.exec(). The attacker doesn't inject new code — they assemble existing classes into a chain that achieves arbitrary execution.
Show an example of vulnerable SQL concatenation vs. safe PreparedStatement.	Vulnerable: "SELECT * FROM users WHERE id = " + userId — input "1 OR 1=1" dumps all users. Safe: PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?"); ps.setInt(1, userId); — input is treated as data, never SQL syntax.
Does using an ORM like Hibernate completely prevent SQL injection?	No. ORMs prevent injection for standard queries, but if you use native SQL, HQL/JPQL with string concatenation, or Criteria API improperly, injection is still possible. Always use parameterized queries even within ORM frameworks.
What is the process for responding when a dependency scanner finds a critical CVE?	(1) Verify the CVE affects your usage (not all code paths trigger it). (2) Update to a patched version if available. (3) If no patch, evaluate workarounds or alternative libraries. (4) Document the risk if accepting it temporarily. (5) Monitor for exploitation attempts.
What happens if you hash passwords with SHA-256 instead of bcrypt?	An attacker with a stolen hash database can try billions of guesses per second on a GPU. With bcrypt (cost 12), they're limited to thousands per second. For an 8-character password, SHA-256 cracking takes minutes; bcrypt cracking takes centuries.
How should Java web applications handle exceptions in production?	Use global exception handlers (@ControllerAdvice in Spring) that catch all exceptions, log the full stack trace server-side with a correlation ID, and return a generic error response (HTTP status + correlation ID only). Never expose internal details to clients.
How do you prevent sensitive data from accidentally appearing in logs?	Use structured logging with explicit field selection (don't log entire objects). Implement log-scrubbing libraries that mask patterns (SSN, credit card, tokens). Mark sensitive fields with annotations. Review log output during code reviews.
How does the Java module system (JPMS) help contain untrusted code?	The module system enforces strong encapsulation: a module must explicitly export packages, and others must explicitly require it. Internal implementation packages are inaccessible by default. This limits what a loaded library can see and access, reducing the blast radius of a compromised dependency.
How should you handle HTTPS certificate validation in Java?	Use the default `HttpsURLConnection` or HttpClient which validates certificates against the JVM's truststore. Never disable certificate checking (e.g., trust-all `TrustManager`) in production.
What are safer alternatives to Java's built-in serialization for data exchange?	JSON (Jackson, Gson), Protocol Buffers (protobuf), XML with external entities disabled, MessagePack, or Avro. These data-format libraries don't instantiate arbitrary Java objects and avoid the gadget-chain RCE vulnerability inherent in `ObjectInputStream` deserializing untrusted data.
What is the purpose of the "work factor" in bcrypt?	The work factor (cost parameter) controls how computationally expensive bcrypt is — each increment doubles the computation time. Higher values make each password guess slower, directly resisting brute-force attacks. Tune it so hashing takes ~100–300ms on current hardware, and increase it as hardware gets faster.

# === Android Security Basics ===

What is the Android permission model?	Apps must declare required permissions in `AndroidManifest.xml`. Since Android 6.0+, dangerous permissions (camera, location, contacts) require explicit runtime user approval.
What is the principle of least privilege on Android?	Request only the permissions your app actually needs. Don't request camera access for a calculator app. Over-requesting permissions increases the attack surface.
How does Android sandbox each app?	Each app runs as its own Linux user with a unique UID. Its data directory (`/data/data/<package>`) is accessible only to that UID. Apps can't read each other's data by default.
Why should you avoid storing secrets on external storage on Android?	External storage (`/sdcard`) is world-readable. Any app with `READ_EXTERNAL_STORAGE` can access files there. Use internal storage or `EncryptedSharedPreferences` for sensitive data.
What is `EncryptedSharedPreferences`?	Part of Android Jetpack Security — it encrypts both keys and values in SharedPreferences using AES-256. Protects data at rest from extraction on rooted devices.
Why should Android apps set `android:usesCleartextTraffic="false"`?	It prevents the app from making unencrypted HTTP connections, ensuring all network traffic goes over HTTPS. This protects against network sniffing and MitM attacks.
What is Network Security Config on Android?	An XML configuration that lets apps customize HTTPS behavior: trust custom CAs, pin certificates, restrict cleartext traffic — all declaratively without code changes.
What are WebView security risks?	WebViews can execute JavaScript. If loading untrusted content, an attacker can inject scripts to access the app's data, call Java methods via JavaScript bridges, or access the file system.
What is the difference between explicit and implicit intents?	Explicit intents target a specific component (safe). Implicit intents let any app handle the request — a malicious app could register for the same intent filter and intercept data.
Why should you set `android:exported="false"` on components not meant for external access?	If a component (Activity, Service, Receiver, Provider) is exported, other apps can invoke it. This could allow unauthorized actions or data access.
What does APK signing guarantee?	It proves the APK was produced by the holder of the signing key and hasn't been tampered with. Android refuses to install or update an APK if the signature doesn't match.
Why is `Log.d()` with sensitive data dangerous in production?	Logcat output on pre-Android 4.1 devices was world-readable. Even on newer devices, any app with shell access or connected via ADB can read logs. Sensitive data in logs can be exfiltrated.
What is Google Play Protect?	Google's malware detection service that scans apps on-device and in the Play Store. It checks for known malware, potentially harmful behavior, and policy violations.
What is root detection and why do apps implement it?	Rooted devices have removed OS security restrictions. Apps (especially banking/security apps) check for root to detect environments where their data could be compromised or tampered with.
What is the difference between "normal" and "dangerous" permissions on Android?	Normal permissions are granted automatically at install time (low privacy/security risk, e.g., INTERNET, VIBRATE). Dangerous permissions affect user privacy or security (e.g., CAMERA, ACCESS_FINE_LOCATION, READ_CONTACTS) and require explicit user approval at runtime (Android 6.0+).
Why can't two apps on Android normally access each other's internal storage?	Each app runs as a unique Linux user with its own UID. The app's internal storage directory (`/data/data/<package>`) is owned by that UID and has permissions set to 700. The kernel denies access to any other UID — including other apps — regardless of their permissions.

# === Automotive Security Basics ===

What is a CAN bus and why is it used in vehicles?	Controller Area Network — a broadcast protocol that allows ECUs (Electronic Control Units) to communicate without a central computer. It's reliable, real-time, and cost-effective.
Why is CAN bus inherently insecure?	CAN has no authentication, no encryption, and no source addressing — any device on the bus can send any message, and all devices see all messages. It was designed for reliability, not security.
What is an ECU (Electronic Control Unit)?	A small embedded computer that controls a specific vehicle function: engine, brakes, steering, airbags, etc. A modern car has 70–100+ ECUs communicating over vehicle networks.
What is the OBD-II port and what security risk does it pose?	On-Board Diagnostics port (mandatory in vehicles) provides direct access to the CAN bus. Malicious devices plugged into it can send commands to any ECU.
Why do over-the-air (OTA) updates for vehicles need signing?	Without signature verification, an attacker could push malicious firmware to millions of vehicles simultaneously, potentially controlling braking, steering, or engine functions.
What attack surface does vehicle connectivity add?	Bluetooth, Wi-Fi, cellular modems, USB ports, and V2X communication all create remote entry points. Each wireless interface is a potential vector for remote exploitation.
What is a relay attack on keyless entry?	Attackers use two relay devices to extend the key fob's signal range. One device near the fob, one near the car — the car thinks the fob is nearby and unlocks/starts.
What is ISO 26262?	A functional safety standard for automotive electrical/electronic systems. It defines safety integrity levels (ASIL A–D) and processes to ensure systems don't cause harm due to malfunction.
What is ISO/SAE 21434?	The automotive cybersecurity engineering standard. It defines requirements for cybersecurity management, threat analysis, risk assessment, and security measures throughout the vehicle lifecycle.
What is V2X (Vehicle-to-Everything) communication?	Wireless communication between vehicles (V2V), infrastructure (V2I), pedestrians (V2P), and cloud (V2C). Enables safety features like collision warnings but adds a wireless attack surface.
Why should infotainment systems be isolated from safety-critical ECUs?	If an attacker compromises the infotainment system (which has internet access), isolation prevents them from reaching the brakes, steering, or engine controllers.
What risks do aftermarket OBD-II devices pose?	Third-party OBD dongles with Bluetooth/Wi-Fi can be exploited remotely to send malicious CAN messages. They often have weak or no authentication.
Why is it difficult to retrofit authentication onto the classic CAN bus?	Standard CAN frames have only 8 bytes of payload, leaving almost no room for a MAC or signature alongside actual data. Adding authentication requires protocol changes, firmware updates across potentially 100+ ECUs from multiple suppliers, and re-validation of safety-critical systems — a massive, expensive undertaking.
How can relay attacks on keyless entry be prevented?	Ultra-Wideband (UWB) ranging (measures precise distance, hard to relay), motion sensors in key fobs (disable the signal when the fob is stationary), Faraday cage signal-blocking pouches, and requiring a PIN or button press confirmation before engine start.

# === Embedded Security Basics ===

Why is physical security important for embedded devices?	Embedded devices are often deployed in accessible locations. An attacker with physical access can extract firmware, dump memory, connect to debug ports, or tamper with the device.
What is secure boot at a high level?	A process where the device verifies the cryptographic signature of each software component during startup, ensuring only authorized, unmodified code runs.
Why must firmware updates be signed?	Without signing, an attacker could install modified firmware containing backdoors or malware. Signature verification ensures the update came from the legitimate manufacturer.
Why are default credentials dangerous on embedded devices?	Many devices ship with well-known default passwords (admin/admin). Attackers scan for and exploit these at scale. The Mirai botnet compromised millions of IoT devices this way.
What are UART and JTAG, and why is their exposure a security risk?	UART (serial debug) and JTAG (hardware debug) are debug interfaces on circuit boards. If exposed, attackers can gain a shell, read/write memory, and extract firmware.
Why is network segmentation important for IoT/embedded devices?	IoT devices often run outdated software with known vulnerabilities. Placing them on a separate network segment prevents a compromised device from accessing critical systems.
What is the difference between Secure Boot and Measured Boot?	Secure Boot verifies each boot component's cryptographic signature and halts the boot process if verification fails — it enforces integrity by blocking unauthorized code. Measured Boot records (measures) each component's hash into a TPM's PCR registers without blocking — it enables remote attestation so an external verifier can later confirm what ran at boot.
How can manufacturers reduce the security risk of exposed UART and JTAG debug ports in production devices?	Disable debug interfaces in production firmware via fuse bits or configuration registers (one-time programmable). Require authentication before accepting debug commands. Physically remove or fill the header pins on the PCB so they cannot be easily connected to. Document and audit all debug interface access.
