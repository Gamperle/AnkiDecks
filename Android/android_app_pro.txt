# ==========================================
# 1.13 Performance & Optimization
# ==========================================

What is Perfetto and how does it improve over Systrace?	Perfetto is a system-wide tracing tool that captures CPU scheduling, GPU rendering, Binder calls, and custom app trace points. It offers SQL-based analysis (Trace Processor), longer traces, and a modern web-based UI — replacing Systrace.
What is the difference between cold, warm, and hot app start?	Cold start: process not running, full initialization. Warm start: process exists but Activity is recreated (onCreate). Hot start: Activity exists in memory, just brought to foreground (onResume). Cold is slowest.
How do Baseline Profiles improve app performance?	They provide a list of critical code paths that are AOT-compiled at install time (via cloud profiles or embedded in AAB), reducing JIT compilation at runtime and improving startup and scroll performance by up to 30-40%.
How do you generate a Baseline Profile?	Using the Macrobenchmark library: write a BaselineProfileGenerator instrumented test that exercises critical user journeys, then run it to produce a baseline-prof.txt included in the APK/AAB.
What is R8 full mode and how does it differ from compatibility mode?	R8 full mode applies more aggressive optimizations: assumes classes are not reflectively accessed (unless kept), removes more dead code, and can inline across module boundaries. Compatibility mode preserves ProGuard semantics.
How does LeakCanary detect memory leaks?	LeakCanary watches destroyed Activities and Fragments using WeakReferences and a ReferenceQueue. If an object isn't GC'd within a threshold, it dumps the heap and analyzes the reference chain to find the leak.
What is inBitmap reuse?	A BitmapFactory.Options flag that tells the decoder to reuse an existing Bitmap's memory allocation for a new decode operation, reducing garbage collection pressure from repeated Bitmap loading
What is the RenderThread in Android?	A separate thread (introduced in Android 5.0) dedicated to executing GPU rendering commands. It offloads drawing operations from the UI thread, enabling smoother animations and reducing jank.
How do you detect GPU overdraw?	Enable "Debug GPU Overdraw" in Developer Options. Areas drawn once appear in their true color; increasing overdraw levels are shown in blue (1x), green (2x), pink (3x), red (4x+).
What is StrictMode?	A development tool that detects accidental disk/network operations on the main thread (ThreadPolicy) and leaked objects like unclosed Cursors or Activities (VmPolicy). Violations can log, flash, or crash.
What causes an ANR (Application Not Responding)?	The main thread is blocked for more than 5 seconds (input/touch event) or a BroadcastReceiver doesn't finish within 10 seconds. In Android 14+, background ANRs are also reported.
How do you analyze ANR traces?	Examine /data/anr/traces.txt (via adb bugreport) — it contains thread dumps of all threads at the time of ANR. Look for the main thread's stack trace to find the blocking operation.
What is a hardware layer and when should you use one?	A hardware layer (View.LAYER_TYPE_HARDWARE) renders a View subtree into an off-screen GPU texture. Useful during animations — the texture can be transformed (translate, alpha, scale) without re-rendering, but costs GPU memory.
What is the difference between the RenderThread and the UI thread?	The UI thread (main thread) builds and updates the View hierarchy, measures/layouts views, and records drawing commands into DisplayLists. The RenderThread (introduced in Android 5.0) takes those DisplayLists and executes the actual GPU rendering commands (OpenGL/Vulkan). This separation means the UI thread can start preparing the next frame while the RenderThread is still rendering the current one, reducing jank.
What is a frame deadline and how does it relate to refresh rate?	A frame deadline is the time budget within which a frame must be rendered to avoid dropped frames. On a 60 Hz display the deadline is ~16.6 ms per frame; on 90 Hz it is ~11.1 ms; on 120 Hz it is ~8.3 ms. If the UI thread + RenderThread work exceeds this budget, the frame is dropped and the user perceives jank.
What is systrace/atrace and how do you use TRACE_TAG markers in app code?	Systrace (now largely replaced by Perfetto) captures kernel and framework trace events via atrace. In app code you add custom trace sections using `android.os.Trace.beginSection("label")` and `Trace.endSection()`, or the Jetpack `androidx.tracing.Trace` wrapper. These markers appear as labeled spans in the Perfetto/systrace timeline, helping you identify exactly which code paths are slow.
What is ClassVerifier overhead and how does it affect app startup?	At APK install or first class load, the Android runtime verifies DEX bytecode to ensure type safety and correctness. Classes referencing APIs not present on the current OS version trigger slower verification paths (soft-fail). This overhead increases startup time. Baseline Profiles and R8 optimizations help by pre-verifying or eliminating problematic code paths.
What is startup tracing with androidx.tracing?	The `androidx.tracing` library provides `Trace.beginSection()`/`endSection()` APIs that work on all API levels and integrate with Perfetto. For startup tracing specifically, you use `androidx.tracing.perfetto` to capture method-level traces during cold start without recompiling. The Macrobenchmark library also automatically captures startup traces for analysis.
What are the common causes of GC pauses and how do you minimize them?	Common causes: excessive object allocation in hot loops, large short-lived objects (especially Bitmaps), autoboxing primitives, string concatenation in tight loops, and creating iterators implicitly (e.g., `for (item in map)` allocating an Iterator). Minimize by reusing objects (object pools, `inBitmap`), using primitive arrays instead of boxed types, avoiding allocations in `onDraw()`/`onBindViewHolder()`, and profiling with Android Studio's Memory Profiler to find allocation hotspots.

# 1.14 Advanced Kotlin & Coroutines
# ==========================================

How does the Kotlin compiler transform a suspend function?	It uses CPS (Continuation Passing Style) — the compiler adds a Continuation parameter and transforms the function body into a state machine where each suspension point is a state transition.
What is a Continuation in Kotlin coroutines?	An interface with resumeWith(Result<T>) that represents the "rest of the computation" after a suspension point. The compiler generates Continuation implementations as state machines.
What is CoroutineContext and what does it contain?	A map-like collection of Element instances: Job (lifecycle), CoroutineDispatcher (thread), CoroutineExceptionHandler (error handling), CoroutineName (debugging). Elements are combined with the + operator.
What is the difference between Dispatchers.Main and Dispatchers.Main.immediate?	Dispatchers.Main always dispatches to the main thread queue. Dispatchers.Main.immediate executes immediately if already on the main thread, avoiding an unnecessary dispatch and reducing latency.
What is structured concurrency?	A design principle where coroutines form a parent-child hierarchy. If a parent is cancelled, all children are cancelled. If a child fails, the parent (and siblings) are cancelled — ensuring no orphaned coroutines.
What is a SupervisorJob and when should you use it?	A Job where child failure does not cancel the parent or siblings. Use it when children are independent tasks (e.g., viewModelScope uses SupervisorJob so one failing coroutine doesn't cancel others).
How does CoroutineExceptionHandler work?	It's an optional CoroutineContext element that catches uncaught exceptions in coroutines launched with launch (not async). It only works at the root scope level — not for child coroutines.
What is the difference between a Channel and a Flow?	Channels are hot — they produce values regardless of receivers, supporting multiple producers and single consumer. Flows are cold — they don't produce values until collected, and each collector gets its own stream.
What are fan-in and fan-out patterns with Channels?	Fan-out: one Channel, multiple coroutines consuming from it (load balancing). Fan-in: multiple coroutines sending to one Channel (aggregation).
What is the actor coroutine pattern?	A pattern using a coroutine with a Channel inbox to process messages sequentially, providing safe mutable state access without locks. Implemented via actor { } builder (now marked as obsolete API).
What is backpressure in Flows and how do you handle it?	When a collector is slower than the emitter, backpressure occurs. Handle with buffer() (concurrent emission/collection), conflate() (drop intermediate), or collectLatest() (cancel slow collection on new emission).
What is the difference between callbackFlow and channelFlow?	callbackFlow bridges callback-based APIs to Flow (e.g., listeners, Firebase), must use awaitClose for cleanup. channelFlow is for concurrent flow emissions from multiple coroutines.
What is KSP (Kotlin Symbol Processing)?	A Kotlin compiler plugin API for annotation processing that is faster than KAPT because it works directly with Kotlin symbols instead of generating Java stubs first. Room, Moshi, and Hilt support KSP.
Why is KSP preferred over KAPT?	KSP is up to 2x faster (no Java stub generation), understands Kotlin-specific features (nullability, extensions, inline classes), and reduces build times significantly in multi-module projects
What does the Compose compiler plugin do?	It transforms @Composable functions into code that manages the slot table (Compose's internal data structure), enables smart recomposition, tracks state reads, and generates code for skipping unchanged composables.
What is the difference between supervisorScope and coroutineScope?	`coroutineScope` cancels all children if any child fails (bidirectional cancellation). `supervisorScope` uses a SupervisorJob internally, so one child's failure does not cancel siblings or the scope — only the failed child is affected. Use `supervisorScope` when children are independent (e.g., loading multiple UI sections where one failure shouldn't block others).
What is NonCancellable and when should you use it?	`NonCancellable` is a special Job that cannot be cancelled. Use it inside `withContext(NonCancellable) { ... }` within a `finally` block to ensure cleanup code (e.g., saving data, closing resources, sending analytics) runs to completion even when the parent coroutine has been cancelled.
What is a Mutex in Kotlin coroutines?	`kotlinx.coroutines.sync.Mutex` is a coroutine-aware mutual exclusion primitive. Unlike `synchronized` blocks (which block the thread), `Mutex.withLock { }` suspends the coroutine while waiting, freeing the thread for other work. Use it to protect shared mutable state accessed from multiple coroutines.
What is the difference between StateFlow and ConflatedBroadcastChannel?	`ConflatedBroadcastChannel` was an early coroutines API for sharing state across multiple subscribers; it is now deprecated. `StateFlow` replaces it with a cleaner API: it always has a current value (`value` property), is a Flow (supports operators), never completes, and conflates emissions so collectors always see the latest state.
What is a SharedFlow replay cache?	The `replay` parameter of `MutableSharedFlow(replay = N)` configures how many previously emitted values are buffered and replayed to new subscribers. `replay = 0` means new collectors only see future emissions; `replay = 1` makes it behave like `StateFlow` (new collectors get the latest value). The cache can be reset with `resetReplayCache()`.
What is the difference between flowOn and withContext for switching dispatchers in Flow?	`flowOn` changes the upstream dispatcher — emissions above `flowOn` in the chain run on the specified dispatcher while collection stays on the caller's dispatcher. `withContext` inside `flow { }` is forbidden and throws an exception because it would break the flow's sequential emission contract. Use `flowOn` for upstream context switching and `withContext` only outside of flow builders.

# 1.15 Advanced Security
# ==========================================

What is the Android Keystore and what protection does it offer?	A system facility that stores cryptographic keys inside a hardware-backed container (TEE or StrongBox) so they cannot be extracted. Keys never leave the secure hardware during crypto operations.
What is the difference between TEE and StrongBox for key storage?	TEE (Trusted Execution Environment) runs on the main processor in a secure world (ARM TrustZone). StrongBox is a dedicated secure element chip (separate hardware), offering stronger physical attack resistance.
What is key attestation in Android?	A mechanism where the Keystore generates a certificate chain that cryptographically proves a key was generated in secure hardware on a specific device, verified by a remote server against Google's root certificate.
What is BiometricPrompt?	A Jetpack API for authenticating users with biometrics (fingerprint, face, iris) that provides a consistent system UI across devices and supports Class 2 (weak) and Class 3 (strong) biometric authenticators.
What is the difference between Class 2 and Class 3 biometrics?	Class 3 (Strong) biometrics have < 7% SAR/IAR and can unlock CryptoObjects for cryptographic operations. Class 2 (Weak) biometrics have looser requirements and cannot be used with CryptoObjects.
What is a CryptoObject in BiometricPrompt?	A wrapper around cryptographic operations (Cipher, Signature, Mac) that requires biometric authentication to unlock the key. Ensures the crypto operation is only performed when the user is biometrically verified.
What is the Network Security Config?	An XML configuration (network_security_config.xml) that customizes TLS settings per-domain: certificate pinning, cleartext traffic, custom CA certificates, and debug overrides — without modifying app code
How does certificate pinning work in Network Security Config?	You specify pin-set entries with SHA-256 hashes of certificates in the trust chain. The app only trusts connections where the server certificate chain matches at least one pin. Include a backup pin for rotation.
What is the Play Integrity API (replacement for SafetyNet)?	A Google API that provides device attestation verdicts: device integrity (genuine, rooted, or emulator), app integrity (real or tampered), and account licensing — used to verify the app runs in a trustworthy environment
What are the OWASP Mobile Top 10 risks?	M1: Improper Credential Usage, M2: Inadequate Supply Chain Security, M3: Insecure Authentication/Authorization, M4: Insufficient Input/Output Validation, M5: Insecure Communication, M6: Inadequate Privacy Controls, M7: Insufficient Binary Protections, M8: Security Misconfiguration, M9: Insecure Data Storage, M10: Insufficient Cryptography
How can an attacker bypass SSL pinning on Android?	Using tools like Frida or Objection to hook the TrustManager or OkHttp's CertificatePinner at runtime, replacing them with permissive implementations that accept any certificate
What tools are commonly used to reverse-engineer Android APKs?	jadx (DEX → Java decompiler), apktool (resource + smali extraction), dex2jar, Ghidra (native code analysis), and Frida (dynamic instrumentation of running apps)
How effective is R8 obfuscation against determined attackers?	R8 obfuscation (renaming) makes reverse engineering harder but is not a strong protection. Determined attackers can deobfuscate using control flow analysis. Combine with string encryption, native code, and server-side logic for defense in depth.
What is root detection and why is it unreliable?	Root detection checks for indicators of a rooted device: presence of `su` binary, Superuser/Magisk apps, writable system partitions, or test-keys in the build fingerprint. It is unreliable because Magisk can hide root from detection (MagiskHide/Zygisk DenyList), Frida can hook detection methods at runtime, and custom ROMs can spoof build properties. It should be one layer in a defense-in-depth strategy, not a sole security gate.
What is runtime application self-protection (RASP)?	RASP is a set of techniques that allow an app to detect and respond to attacks at runtime. This includes detecting debuggers attached to the process (`Debug.isDebuggerConnected()`), Frida/Xposed hooking frameworks, code injection, memory tampering, and emulator environments. When a threat is detected the app can degrade functionality, wipe sensitive data, or terminate.
What is code tampering detection?	Code tampering detection verifies that the APK has not been modified after signing. Techniques include checking the APK signature at runtime (`PackageManager.getPackageInfo().signatures`), computing checksums of the DEX files or native libraries, and verifying the installer package is the Play Store. Attackers repackage APKs with modified code, so tamper detection helps catch this — though it can itself be bypassed by patching the detection code.
What is the difference between symmetric and asymmetric encryption in Android Keystore?	Symmetric keys (AES, HMAC) use the same key for encryption and decryption — fast and suitable for encrypting local data. Asymmetric keys (RSA, EC) use a public/private key pair — the public key encrypts or verifies, the private key decrypts or signs. In Android Keystore, both types can be hardware-backed and non-extractable. Use symmetric (AES-GCM) for bulk data encryption and asymmetric (ECDSA) for digital signatures and key exchange.
What is the SafetyNet/Play Integrity attestation flow?	The app requests a nonce from your server, passes it to the Play Integrity API, which returns a signed attestation token. The app sends this token to your server, which calls Google's server to decrypt and verify it. The response contains verdicts on device integrity (genuine/rooted/emulator), app integrity (unmodified from Play Store), and licensing status. All verification must happen server-side — never trust client-side checks alone.

# 1.16 Gradle Plugin Development & Build Internals
# ==========================================

Where can you define a custom Gradle plugin?	In buildSrc/ (available to all modules, rebuilt every time), in an included build (composite build, cached), or as a standalone Gradle plugin project published as a dependency
What is a Gradle Task and what is @TaskAction?	A Task is a unit of work in a Gradle build (e.g., compile, copy). @TaskAction annotates the method that contains the task's execution logic.
What are incremental tasks in Gradle?	Tasks that track which input files changed and only process those files, avoiding reprocessing unchanged inputs. Requires declaring @InputFiles, @OutputDirectory, and implementing InputChanges.
What are up-to-date checks in Gradle?	Gradle compares task input and output fingerprints between builds. If nothing changed, the task is marked UP-TO-DATE and skipped, saving build time.
What replaced the Transform API in AGP 7+?	AsmClassVisitorFactory — a more efficient bytecode transformation API where you register ASM ClassVisitor implementations that AGP applies during the dexing pipeline
What is the AGP Variant API?	An API that lets plugins customize per-variant (debug/release/flavor) build configuration: modify sources, add generated sources, manipulate artifacts (APK, manifest, resources), and register custom tasks.
What is the AGP Artifacts API?	An API for accessing and transforming build artifacts (manifests, classes, resources, APKs) in a type-safe way. Plugins can use get(), transform(), and append() operations on artifact types.
What is UAST in Android Lint?	Universal Abstract Syntax Tree — a language-agnostic AST representation used by Lint to analyze both Java and Kotlin source code with the same visitor, simplifying custom lint check development
What is PSI in the context of custom Lint rules?	Program Structure Interface — IntelliJ's lower-level syntax tree. Lint uses PSI under UAST for language-specific analysis. UAST is preferred for most checks; PSI for language-specific edge cases.
What is the role of IssueRegistry in custom Lint?	IssueRegistry is the entry point for a custom Lint library. It lists all Issues (rules) provided by the library and is registered via META-INF/services or Gradle.
Why migrate from KAPT to KSP for annotation processing?	KAPT generates Java stubs for every Kotlin file before running annotation processors — slow and memory-intensive. KSP operates directly on Kotlin symbols, is up to 2x faster, and preserves Kotlin information.
What is a Gradle extension and how do you create one?	A Gradle extension is a custom configuration block that a plugin exposes in `build.gradle` (e.g., `android { }`, `myPlugin { }`). You create one by defining a class with configurable properties and registering it via `project.extensions.create("name", MyExtension::class.java)`. In Kotlin DSL, use `abstract class` with `@Inject` constructor and `Property<T>`/`ListProperty<T>` for lazy, Gradle-managed values.
What is the difference between a binary plugin and a script plugin in Gradle?	A script plugin is a `.gradle` or `.gradle.kts` file applied with `apply from: "script.gradle"` — simple but not reusable or testable. A binary plugin is a compiled class implementing `Plugin<Project>`, distributed as a JAR, applied with `plugins { id("...") }` — it supports versioning, testing, caching, and sharing across projects.
What is a buildSrc convention plugin?	A convention plugin defined in the `buildSrc/` directory (or an included build) that applies shared build logic across modules — e.g., common `compileSdk`, Kotlin options, dependency versions. It is a regular `Plugin<Project>` class that configures conventions, applied in modules via `plugins { id("my-convention") }`. This reduces duplication across multi-module `build.gradle.kts` files.
What are the three Gradle build phases: initialization, configuration, and execution?	Initialization: Gradle evaluates `settings.gradle(.kts)` to determine which projects are included in the build. Configuration: all `build.gradle(.kts)` scripts are evaluated, the task dependency graph (DAG) is built, and extensions/tasks are configured — no task actions run yet. Execution: Gradle runs the selected tasks in dependency order, executing only those that are not up-to-date. Expensive work should only happen in the execution phase (inside task actions), not during configuration.
What is artifact transformation in Gradle?	Artifact transformation lets Gradle convert dependencies from one format to another automatically (e.g., JAR → desugared JAR, AAR → classes). You register a `TransformAction` with `dependencies.registerTransform(MyTransform::class) { from.attribute(...); to.attribute(...) }`. Gradle caches transforms and applies them lazily when a consumer requests an artifact with specific attributes that differ from the published format.
