# 1.7 Jetpack Compose
# ==========================================

What is Jetpack Compose?	Android's modern, fully declarative UI toolkit where UI is built using Kotlin functions annotated with @Composable, replacing XML layouts
What is a @Composable function?	A Kotlin function annotated with @Composable that describes a piece of UI. The Compose compiler transforms it into code that builds and updates the UI tree.
What is recomposition in Jetpack Compose?	The process of calling @Composable functions again when their inputs (state) change, so the UI reflects the latest data. Only composables that read changed state are re-executed.
What is smart recomposition?	Compose's ability to skip recomposing composables whose inputs have not changed, improving performance by only updating what's necessary
What is state hoisting in Compose?	A pattern where state is moved up to a caller composable and passed down as parameters, making the child composable stateless and reusable. The child receives the state value and event callbacks.
What does remember do in Compose?	remember stores a value across recompositions within the same composition. Without it, the value would be reset every time the composable recomposes.
What is the difference between remember and rememberSaveable?	remember survives recompositions but not configuration changes or process death. rememberSaveable also survives configuration changes and process death (saves to Bundle).
What does mutableStateOf do in Compose?	Creates a MutableState<T> object that Compose observes — when the value changes, any composables reading it will automatically recompose
How does Modifier ordering affect composable behavior?	Modifiers are applied in order from first to last. For example, padding before background adds padding outside the background, while background before padding adds padding inside the background area.
How do you create a custom Modifier in Compose?	By defining an extension function on Modifier that chains composed {}, drawBehind {}, layout {}, or other Modifier factory functions
What is the difference between LazyColumn and Column?	Column lays out all children at once regardless of visibility. LazyColumn only composes and lays out visible items (like RecyclerView), making it efficient for long scrollable lists.
What is LazyVerticalGrid?	A Compose component that displays items in a vertically scrollable grid with configurable column count (GridCells.Fixed or GridCells.Adaptive)
What is Scaffold in Material 3 Compose?	A top-level layout composable that provides slots for TopAppBar, BottomAppBar, FloatingActionButton, Drawer, and SnackbarHost, handling their coordination
What is LaunchedEffect in Compose?	A side-effect composable that launches a coroutine when it enters composition and cancels it when it leaves. Takes key parameters — the coroutine restarts when keys change.
What is DisposableEffect in Compose?	A side-effect composable for effects that require cleanup. Provides an onDispose block that runs when the composable leaves composition or when keys change.
What is SideEffect in Compose?	A side-effect composable that runs after every successful recomposition. Used to publish Compose state to non-Compose code (no cleanup, no suspend).
What is rememberCoroutineScope in Compose?	Returns a CoroutineScope bound to the composable's lifecycle, useful for launching coroutines from event handlers (e.g., button clicks) where LaunchedEffect is not appropriate
What does derivedStateOf do?	Creates a State object that only updates when the result of its calculation changes, even if the inputs change frequently. Avoids unnecessary recompositions for derived values.
What does snapshotFlow do?	Converts Compose State reads into a Kotlin Flow, emitting values whenever the read state changes. Useful for bridging Compose state to Flow-based APIs.
What is produceState?	A composable that launches a coroutine scoped to the composition and returns a State object. Useful for converting non-Compose observable data (Flow, LiveData, callbacks) into Compose State.
What is CompositionLocal in Compose?	A mechanism to pass data implicitly down the composition tree without explicitly passing it through every composable parameter. Created with compositionLocalOf or staticCompositionLocalOf.
When should you use staticCompositionLocalOf vs compositionLocalOf?	Use staticCompositionLocalOf for values that rarely change (expensive recomposition of entire subtree on change). Use compositionLocalOf for values that change frequently (only invalidates readers).
How does navigation work in Jetpack Compose?	Using Navigation Compose: NavHost defines the navigation graph, NavController manages navigation, and composable() defines destinations with routes (string-based or type-safe)
What are type-safe routes in Navigation Compose?	Using Kotlin serializable classes/objects as route definitions instead of string-based routes, providing compile-time safety for arguments and destination names (Navigation 2.8+)
What is AnimatedVisibility in Compose?	A composable that animates the appearance and disappearance of its content with configurable enter/exit transitions (fadeIn, slideIn, expandIn, etc.)
What is the difference between animateAsState and updateTransition?	animateAsState animates a single value between states. updateTransition can coordinate multiple animations simultaneously when transitioning between states.
What is AndroidView in Compose?	A composable that allows embedding a traditional Android View inside a Compose layout, used for interop with legacy View-based components (e.g., MapView, AdView)
What is ComposeView?	An Android View that hosts Compose content, enabling you to embed Compose UI inside existing View-based layouts (Fragment XML, RecyclerView items)
What makes a type "stable" in Compose and why does it matter?	A stable type has immutable public properties or notifies Compose of changes via MutableState. Compose can skip recomposition of composables when stable parameters haven't changed. Annotate with @Stable or @Immutable.
What is the Compose semantics system?	A tree of semantic properties attached to composables that describes their meaning for accessibility services and UI testing. Used by TalkBack and ComposeTestRule.
What is a Slot API pattern in Compose?	A pattern where a composable accepts other composable functions as parameters (lambda content slots) instead of fixed data. For example, Button(onClick = {}) { Text("Click") } — the content lambda is a slot. This allows callers to provide arbitrary UI for specific areas, making components highly flexible and reusable.
What is the difference between Column, Row, and Box in Compose?	Column arranges children vertically (top to bottom). Row arranges children horizontally (start to end). Box stacks children on top of each other (back to front, like FrameLayout). All three accept Arrangement and Alignment parameters to control spacing and positioning of their children.
What is intrinsic measurement in Compose?	A mechanism that lets a composable query the preferred (intrinsic) width or height of its children before the actual measurement pass. Accessed via Modifier.height(IntrinsicSize.Min) or IntrinsicSize.Max. Useful when you need to size one child based on another — e.g., making a divider match the tallest sibling in a Row.
What does movableContentOf do in Compose?	movableContentOf wraps a composable lambda so that when it moves to a new location in the composition tree, its state is preserved and transferred rather than destroyed and recreated. Useful for animations or layout transitions where the same content shifts position without losing remembered state.
What is a SubcomposeLayout and when is it used?	A layout that defers composition of some children until the measurement phase, when the size of earlier children is known. Used when child composition depends on measured sizes — e.g., LazyColumn uses it to compose only visible items based on available viewport height.
What is the Compose phase system (composition, layout, draw)?	Compose processes frames in three phases: (1) Composition — runs @Composable functions to build/update the UI tree, (2) Layout — measures and places each node (Measure → Place), (3) Draw — renders the UI to the canvas. Skipping earlier phases when possible (e.g., only re-drawing for color changes via lambda-based modifiers) improves performance.
What is a CompositionLocalProvider?	A composable that supplies values to CompositionLocal keys for its subtree. Usage: CompositionLocalProvider(LocalContentColor provides Color.Red) { /* children read LocalContentColor.current as Red */ }. It scopes the provided values so they only affect composables within its content block.

# 1.8 Advanced Jetpack Libraries
# ==========================================

What is SavedStateHandle in ViewModel?	A key-value map that survives process death, allowing ViewModels to save and restore state. It is automatically saved to a Bundle and restored when the process is recreated.
Why is SavedStateHandle important beyond onSaveInstanceState?	ViewModel normally only survives configuration changes. SavedStateHandle extends survival to process death, ensuring critical state (navigation args, user input) is preserved.
How do you create a ViewModel with constructor parameters (besides Hilt)?	By creating a ViewModelProvider.Factory that instantiates the ViewModel with the required parameters, or using the viewModelFactory DSL (Lifecycle 2.5+)
What is repeatOnLifecycle and when should you use it?	A suspend function that restarts a block of code when the lifecycle reaches a specified state (e.g., STARTED) and cancels it when it drops below. Used to safely collect Flows in Activities/Fragments.
What is the difference between LifecycleOwner and LifecycleObserver?	LifecycleOwner (Activity, Fragment) has a Lifecycle. LifecycleObserver observes lifecycle events from a LifecycleOwner to perform actions at specific lifecycle states.
What is a PagingSource in Paging 3?	A class that defines how to load pages of data from a single source (network or database). Implements the load() function which returns LoadResult.Page or LoadResult.Error.
What is a RemoteMediator in Paging 3?	A class that handles loading data from a remote source and storing it in a local database (Room). It coordinates between the network and the local PagingSource for offline-first pagination.
What are LoadStates in Paging 3?	Objects representing the loading state (Loading, NotLoading, Error) for three positions: refresh (initial load), prepend (loading before), and append (loading after)
What are the three main use cases in CameraX?	Preview (viewfinder), ImageCapture (taking photos), and ImageAnalysis (real-time frame processing, e.g., barcode scanning, ML inference)
How is CameraX lifecycle-aware?	CameraX binds to a LifecycleOwner — it automatically opens the camera on STARTED and releases it on STOPPED, preventing resource leaks
What is Media3 (ExoPlayer)?	The Jetpack media playback library (successor to standalone ExoPlayer) providing MediaSession, Player API, streaming support (HLS, DASH), DRM, and notification integration
What is the App Startup library?	A Jetpack library that provides a way to initialize components at app startup efficiently, replacing multiple ContentProviders with a single mechanism and supporting lazy initialization
What is the difference between Macrobenchmark and Microbenchmark?	Microbenchmark measures small code units (function execution time) in-process. Macrobenchmark measures app-level metrics (startup time, scroll jank, animations) from a separate test APK.
What is a Baseline Profile?	A list of methods/classes that should be AOT-compiled at install time, improving app startup and runtime performance by avoiding JIT compilation for critical code paths
What is the SplashScreen API?	A Jetpack / platform API (Android 12+) that standardizes the app splash screen with an icon animation, background color, and exit animation, replacing custom splash screen Activities
What is collectAsStateWithLifecycle and why prefer it over collectAsState?	collectAsStateWithLifecycle (from lifecycle-runtime-compose) collects a Flow into Compose State while respecting the lifecycle — it stops collecting when the lifecycle drops below a given state (default STARTED). collectAsState keeps collecting even when the app is in the background, wasting resources and potentially updating UI that isn't visible.
What is the Navigation safe args equivalent in Compose Navigation?	In Compose Navigation 2.8+, type-safe routes using @Serializable Kotlin classes replace the old string-based routes and safe args plugin. You define a data class for each destination with typed parameters, and the framework handles serialization/deserialization automatically, providing compile-time argument safety.
What is a Pager in Compose (HorizontalPager, VerticalPager)?	HorizontalPager and VerticalPager (from foundation library) provide swipeable page-by-page layouts similar to ViewPager. They use a PagerState to track the current page and support snapping, fling behavior, and page transformations via Modifier.graphicsLayer.
What is WindowSizeClass and adaptive layouts?	WindowSizeClass (from material3-window-size-class) categorizes the window into Compact, Medium, or Expanded width/height buckets. Apps use these classes to choose layouts — e.g., bottom nav on Compact, NavigationRail on Medium, permanent NavigationDrawer on Expanded — enabling adaptive UI for phones, tablets, and foldables.
What is the difference between PeriodicWorkRequest and OneTimeWorkRequest?	OneTimeWorkRequest runs a task once and completes. PeriodicWorkRequest repeats at a specified interval (minimum 15 minutes). Periodic work cannot be chained, always re-enqueues after each run, and uses a flex window to allow the system to batch execution for battery efficiency.
What are WorkManager constraints (network, battery, charging)?	Constraints specify conditions that must be met before work runs: setRequiredNetworkType (CONNECTED, UNMETERED), setRequiresBatteryNotLow(true), setRequiresCharging(true), setRequiresDeviceIdle(true), setRequiresStorageNotLow(true). Work is deferred until all constraints are satisfied.

# 1.9 Architecture Patterns
# ==========================================

Describe the MVVM pattern in Android.	Model holds data, View (Activity/Fragment/Compose) observes and displays, ViewModel holds UI state and business logic, exposes data via StateFlow/LiveData. View observes ViewModel; ViewModel doesn't reference View.
What is MVI (Model-View-Intent) and how does it differ from MVVM?	MVI uses unidirectional data flow: View sends Intents (user actions) → ViewModel processes them and updates a single immutable ViewState → View renders the state. This makes state management more predictable than MVVM.
What are the three layers in Clean Architecture for Android?	Presentation (UI + ViewModel), Domain (use cases, business logic, pure Kotlin — no Android dependencies), Data (repositories, data sources, API/DB implementations)
What is a Use Case (Interactor) in Clean Architecture?	A class in the Domain layer that encapsulates a single business operation (e.g., GetUserProfileUseCase). It depends only on repository interfaces, keeping business logic independent of frameworks.
What is the Repository pattern?	A pattern where the Repository acts as a single source of truth, abstracting data sources (network, database, cache) behind a clean API. The ViewModel interacts only with the Repository.
What does "offline-first" mean in the Repository pattern?	The local database is the single source of truth. Data is loaded from the database and displayed immediately, while the Repository fetches fresh data from the network in the background to update the DB.
What annotations set up Hilt in an Android app?	@HiltAndroidApp on Application, @AndroidEntryPoint on Activity/Fragment, @Inject on constructor for injectable classes, @Module + @InstallIn for providing dependencies
What is the difference between @Provides and @Binds in Hilt?	@Provides creates instances in a @Module method (for third-party classes you can't annotate). @Binds maps an interface to its implementation (more efficient, abstract method).
What do Hilt scopes control?	The lifetime of a provided dependency. @Singleton lives with the app, @ViewModelScoped lives with the ViewModel, @ActivityScoped lives with the Activity, etc.
What is @Qualifier in Hilt?	An annotation used to distinguish between multiple bindings of the same type. For example, two different OkHttpClient instances can be qualified with @Named("auth") and @Named("default").
What is @EntryPoint in Hilt?	An interface annotated with @EntryPoint that allows accessing Hilt-provided dependencies from classes that Hilt doesn't directly inject (e.g., ContentProvider, custom non-standard entry points)
What is Kotlin Multiplatform (KMP) and how does it relate to Android architecture?	KMP allows sharing Kotlin code across Android, iOS, and other platforms. Architecture-wise, the shared module typically contains the Domain and Data layers, while platform-specific code handles UI.
What is modularization in Android and why use it?	Splitting an app into multiple Gradle modules (feature, core, library) to improve build times (parallel compilation, cached modules), enforce boundaries, enable code reuse, and improve team scalability
What are the typical module types in a modularized Android app?	app (entry point), feature modules (UI + ViewModel), core modules (shared utilities, networking, database), domain module (use cases), and data modules (repositories, data sources)
What is the difference between ViewModel and AndroidViewModel?	ViewModel has no access to the Application context. AndroidViewModel extends ViewModel and receives the Application as a constructor parameter, useful when you need a context (e.g., for accessing resources or system services). Prefer plain ViewModel when possible to avoid tight coupling to the Android framework and improve testability.
What is the single source of truth principle?	A design principle where each piece of data is owned and modified by exactly one authoritative source (e.g., a local database). All other layers read from and write through this source. This avoids inconsistencies from multiple copies of the same data and simplifies state management.
What is unidirectional data flow (UDF) in Android?	A pattern where state flows down from ViewModel to UI, and events flow up from UI to ViewModel. The ViewModel processes events, updates state, and the UI observes the new state. This one-way cycle makes state changes predictable, easier to debug, and prevents the UI from modifying state directly.
What is the circuit breaker pattern for network calls?	A resilience pattern where the app monitors network call failures and, after a threshold is reached, temporarily "opens" the circuit — failing fast without making actual requests. After a cooldown period, it allows a trial request; if successful, it "closes" the circuit again. This prevents overwhelming a failing server and improves user experience.
What is feature flagging and why use it?	Feature flagging uses boolean or configuration flags (often remotely controlled) to enable or disable features at runtime without deploying new code. Benefits include gradual rollouts, A/B testing, instant kill switches for broken features, and decoupling deployment from release. Common tools include Firebase Remote Config and LaunchDarkly.

# 1.10 Testing
# ==========================================

What is the difference between JUnit 4 and JUnit 5 on Android?	JUnit 5 has better assertion APIs, nested tests, parameterized tests, and lifecycle callbacks (@BeforeEach). Requires the android-junit5 plugin. JUnit 4 is still the default in many projects.
What is MockK and why is it popular for Android Kotlin testing?	MockK is a mocking framework built for Kotlin, supporting coroutines, extension functions, top-level functions, and DSL-style mock definitions (every { }, verify { })
What is the difference between Mockito and MockK?	Mockito is Java-first and has issues with Kotlin final classes (requires mockito-kotlin). MockK is Kotlin-native, supports suspend functions, extension functions, and has a cleaner Kotlin DSL.
What is Robolectric?	A framework that runs Android unit tests on the JVM by providing shadow implementations of Android SDK classes, enabling fast testing without a device or emulator
What are the three main components of an Espresso test?	ViewMatcher (find the view: onView(withId(...))), ViewAction (perform action: .perform(click())), ViewAssertion (verify result: .check(matches(isDisplayed())))
What is an IdlingResource in Espresso?	A mechanism that tells Espresso to wait for an async operation (network call, animation) to complete before proceeding with assertions, ensuring test synchronization
What is UI Automator and how does it differ from Espresso?	UI Automator can interact with any app on the device (cross-app testing, system UI). Espresso only tests within your own app but provides deeper view hierarchy access and auto-synchronization.
How do you test Compose UI?	Use createComposeRule() or createAndroidComposeRule(), set content, then find nodes with onNodeWithText/onNodeWithTag and assert with assertIsDisplayed, performClick, etc.
What is Turbine?	A testing library for Kotlin Flows that provides a simple API to collect and assert emissions, errors, and completion using awaitItem(), awaitError(), awaitComplete()
What is @HiltAndroidTest?	An annotation for instrumented or Robolectric tests that sets up Hilt dependency injection. Combined with @UninstallModules to replace production modules with test modules.
What is the difference between a fake and a mock?	A fake is a working implementation with shortcuts (e.g., in-memory database). A mock is a configured object that records interactions for verification. Fakes test behavior; mocks test interactions.
What is TestDispatcher in coroutine testing?	A CoroutineDispatcher for tests that gives you control over virtual time. StandardTestDispatcher queues coroutines (you advance manually). UnconfinedTestDispatcher executes eagerly.
What does runTest do?	A coroutine test builder that runs a test in a TestScope with a TestDispatcher, automatically advancing time and failing on uncaught exceptions
What is a test double?	A generic term for any object used in place of a real dependency during testing. The five types are: dummy (passed but unused), stub (returns canned answers), spy (records calls for verification), mock (pre-programmed with expectations), and fake (working simplified implementation, e.g., in-memory database).
What is the AAA (Arrange-Act-Assert) pattern?	A test structuring pattern: Arrange — set up test data, dependencies, and preconditions; Act — execute the code under test; Assert — verify the outcome matches expectations. Also known as Given-When-Then in BDD. It keeps tests readable, consistent, and focused on a single behavior.
What is screenshot testing in Android?	A testing approach where UI screenshots are captured and compared against reference (golden) images to detect unintended visual changes. Tools include Paparazzi (JVM-based, no device needed), Roborazzi, and the Compose Preview Screenshot Testing library. Useful for catching layout regressions.
What is the difference between instrumented and local unit tests?	Local unit tests run on the JVM (src/test/) — they are fast but have no real Android framework (use Robolectric for shadows). Instrumented tests run on a device or emulator (src/androidTest/) — they have access to the full Android framework, Context, and UI, but are slower to execute.
What is a test rule in JUnit?	A test rule intercepts test execution to add behavior before and after each test. In JUnit 4, rules implement TestRule (e.g., InstantTaskExecutorRule for LiveData, GrantPermissionRule, CoroutineTestRule). In JUnit 5, the equivalent is extensions (@ExtendWith). Rules help reduce boilerplate setup across tests.

# 1.11 Build System & Tooling
# ==========================================

What is the difference between build.gradle and build.gradle.kts?	build.gradle uses Groovy DSL. build.gradle.kts uses Kotlin DSL, providing type-safe configurations, IDE autocompletion, and refactoring support.
What are build variants in Android?	Combinations of buildTypes (debug, release) and productFlavors that generate different versions of your app (e.g., freeDebug, paidRelease)
What are productFlavors used for?	To create different versions of an app from the same codebase — e.g., free vs paid, mock vs production, or different branding/client versions
What is the difference between APK and AAB (Android App Bundle)?	APK is a self-contained installable package. AAB is a publishing format where Google Play generates optimized APKs per device configuration (screen density, ABI, language), reducing download size.
What does R8 do in Android builds?	R8 is the code shrinker, optimizer, and obfuscator. It removes unused code (tree-shaking), optimizes bytecode, and renames classes/methods to short names, reducing APK size and making reverse engineering harder.
What is the difference between R8 and ProGuard?	R8 is Google's replacement for ProGuard, built into AGP. It's faster, produces smaller output, and supports desugaring. It uses ProGuard-compatible rules but has its own optimizations.
What does @Keep do in R8/ProGuard?	Prevents the annotated class, method, or field from being removed or renamed during shrinking/obfuscation
What are consumer ProGuard rules?	ProGuard/R8 rules bundled inside a library (AAR) via consumerProguardFiles. They are automatically included when the app consuming the library is minified, ensuring the library works correctly.
What is a Gradle Version Catalog (libs.versions.toml)?	A centralized file in gradle/ that declares all dependency versions, libraries, bundles, and plugins in one place, enabling consistent version management across modules
What is the Gradle configuration cache?	A feature that caches the result of the configuration phase, so subsequent builds skip re-evaluating build scripts when nothing has changed, significantly improving build times
What is Android Lint?	A static analysis tool that checks Android project source files for potential bugs, performance issues, security vulnerabilities, accessibility problems, and best practice violations
How do you create a custom Lint rule?	By extending Detector, implementing a Scanner interface (e.g., UastScanner), registering issues in an IssueRegistry, and packaging it as a lint check module
What is the difference between implementation and api dependency configurations?	implementation keeps the dependency internal — downstream modules cannot see it, which reduces recompilation. api exposes the dependency transitively — downstream modules can use its classes directly. Use implementation by default; use api only when your module's public API surface leaks types from the dependency.
What is a build cache in Gradle?	A cache that stores task outputs keyed by their inputs. When a task's inputs haven't changed (or match a previous build, even on another machine with a remote cache), the output is reused instead of re-executing the task. Enabled with org.gradle.caching=true. Dramatically speeds up incremental and CI builds.
What is a Gradle composite build?	A build that includes other independent Gradle builds via includeBuild() in settings.gradle. Unlike multi-module projects (which share a single build), composite builds let you develop and test libraries alongside your app with automatic dependency substitution — changes in the included build are immediately reflected without publishing.
What is AGP (Android Gradle Plugin) and what does it do?	AGP (com.android.application / com.android.library) extends Gradle with Android-specific build capabilities: compiling Kotlin/Java to DEX, processing resources and manifests, generating APK/AAB, running R8/ProGuard, managing build variants and flavors, and integrating with the Android SDK and NDK.
What is the difference between compile SDK, target SDK, and min SDK?	minSdk is the lowest Android version your app supports — devices below this cannot install it. compileSdk is the API version used to compile your code (determines which APIs are available at compile time). targetSdk declares which API level your app has been tested against — the system may apply compatibility behaviors for apps targeting older versions.

# 1.12 Background Processing & IPC
# ==========================================

What are the three types of Android Services?	Foreground service (visible notification, user-aware work), Background service (no UI, restricted since Android 8), Bound service (client-server interface via onBind)
What is the foreground service type requirement (Android 14+)?	Apps targeting Android 14+ must declare a foregroundServiceType in the manifest (e.g., camera, location, mediaPlayback, dataSync) and hold the corresponding permission
When should you use WorkManager vs a foreground service?	WorkManager for deferrable, guaranteed background work (even across restarts). Foreground service for ongoing user-initiated tasks that need immediate execution with a visible notification.
What is the difference between JobScheduler and WorkManager?	JobScheduler is a platform API with limited features. WorkManager wraps JobScheduler (API 23+), adding chaining, constraints, retries, and backward compatibility. Prefer WorkManager.
What is the difference between exact and inexact alarms in AlarmManager?	Exact alarms fire at the precise time (requires SCHEDULE_EXACT_ALARM permission on Android 12+). Inexact alarms may be batched by the system for battery optimization.
What is a BroadcastReceiver and how is it registered?	A component that receives system or app broadcast intents. Register statically in manifest (limited since Android 8) or dynamically via Context.registerReceiver() for runtime events.
What is the difference between LocalBroadcastManager and system broadcasts?	LocalBroadcastManager sends broadcasts only within the app process (faster, secure, no IPC). System broadcasts cross process boundaries. Note: LocalBroadcastManager is deprecated — use LiveData, Flow, or EventBus instead.
What is AIDL (Android Interface Definition Language)?	A language for defining IPC interfaces between processes. The AIDL compiler generates Stub (server) and Proxy (client) classes based on the interface definition.
How does Binder IPC work at a high level?	The client calls a method on the Proxy, which serializes arguments into a Parcel, sends it to the kernel Binder driver, which delivers it to the server's Stub.onTransact() for deserialization and execution.
What is Parcelable and how does it differ from Serializable?	Parcelable is Android's high-performance serialization for IPC via Binder. It requires explicit marshalling code but is much faster than Java Serializable, which uses reflection.
What is the Messenger pattern for IPC?	A simplified IPC mechanism using a Handler on the server side and a Messenger that clients can send Messages to. Simpler than AIDL but limited to one-way, serialized message delivery.
What is WorkManager chaining?	WorkManager allows sequencing work requests using beginWith().then().then().enqueue(). Chained work runs sequentially — each step starts only after the previous one succeeds. You can also combine parallel chains using WorkContinuation.combine(). Output data from one worker is passed as input to the next via Data objects.
What is the foreground service notification requirement?	Starting with Android 8 (API 26), foreground services must display a persistent notification via startForeground(id, notification) within 5 seconds of calling startForegroundService(). On Android 12+, the notification cannot be dismissed by the user while the service runs. On Android 13+, the POST_NOTIFICATIONS permission is also required.
What is the difference between bound and started services?	A started service is launched via startService()/startForegroundService() and runs until it calls stopSelf() or is stopped externally. A bound service is connected to via bindService() and runs only while clients are bound. A service can be both started and bound — it stops only when it is both stopped and all clients unbind.
What is Context.RECEIVER_NOT_EXPORTED (Android 13+)?	A flag passed to registerReceiver() on Android 13+ (API 33) that prevents other apps from sending broadcasts to your receiver. If your receiver only listens for app-internal or system broadcasts, use RECEIVER_NOT_EXPORTED for security. Use RECEIVER_EXPORTED only when other apps need to send you broadcasts. This flag is required on API 33+.
What is process death and how does it differ from Activity destruction?	Activity destruction (e.g., from configuration change or navigating away) is handled by the normal lifecycle — ViewModel survives it. Process death occurs when the system kills the entire app process (low memory, user swipe-away). All in-memory state including ViewModels is lost. Only data saved via onSaveInstanceState/SavedStateHandle, persistent storage (Room, DataStore), or WorkManager survives process death.
