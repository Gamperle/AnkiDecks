# ==========================================
# 2.12 AOSP Customization & Porting
# ==========================================

What are the key files needed to port AOSP to a new device?	BoardConfig.mk (hardware config: partition sizes, arch, kernel), device.mk (packages, overlays), AndroidProducts.mk (lunch targets), vendorsetup.sh (lunch registration), plus kernel sources and HAL implementations
What is BoardConfig.mk?	A makefile that defines board-level hardware configuration — target architecture, partition sizes, bootloader type, kernel paths, SELinux policy paths, and hardware feature flags
What does inherit-product do in device makefiles?	Inherits (includes) product configuration from a parent makefile. For example, $(call inherit-product, $(SRC_TARGET_DIR)/product/full_base.mk) pulls in the standard AOSP base product config.
How do you add a new AIDL system service to AOSP?	Define the AIDL interface, implement the Stub in a Java/Kotlin service class, register it with ServiceManager in SystemServer, create a Manager wrapper, add SELinux policies for the new service context.
How do you register a custom system service with ServiceManager?	In SystemServer, call ServiceManager.addService("my_service", new MyServiceImpl()), then define a service_contexts entry and allow rules in SELinux policy
How do you implement a custom HAL?	Define the HAL interface in AIDL (or HIDL), implement the server in C++, create an init.rc service entry, add VINTF manifest entries, write SELinux policies for the HAL domain, and build with Android.bp.
What is a native daemon in AOSP?	A C/C++ background process that runs as a system service (e.g., surfaceflinger, audioserver, netd). Defined in init.rc with service entries, started at boot.
How do you integrate a native daemon with init?	Write an init .rc file with a service entry specifying the binary path, user/group, SELinux domain (seclabel), class, restart policy, and any socket/file permissions needed.
What is RRO (Runtime Resource Overlay)?	A mechanism to replace resources (strings, drawables, layouts) of a target app/framework at runtime using overlay APKs, without modifying the target's source code
What is the difference between static and runtime resource overlays?	Static overlays are baked into the system image and always active. Runtime overlays (RRO) can be enabled/disabled dynamically using OverlayManagerService.
How do you customize the System UI in AOSP?	Modify packages/apps/SystemUI — customize StatusBar, NavigationBar, LockScreen, QuickSettings, and Launcher. SystemUI uses a plugin architecture for modular customization.
What is the SystemUI plugin architecture?	An extensibility mechanism where SystemUI loads plugins dynamically at runtime via ExtensionController. Plugins can override or extend UI elements without modifying SystemUI source.
How does the A/B OTA update system work?	The device has two slots (A and B) for system/vendor partitions. OTA applies updates to the inactive slot while the device runs normally. On reboot, the bootloader switches to the updated slot. Rollback is possible by switching back.
What is virtual A/B?	A storage-efficient variant of A/B updates that uses snapshots (dm-snapshot) instead of full duplicate partitions. It writes delta changes to userdata and merges them after reboot.
What is update_engine?	The native daemon that handles OTA payload application — downloads/reads the payload, verifies signatures, applies file/block differences to the inactive slot, and verifies the result
What is the difference between DEVICE_MANIFEST_FILE and DEVICE_FRAMEWORK_COMPATIBILITY_MATRIX?	DEVICE_MANIFEST_FILE declares the HAL interfaces the device (vendor) provides (e.g., camera@2.0, audio@7.0). DEVICE_FRAMEWORK_COMPATIBILITY_MATRIX declares what the framework requires from the device. VINTF checks at boot that every requirement in the compatibility matrix is satisfied by a manifest entry, ensuring system/vendor compatibility.
What are static libraries vs shared libraries in AOSP builds?	Static libraries (.a) are linked entirely into the binary at build time — every binary gets its own copy of the code. Shared libraries (.so) are loaded at runtime and shared across processes, saving disk/memory. In Android.bp: `cc_library_static` vs `cc_library_shared`. Prefer shared for large, widely-used libraries (libc, libutils) and static for small, single-use helpers or when you need hermetic builds.
What is a sepolicy vendor extension?	A set of SELinux policy files placed in `device/<vendor>/<device>/sepolicy/` that extend the platform's base policy with device-specific rules. Vendor extensions define new types, domains, and allow rules for vendor HALs, daemons, and device nodes. They are compiled separately and merged with the platform policy at build time, respecting the Treble separation between system and vendor policy.
What is the difference between system_ext and product partitions?	`system_ext` holds OEM extensions tightly coupled to the framework (custom system services, modified framework classes) that may depend on internal framework APIs. `product` holds product-specific apps, overlays, and configs that use only stable/public APIs. system_ext is updated with the system partition; product can potentially be updated independently. This separation allows finer-grained Treble compatibility.
How do you add a prebuilt APK to the system image?	Create an Android.bp (or Android.mk) with an `android_app_import` (Soong) or `BUILD_PREBUILT` (Make) entry specifying the APK path, certificate (presigned or platform), and optional privileges. Add the module name to `PRODUCT_PACKAGES` in the device makefile. For privileged apps, place a privapp-permissions XML in `/etc/permissions/` granting the required permissions.

# ==========================================
# 2.13 Build System Deep Dive
# ==========================================

How does Soong work internally?	Soong reads Android.bp files (Blueprint format), resolves dependencies, generates Ninja build files, and then Ninja executes the actual build. Soong replaces the Make-based build system.
What are namespaces in Soong?	A scoping mechanism that prevents module name collisions across different parts of the source tree. Each namespace has its own set of module names.
What is a genrule in Soong?	A module type that runs a custom command to generate source files or other outputs from inputs. Used for code generation, preprocessing, or custom build steps.
What is the difference between cc_library, cc_library_shared, and cc_library_static?	cc_library builds both shared and static versions. cc_library_shared builds only a .so. cc_library_static builds only a .a. Choose based on whether the library should be linked statically or dynamically.
What is PRODUCT_PACKAGES?	A make variable that lists packages (APKs, native binaries, libraries) to include in the built system image. Adding a module here causes it to be built and installed.
What is PRODUCT_COPY_FILES?	A make variable that copies files from the source tree to specific locations in the build output. Format: source_path:destination_path. Used for config files, firmware, etc.
What is the mixed build approach with Bazel + Soong?	Google is migrating AOSP's build from Soong to Bazel. During the transition, mixed builds allow some modules to be built by Bazel while others use Soong, with inter-build-system dependencies.
What is VNDK (Vendor Native Development Kit)?	A set of system libraries that the vendor partition is allowed to use, defined to maintain a stable interface between system and vendor. VNDK libraries are versioned and snapshotted.
What is the difference between VNDK, VNDK-SP, and LLNDK?	VNDK: vendor-available system libraries. VNDK-SP (Same-Process): VNDK libraries loaded into vendor processes (same process as framework, e.g., OpenGL). LLNDK: low-level NDK (liblog, libc, libm) available to everyone.
What is a vendor snapshot?	A prebuilt snapshot of VNDK libraries at a specific Android version, allowing the vendor partition to be built against a fixed interface without needing the full framework source
What is a phony package in Soong/Make?	A phony package is a build target with no actual output artifact — it exists only to group dependencies. In Make, declared with `.PHONY`. In Soong, `phony { name: "my-group", required: ["mod-a", "mod-b"] }`. Used to create convenient build aliases (e.g., `make my-group` builds all listed modules). Common examples: `droidcore`, `apps_only`.
What is the difference between java_library and android_library in Soong?	`java_library` produces a plain JAR with no Android resource processing — used for pure Java/Kotlin utility libraries. `android_library` produces an AAR-equivalent that runs aapt2 to compile Android resources (res/, AndroidManifest.xml) and generates R.java. Use `android_library` when the module has resources or manifests; use `java_library` for resource-free code.
What is filegroup in Soong?	A module type that names a set of source files so they can be referenced by other modules via `srcs: [":my_filegroup"]`. It produces no build output itself. Useful for sharing source files across modules, organizing AIDL/proto files, or splitting large srcs lists into logical groups without duplicating paths.
What is the soong_config_module_type mechanism?	A way to create configurable module types whose properties vary based on vendor-defined variables. You declare a `soong_config_module_type` referencing a `soong_config_string_variable` or `soong_config_bool_variable`, then set the variable in a `soong.variables` JSON or via `SOONG_CONFIG_*` environment variables. This enables conditional compilation (e.g., enabling a feature flag for a specific product) without modifying the module definition.
What is the difference between defaults and inheritance in Android.bp?	`defaults` (via `cc_defaults`, `java_defaults`, etc.) is a Soong-specific composition mechanism — modules reference defaults with `defaults: ["my-defaults"]` and properties are merged (not overridden). A module can use multiple defaults, and defaults can chain. There is no class-style inheritance in Blueprint. Unlike Make's `inherit-product`, defaults only merge properties within a single module definition, providing reusable property bundles without hierarchy.

# 2.14 APEX & Mainline Modules
# ==========================================

What is APEX (Android Pony EXpress)?	A container format for installing and updating modular system components (like native libraries, framework JARs, and HALs) independently via the Play Store or OTA, without a full system update
Name five Mainline modules delivered via APEX.	ART (runtime), Conscrypt (TLS), Media codecs, DNS resolver, NetworkStack, tethering, Wi-Fi, Bluetooth, NNAPI (Neural Networks), CellBroadcast, DocumentsUI
What is inside an APEX file?	An apex_manifest.pb (metadata), a payload disk image (ext4 or f2fs) containing the files, an fs_metadata file, and a v4 APK signing signature for verification
How does APEX activation work?	apexd (APEX daemon) verifies the APEX signature, mounts the payload image via loop device, and bind-mounts the contents to the expected filesystem paths (e.g., /apex/com.android.art/)
What is the difference between flattened and compressed APEX?	Flattened APEX: files extracted directly to filesystem (no loop mount, used on devices without loop device support). Compressed APEX: payload is compressed (zlib/deflate), decompressed at activation time.
What is apexd?	The APEX daemon that manages APEX lifecycle — scans for APEXes, verifies signatures, activates (mounts) them at boot, handles updates and rollbacks
How does APEX enforce module boundaries?	Each APEX module exposes stable APIs via APEX stubs — header-only libraries that define the public interface. Internal symbols are hidden, preventing vendor code from depending on unstable internals.
What is the difference between updatable and non-updatable APEX?	Updatable APEXes can be updated via the Play Store or OTA independently of a full system update — they require signing with a unique key and must maintain stable APIs. Non-updatable APEXes (`updatable: false` in Android.bp) are only updated with the system image. Non-updatable APEXes can use unstable APIs and have fewer restrictions but lose the ability to be patched independently.
What is apex_available and how does it restrict library usage?	`apex_available` is a property on library modules (cc_library, java_library, etc.) that lists which APEX modules are allowed to include that library. If a library specifies `apex_available: ["com.android.art"]`, only the ART APEX can use it. The special value `"//apex_available:platform"` allows use from the platform (non-APEX). This enforces module boundaries and prevents accidental cross-APEX dependencies.
What is the bootstrap APEX?	The bootstrap APEX is the version of an APEX that is pre-installed on the system partition and used during early boot before apexd is fully running. It provides the initial copies of critical modules (e.g., ART, tzdata). Once apexd starts, it may replace the bootstrap APEX with an updated version from /data/apex/active/ if one is available.
How does APEX handle native library versioning?	APEX uses stub libraries to define stable native API surfaces. The APEX build system generates versioned stubs (e.g., libfoo.stubs with version 1, 2, 3) that external consumers link against. The actual implementation inside the APEX can change freely as long as stub symbols remain compatible. Symbol versioning and `stubs.versions` in Android.bp control which symbols are exported at which version.
What is the pre-installed vs updated APEX flow?	At build time, APEXes are placed in `/system/apex/` (pre-installed). On first boot, apexd activates these pre-installed copies. When an update arrives (via Play Store or OTA), the new APEX is stored in `/data/apex/active/`. On subsequent boots, apexd compares versions and activates the higher-versioned APEX. If the updated APEX fails verification or causes a boot loop, apexd rolls back to the pre-installed version.

# 2.15 Debugging & Tracing AOSP
# ==========================================

What are the logcat log buffers?	main (app logs), system (system framework logs), crash (crash dumps), events (binary event logs), kernel (kernel messages). Access specific buffers with logcat -b <buffer>.
What are the logcat log levels?	Verbose (V), Debug (D), Info (I), Warn (W), Error (E), Fatal/Assert (F/A). Filter with logcat *:W to show only Warning and above.
What does 'adb shell dumpsys' do?	Calls the dump() method on a system service's Binder, outputting its current state. Example: dumpsys activity (AMS state), dumpsys battery, dumpsys window (WMS state), dumpsys meminfo.
What is Perfetto in the AOSP context?	A system-wide tracing infrastructure that captures kernel ftrace events, atrace categories, custom user-space trace points, and hardware counters. Analysis via SQL queries on the Trace Processor.
What is ftrace and how does atrace relate to it?	ftrace is the Linux kernel trace framework. atrace is Android's wrapper that enables specific ftrace categories (sched, gfx, input, etc.) and adds userspace tracing via ATRACE macros.
What is Simpleperf?	Android's CPU profiling tool (similar to Linux perf) that uses hardware PMU counters. Produces call graphs, flame charts, and per-function CPU samples for both Java and native code.
What is a tombstone in Android?	A crash dump file generated when a native process crashes (SIGSEGV, SIGABRT). Contains register state, stack trace, memory maps, and backtrace. Stored in /data/tombstones/.
How do you remotely debug native code in AOSP?	Use gdbserver or lldb-server on the device, forward ports via adb, and attach GDB/LLDB from the host. Android Studio also supports native debugging via LLDB.
What does bugreport/bugreportz contain?	A comprehensive device state capture: logcat dumps (all buffers), dumpsys of all services, dmesg, proc filesystem state, ANR traces, battery stats, running processes, and more — zipped for analysis.
What is the difference between dumpsys and dumpstate?	`dumpsys` queries a single system service's Binder dump() method and prints its internal state (e.g., `dumpsys activity`, `dumpsys battery`). `dumpstate` is a comprehensive tool that collects the entire device state — it calls dumpsys on every service, plus captures logcat, dmesg, proc info, ANR traces, and more. bugreportz invokes dumpstate internally.
What is showmap and how does it help debug memory?	`showmap` displays a detailed per-mapping memory breakdown for a process, showing virtual size, RSS, PSS, and shared/private clean/dirty pages for each memory region (stack, heap, .so files, dex, etc.). It is more granular than `dumpsys meminfo` and helps identify which specific libraries or allocations are consuming the most physical memory.
What is the Android Debug Bridge (adb) architecture?	adb has three components: (1) the adb client on the host (CLI commands), (2) the adb server on the host (manages connections, runs as a background process on port 5037), and (3) the adbd daemon on the device (runs as root or shell user). The client communicates with the server via TCP, and the server communicates with adbd over USB or TCP/IP. The server multiplexes multiple client connections to multiple devices.
What are atrace categories and how do you enable custom ones?	atrace categories (e.g., gfx, input, view, sched, freq) map to ftrace tracepoints and userspace ATRACE macros. Enable them with `atrace --list_categories` and `atrace -c <cat1> <cat2>`. To add custom categories, use `ATRACE_BEGIN("label")`/`ATRACE_END()` in C++ or `Trace.beginSection("label")`/`Trace.endSection()` in Java, and register a new category string in atrace's category list.
What is the difference between Perfetto and systrace?	systrace is the legacy tracing tool that uses atrace + ftrace and produces an HTML trace viewable in Chrome. Perfetto is the modern replacement — it uses the same kernel ftrace backend but adds a daemon (traced), protobuf-based trace format, SQL-based analysis (Trace Processor), and support for custom data sources, heap profiling, and long traces. Perfetto supports streaming to disk and has much lower overhead. systrace is essentially deprecated in favor of Perfetto.

# ==========================================
# 2.16 Kernel & Low-Level
# ==========================================

What is GKI and why was it introduced?	Generic Kernel Image — a Google-maintained common kernel that separates core kernel functionality from vendor modules via a stable KMI. This allows kernel updates independent of vendor drivers.
What is KMI (Kernel Module Interface)?	A set of exported kernel symbols with a stable ABI that vendor modules can depend on. Google monitors KMI stability — breaking changes are not allowed within a KMI generation (e.g., android14-6.1).
How is KMI stability monitored?	Using ABI monitoring tools that compare the list of exported symbols and their type signatures. Patches that break KMI symbols are rejected from the GKI branch.
What is eBPF and how is it used on Android?	Extended Berkeley Packet Filter — programs running in the kernel VM for network traffic monitoring, per-UID data usage tracking, power stats collection, and CPU frequency tracking. Used by netd and system health HAL.
What are cgroups in Android?	Linux control groups used to limit, isolate, and prioritize resource usage (CPU, memory, I/O) for groups of processes. Android uses cgroups for process scheduling and memory management.
What is LMKD (Low Memory Killer Daemon)?	A userspace daemon that monitors memory pressure via PSI (Pressure Stall Information) or vmpressure and kills processes based on their oom_adj_score to free memory, replacing the kernel-side LMK.
What is oom_adj_score?	A priority value (-1000 to 1000) assigned to each process by AMS. Lower score = higher priority (foreground = 0, cached = 900+). LMKD kills highest-score processes first when memory is low.
How did Android migrate from ION to DMA-BUF heaps?	ION was an Android-specific kernel memory allocator. The migration replaced ION with the mainline Linux DMA-BUF heap framework, which provides the same functionality through standard Linux APIs.
What replaced ashmem in modern Android?	memfd_create — a standard Linux system call that creates anonymous file descriptors for shared memory. Unlike ashmem, memfd is part of mainline Linux.
What is binderfs?	A filesystem (/dev/binderfs/) for dynamically creating Binder device nodes, replacing the static /dev/binder devices. Used to support multiple Binder domains in containers and virtualization.
What is seccomp-bpf in the Android kernel?	Secure computing with BPF filters — a kernel feature that restricts which system calls a process can make. Android applies seccomp-bpf policies to app processes to minimize kernel attack surface.
What is IMA (Integrity Measurement Architecture)?	A kernel security feature that measures file integrity by computing and verifying hashes of files before they are loaded/executed. Used for verified execution and supply chain integrity.
