# Android Jetpack
# Format: Front[TAB]Back

What is Android Jetpack?	A suite of libraries, tools, and architectural guidance to help developers write high-quality Android apps more easily.
What is ViewModel?	A Jetpack class that stores and manages UI-related data in a lifecycle-conscious way, surviving configuration changes like screen rotation.
What is LiveData?	An observable data holder class that is lifecycle-aware; it only notifies active observers (those in STARTED or RESUMED state).
What is the difference between `LiveData` and `StateFlow`?	`LiveData` is lifecycle-aware and Java-friendly; `StateFlow` is a Kotlin coroutine construct with an initial value, used in viewModelScope, offering more flexibility.
What is Room?	A Jetpack persistence library that provides an abstraction layer over SQLite, enabling compile-time SQL verification and object mapping.
What are the three main components of Room?	`@Entity` (table), `@Dao` (data access object with queries), `@Database` (database holder and entry point).
What is the Navigation component?	A Jetpack library for implementing navigation between destinations (fragments/activities) using a navigation graph, NavController, and NavHostFragment.
What is WorkManager?	A Jetpack library for deferrable, guaranteed background work that runs even if the app exits or the device restarts.
What is Hilt?	A dependency injection library for Android built on top of Dagger. It simplifies DI by providing standard components and annotations.
What is DataStore?	A Jetpack data storage solution that replaces SharedPreferences, offering asynchronous, consistent, and transactional data storage using Kotlin coroutines and Flow.
What is Paging 3?	A Jetpack library for loading and displaying pages of data from a larger dataset efficiently, with built-in support for RecyclerView and Flow.
What is the difference between `@ViewModelScope` and `@LifecycleScope`?	`viewModelScope` is tied to the ViewModel's lifetime; `lifecycleScope` is tied to a lifecycle owner (Activity/Fragment) and is cancelled when the lifecycle is destroyed.
What is `by viewModels()` in a Fragment?	A Kotlin property delegate that lazily creates and retrieves a ViewModel scoped to the Fragment.
What is `by activityViewModels()`?	A Kotlin property delegate in a Fragment that retrieves a ViewModel scoped to the parent Activity, enabling data sharing between fragments.
